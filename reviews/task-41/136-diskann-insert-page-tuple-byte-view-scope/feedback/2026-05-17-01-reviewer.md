# Reviewer Feedback — 136 DiskANN Insert Page-Tuple Byte View Scope

Verdict: ⚠️ **concerns, non-blocking**.

## Confirmed

- `with_page_tuple_bytes` + `DuplicateBindApplyOutcome` enum centralizes the
  slice construction; outcomes mapped to NoChange/Changed/Retry preserve the
  prior `continue`/`break` semantics (`src/am/ec_diskann/insert.rs:1706–1718`).
- Backlink tuple-byte views are now closure-scoped.

## Concerns (non-blocking)

- `with_page_tuple_bytes` signature is `FnOnce(*mut u8, &[u8]) -> Result<R, String>`.
  Handing both a `*mut u8` and a `&[u8]` over the same memory is pedantically
  unsound: writing through `*mut u8` while `&[u8]` is live violates Rust's
  aliasing rules. Today no caller reads the borrow and writes the pointer
  concurrently, so behaviorally safe, but the helper API does not enforce it.
  Recommend a follow-up packet to split into read-only (`FnOnce(&[u8]) -> R`)
  vs writable (`FnOnce(&mut [u8]) -> R`) helpers.
- Same shape as 137's `with_vacuum_page_tuple_bytes` (`routine.rs:1834`); both
  should be unified once the read/write split is in place.
