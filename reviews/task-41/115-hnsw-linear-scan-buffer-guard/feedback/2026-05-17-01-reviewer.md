## Summary

Uses `LockedBufferGuard` in `src/am/ec_hnsw/scan.rs` for the
HNSW linear fallback scan path:
- PG18 `read_stream_next_buffer` linear scan buffers (via
  `lock_pinned`)
- non-PG18 `ReadBufferExtended` linear scan buffers (via
  `read_main`)

## Confirmations

- **PG18 + non-PG18 parity.** Same primitive serves both
  paths — `lock_pinned` for read-stream buffers, `read_main`
  for the direct read.
- **Scoring outside the lock.** Selected tuple payload is
  copied into owned Rust data, then guard drops, then
  `score_scan_element_result` runs on the owned data. No
  buffer pin held across scoring — minimizes lock contention.
  ✔
- **Selector helper migrated.** `select_linear_scan_result_from_buffer`
  no longer calls raw `LockBuffer` / `BufferGetPage` /
  `BufferGetPageSize`; uses `guard.page()` / `guard.page_size()`
  instead.
- **Graph-prefetch path untouched.** The remaining raw
  `LockBuffer` calls in `scan.rs` are the separate
  graph-prefetch path, not the linear fallback — request
  correctly flags this scope boundary.
