# Reviewer Feedback — 115 HNSW Detoast Guard-Owned Borrows

Code commit reviewed: `ffe7338d`. Verdict: **no blockers**; the slice closes
the "who owns the detoast pfree" half of HNSW invariant #2.

## Confirmed

- `DetoastedFloat4Datum` (`source.rs`) is a correct RAII guard: it carries
  `varlena: *mut pg_sys::varlena` plus `owned: bool` (`!ptr::eq(varlena, original)`),
  and `Drop` pfrees only when `owned`. Matches the prior `(ptr, owned)` pair
  but moves the lifetime into a single owner.
- The change to `FlatFloat4ArrayRef<'datum>` / `FlatFloat4VarlenaRef<'datum>`
  to hold `_detoasted: DetoastedFloat4Datum` as a field correctly transfers
  ownership. The raw `data_ptr` captured during construction remains valid
  after the move into `Self` because it points into palloc'd memory, not into
  the moved guard. The wrapper's `Drop` removal is fine — the guard's `Drop`
  runs on field drop.
- The `as_bytes(&self) -> &[u8]` and `as_array_ptr(&self) -> *mut ArrayType`
  helpers tie the borrow to `&self`, so callers cannot outlive the guard
  through these accessors.
- Error labels (`ec_hnsw does not support NULL …`, `ec_hnsw could not detoast …`,
  `ec_hnsw {label} bytea payload …`) are preserved verbatim.

## Concerns (non-blocking)

- **`pgrx::error!` between detoast and `Self {…}`** (validation branches in
  `FlatFloat4ArrayRef::from_datum` / `FlatFloat4VarlenaRef::from_datum`) does
  not run the local `DetoastedFloat4Datum::drop` if pgrx falls back to
  `longjmp` rather than unwinding. In ECAZ today the task contract assumes
  `pg_guard` + pgrx unwinds Rust frames first, so Drop should run; but in any
  fallback path the palloc'd copy is reclaimed by Postgres memory-context
  cleanup on abort, so this is not a real leak — only a property worth a
  one-line comment near `DetoastedFloat4Datum` describing the dual reliance
  (Rust Drop on the happy path, MemoryContext cleanup on ERROR fallback).
- Once 117 (HNSW build detoast) and 118 (DiskANN build detoast) land, the
  `DetoastedFloat4Datum` / `DetoastedBuildDatum` shapes can be consolidated
  into a shared `crate::am::detoast` helper. Not a blocker for this packet.
