   DiskANN closure helper is `impl FnOnce(&[f32]) -> T`, not
   higher-ranked, so the type system does not prevent `T = &'a [f32]`
   escape. The current caller is safe by inspection. Mechanical fix:
   change the signature to `for<'a> FnOnce(&'a [f32]) -> T` so it
   matches the 114 pattern. Recommend a small follow-up packet.

   `DetoastedBuildDatum`/`DetoastedFloat4Datum`/`DetoastedEcvectorDatum`
   /`DetoastedScanDatum` types now exist across HNSW/IVF/DiskANN/SPIRE.
   The strategy doc says ≥2 packets justify extraction; this threshold
   has clearly passed. Worth a single consolidation packet into
   `crate::am::detoast` (or equivalent), keeping per-AM error labels via
   a `&str` parameter. Reduces future drift risk.

3. **123 contract wording**: clarify that `pgrx::error!` does run Drop
   (via Rust panic), and memory-context cleanup is the longjmp fallback,
   not the primary mechanism. See 123 feedback for suggested wording.

4. **Strategy doc could state explicitly** that "scoped closures must be
   HRTB", to make the 118 follow-up rule durable.
