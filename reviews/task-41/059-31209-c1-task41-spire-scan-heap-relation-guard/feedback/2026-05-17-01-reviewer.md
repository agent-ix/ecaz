# Review: Task 41 SPIRE scan heap relation guard (31209)

## Summary

Refactors `ResolvedScanHeapRelation` in
`src/am/ec_spire/scan/relation.rs` from a `bool owned` flag +
manual `Drop` impl to **`Option<HeapRelationGuard>`** — the
owned case stores the shared guard, the borrowed case stores
`None`. Local `Drop` impl deleted. Baseline neutral (4,140 →
4,140).

This is a clean realization of my 31192 finding #1 suggestion
(the "Borrowed vs Owned" wrapper pattern). The coder used
`Option<Guard>` rather than an enum — equivalent semantics,
simpler.

## Confirmations

- **Owned heap relations close via `HeapRelationGuard::Drop`.**
  When `owned(guard)` constructs the wrapper, the guard is moved
  into `_owned: Some(guard)`. When the wrapper drops, the field
  drops, the guard drops, `table_close` runs. ✔
- **Borrowed relations not closed.** `borrowed(relation)` stores
  `_owned: None`. The wrapper's drop is a no-op for the inner
  raw pointer; the caller retains close responsibility (or the
  pointer was scan-owned and closed elsewhere). ✔
- **`relation` derived from guard before storing.**
  `let relation = guard.as_ptr();` runs before the move, so the
  stored raw pointer is valid for the wrapper's lifetime
  (which is the same as the moved-in `Option<Guard>`'s
  lifetime). ✔
- **`_owned` underscore prefix.** Standard Rust idiom for
  "kept for Drop, not read directly" — silences unused-field
  lints without `#[allow]`.

## Findings

### 1. Encoding "borrowed vs owned" via `Option<Guard>` is the right call

The prior `bool owned` + manual Drop tied the two pieces of
state together by convention; one bit of state told the Drop
impl whether to close. With `Option<HeapRelationGuard>`, the
state and the Drop logic are unified — the type system handles
it. No conditional inside Drop. No risk of someone setting
`owned = true` but not actually opening the relation.

Pattern worth replicating anywhere else a wrapper needs to
optionally own a resource. The `OpenedStoreRelation` /
`OpenedRelationsGuard` shape from 31174 (still pending in my
status report) could benefit from this exact treatment.
