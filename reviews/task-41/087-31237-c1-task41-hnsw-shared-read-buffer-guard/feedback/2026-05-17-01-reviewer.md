# Review: Task 41 HNSW shared read buffer guard (31237)

## Summary

Adds `LockedBufferGuard::lock_pinned(buffer, lockmode)` — a
third constructor that takes an already-pinned PG buffer and
locks it. Uses the guard in `src/am/ec_hnsw/shared.rs` for
`count_element_tuples`, `highest_level_live_entry_candidate`,
`read_metadata_page`.

## Confirmations

- **Third constructor on `LockedBufferGuard`.** Now covers:
  - `read_main`: read + lock (most common)
  - `read_main_locked`: read with already-locked semantics (RBM_ZERO_AND_LOCK)
  - `lock_pinned`: adopt pre-pinned buffer + lock it (read-stream + lock pattern)
  
  Three constructors, one Drop. The `lock_pinned` case
  complements `PinnedBufferGuard::from_pinned` from 31235 —
  one for pin-only ownership, one for pin-plus-lock ownership
  starting from a pre-pinned buffer.
- **PG read-stream + lock idiom.** When a read-stream hands a
  pre-pinned buffer and the caller needs exclusive access,
  `lock_pinned` is the right adapter — caller doesn't need to
  re-derive the pin lifetime separately from the lock.

## Findings

### 1. Constructor surface is filling out cleanly
