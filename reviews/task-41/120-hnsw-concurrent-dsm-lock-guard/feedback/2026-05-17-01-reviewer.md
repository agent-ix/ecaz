# Review: Task 41 HNSW concurrent DSM lock guard (120)

## Summary

Adds `EcHnswConcurrentDsmLockGuard` to
`src/am/ec_hnsw/build_parallel.rs` — a **module-local RAII
wrapper for LWLock release**. This is **the first LWLock guard
work in the codebase**.

## Confirmations

- **LWLock ownership.** Guard owns a `*mut pg_sys::LWLock` plus
  a function pointer to the release function. Drop calls
  `(release)(lock)`. This handles the `LWLockRelease` /
  `LWLockReleaseAll` distinction via the function-pointer field.
- **Module-local scope.** Only the HNSW concurrent DSM builder
  uses this for now.

## Findings

### 1. (significant) First LWLock wrapper — should be shared

Task 41's task doc explicitly names `LwLockGuard` as one of the
four RAII wrapper families for invariant #3:

> 4. **Resource release.** Wrap PG resources in RAII types:
>    - `BufferPin` (drops via `ReleaseBuffer`),
>    - **`LwLockGuard` (drops via `LWLockRelease`)**,
>    - `Snapshot` (drops via `UnregisterSnapshot`),
>    - `SpiTuptable` (drops via `SPI_freetuptable`).

This packet introduces an LWLock guard as **module-local**.
Same situation as the early relation-guard work (31172/31173)
that ended up needing consolidation in 31180. Suggest a
follow-up packet that:

1. Promotes the guard to `src/storage/lock_guard.rs` as
   `LwLockGuard`.
2. Generalizes the function-pointer-release pattern into
   constructor variants (`acquire_shared(lock)`,
   `acquire_exclusive(lock)`, etc. — each setting the
   appropriate release function internally).
3. Migrates this caller to the shared type.

Doing this *now* — when there's exactly one caller — is the
cheapest moment. By the time LWLock usage spreads (parallel
build, shared metadata, SPIRE coordinator), three or more
module-local copies will exist and consolidation gets harder.

### 2. Function-pointer release shape

The `release: unsafe fn(*mut pg_sys::LWLock)` field is one way
to handle the lock-mode-specific release call. An alternative
is to store the lockmode and dispatch internally:

```rust
pub(crate) struct LwLockGuard {
    lock: *mut pg_sys::LWLock,
    mode: pg_sys::LWLockMode::Type,
}
impl LwLockGuard {
    pub(crate) fn acquire(lock: ..., mode: ...) -> Self { ... }
}
impl Drop for LwLockGuard {
    fn drop(&mut self) {
        unsafe { pg_sys::LWLockRelease(self.lock) };
    }
}
```

`LWLockRelease` releases any held mode — no dispatch needed.
The function-pointer-release shape in this packet is fine but
slightly heavier than necessary. Worth simplifying when
promoting to shared.
