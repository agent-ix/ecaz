# Review: Task 41 DiskANN vacuum heap relation guard (31216)

## Summary

Replaces the `(Relation, owned: bool)` convention + manual
`release_owned_vacuum_heap_relation` helper with
**`ResolvedVacuumHeapRelation`**, which stores
`Option<HeapRelationGuard>` for owned heap relations and `None`
for borrowed. Same pattern as 31209's `ResolvedScanHeapRelation`
refactor. Production vacuum path.

## Confirmations

- **`Option<Guard>` encoding replaces `bool owned`.** Same
  type-driven simplification as 31209 — state and Drop logic
  unified, no conditional inside Drop, no risk of an
  inconsistent `owned = true` + no-actual-open state.
- **Production vacuum path.** This is in DiskANN's vacuum
  callback chain — every DiskANN vacuum cycle goes through
  this. Mechanically and semantically equivalent to the prior
  flag-based code; the cleanup is now type-enforced.
- **Borrowed relations not closed.** `_owned: None` case stores
  a raw pointer from a caller-owned relation; wrapper's Drop is
  a no-op for that arm.

## Findings

No blockers. The `Option<Guard>` pattern is now applied to two
production wrappers (this + 31209) — convention is established
for any future "may-or-may-not-own" wrapper.
