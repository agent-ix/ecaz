## Confirmed

- `DetoastedTypmodArray` Drop pfrees the typmod array copy on every exit
  path. Manual pfree no longer needed in the validation branches.
- `typmod_in` is only invoked during DDL / column-type parsing, so the
  guard's lifetime never crosses statement-end.
- Existing user-facing error messages preserved.
- pgrx::error! sites between `from_datum` and `Self {…}` rely on pgrx
  catching the ereport as a Rust panic and unwinding Drop before re-raising;
  in the alternative longjmp-pass-through fallback the typmod context cleans
  up at txn abort. Both paths are leak-free.

## Concerns (non-blocking)

- `as_ptr(&self) -> *mut ...` returns a raw pointer without tying it to
  `&self`; a caller could in principle keep it past Drop. Only used inline
  today, but consider returning a borrow or making the field private and
  forcing all access through `&self`-bound accessors for symmetry with the
  other guards.
