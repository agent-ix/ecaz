## Cutouts / Shortcuts identified (all non-blocking, none unsound today)

1. **Writable-page helper aliasing.** `with_writable_page_tuple_bytes`
   (143) and `with_writable_vacuum_tuple_bytes` (144), and the DiskANN
   equivalents `with_page_tuple_bytes` (`insert.rs:1706`) and
   `with_vacuum_page_tuple_bytes` (`routine.rs:1834`), pass `*mut u8` and
   `&[u8]` over the same memory into the closure. Today no caller reads
   the borrow and writes the pointer concurrently, so the soundness
   problem is API-shape, not behavior. Recommend a follow-up packet to
   split read-only (`FnOnce(&[u8]) -> R`) from writable
   (`FnOnce(&mut [u8]) -> R`) variants, then consolidate the four helpers
   into shared modules.
2. **No HRTB on the new closure helpers.** The page-byte-view helpers,
   the DSM scoped helpers (131), and `with_ecvector_datum_slice` (118)
   all use `impl FnOnce(&[u8]) -> R` (or `&[f32]`). Lifetime elision is
   correct today because the slice originates from a function-arg-derived
   lifetime, but a uniform `for<'a> FnOnce(&'a [u8]) -> R` would prevent
   future drift, matching the 114 pattern.
3. **Three return-signature styles** across page-byte-view helpers:
   IVF page returns `Result<PageTupleVisit<R>, String>`; HNSW shared
   returns `Option<R>` and `pgrx::error!`s on OOB; DiskANN returns
   `Result<R, String>`. Unify in a follow-up.
4. **`Detoasted*Datum` duplication.** Five structurally identical types
   across HNSW/IVF/DiskANN/SPIRE. Consolidation packet was suggested in
   121's closeout; still outstanding.
5. **`scan_debug.rs` debug helper** (145) duplicates
   `shared::with_page_line_tuple_bytes`. Wrap the shared helper instead.
6. **Leader-setup DSM `from_raw_parts_mut`** at
   `build_parallel.rs:1056/1064/1071`: single-writer init context, safe
   by phase, but not closure-scoped like the runtime paths from 131.
   Worth wrapping for symmetry.
7. **`uuid_source_identity_payload`** at `ec_spire/build/drafts.rs:537`
   reads `from_raw_parts(datum.cast_mut_ptr::<u8>(),
   SPIRE_STABLE_GLOBAL_SOURCE_ID_PAYLOAD_BYTES)` directly and copies into
   an owned `[u8; N]`. Synchronous copy from a uuid Datum's
   pass-by-reference payload — safe, but not routed through a
   `Detoasted*` guard. Not called out explicitly in the inventory's
   accepted categories; either add a "fixed-size Datum payload copy"
   bullet or wrap in a tiny RAII helper.
8. **123 contract comment wording.** The pgrx unwind vs longjmp
   distinction is still inverted in the contract comment per 123's
   feedback; rewording recommended.
