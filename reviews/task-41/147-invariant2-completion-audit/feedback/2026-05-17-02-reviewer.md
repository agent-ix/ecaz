## Action Items

### A. Split writable-page helpers into read-only / writable variants

**Why:** `with_writable_page_tuple_bytes` (`src/am/ec_hnsw/insert.rs:1210`),
`with_writable_vacuum_tuple_bytes` (`src/am/ec_hnsw/vacuum.rs:722`),
`with_page_tuple_bytes` (`src/am/ec_diskann/insert.rs:1706`), and
`with_vacuum_page_tuple_bytes` (`src/am/ec_diskann/routine.rs:1834`) all
hand the closure both `*mut u8` and `&[u8]` over the same memory. This is
aliasing-pedantic UB and `cargo miri` can flag it.

**What:** Change each helper signature to either:

- read-only:  `unsafe fn …<R, F>(…, visit: F) -> Result<R, String>
                where F: for<'a> FnOnce(&'a [u8]) -> Result<R, String>`
- writable:   `unsafe fn …<R, F>(…, visit: F) -> Result<R, String>
                where F: for<'a> FnOnce(&'a mut [u8]) -> Result<R, String>`

For sites that currently do "read, decide, then write back", split the
caller into a read closure that returns a `Decision { …owned bytes/struct… }`,
followed by a writable closure that copies the decision back. Both
closures should re-acquire the line pointer offset/length internally so
each owns its slice exclusively.
