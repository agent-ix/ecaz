## Concerns (non-blocking)

- **Wording slightly imprecise.** The current text contrasts "Drop frees
  copied varlena on normal Rust paths" against "memory-context cleanup
  covers ERROR abort fallbacks", which reads as if `pgrx::error!` itself
  is the fallback path. In the current pgrx, `pgrx::error!` triggers a
  Rust panic that DOES unwind through Drop before pgrx catches and
  re-raises the ereport. The path where Drop is bypassed is the
  longjmp-out-of-C case where no Rust-frame unwind happens at all (e.g.
  an inner C function ERRORing without `pg_guard`). Suggested rewrite:

      // Detoast copies are palloc-owned. Drop runs whenever Rust frames
      // unwind (including `pgrx::error!`, which raises a Rust panic
      // before re-raising the PG ERROR), freeing the copy. As a
      // belt-and-braces fallback, PostgreSQL memory-context cleanup at
      // (sub)txn abort reclaims any copy whose Drop did not run because
      // control longjmp'd out without unwinding Rust frames.

  The current wording is not wrong in conclusion (no leak), but it
  obscures which mechanism does the work on the happy ERROR path.

- Style nit: comments are interleaved between `#[derive(Debug)]` and the
  `struct` keyword. Placing the doc comment above the derive is the
  conventional position and renders correctly in rustdoc.

## Closing Note

The factual content is correct; rewording would make the contract easier
for future readers to reason about. Not a blocker.
