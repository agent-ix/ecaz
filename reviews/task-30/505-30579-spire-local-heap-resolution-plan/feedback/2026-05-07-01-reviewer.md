# Local heap resolution plan review

First executable surface that decodes the 6-byte coordinator-local
row locator into heap block/offset work items. No blocking
findings.

## Confirmations

1. Reuses `remote_search_coordinator_local_candidates_result`,
   which fail-closes on any nonlocal target before opening the
   object store — so the heap-resolution path can never be invoked
   on a remote-target plan today.
2. `ItemPointer::decode(&candidate.row_locator)` is the right
   inverse of `remote_search_row_locator`'s `encode_into`
   (`hierarchy_snapshots.rs:90-96`). Encoding/decoding round-trip
   is structurally guaranteed by the shared `ItemPointer` codec.
3. Status `ready` is correct for the local rows; remote rows
   never enter this surface because the upstream planner errors
   first.

## Findings

- **Locator decode error path.** If a local row locator ever fails
  to decode (corrupt bytes), the helper bubbles an Err out of the
  `map`/`collect` chain (line 410-424), which becomes a
  `pgrx::error!` at the SQL boundary. Good. But the failure
  message will be the raw `ItemPointer::decode` error string,
  which doesn't mention which `pid` / `vec_id` produced the bad
  bytes. Adding context to the decode error (similar to other
  helpers in this file) would help operators triage.

- **Pipeline redundancy with the summary surface.** Same root
  cause as 30577 — see cross-cutting on 30584.
