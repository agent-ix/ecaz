## What's solid (deep-read three packets)

- **`704` intent transitions**. The right shape. Encoding allowed
  transitions as explicit context enum + validator function + test-only
  invariant gives three orthogonal lines of defense against state-machine
  drift. A regression that flips an allowed transition would now fail
  the validator unit tests **and** the test-build invariant — not just
  one of them. The validator is a non-FFI helper, so test-only
  invariants don't bleed into production code paths.
- **`705` recall fixtures**. Brute-force reference set is constructed
  in the same test as the indexed query, so a regression in the
  brute-force helper would show up as a recall=0 not a silent pass.
  Monotonicity assertion uses `<=` on prior recall, catching both
  decreases and accidental zeroes.
- **`701` payload preflight**. Pure-Rust adapter test with an
  intentionally invalid conninfo string is the right pattern — it
  proves the cap fires **before** the connection attempt, which is the
  load-bearing claim of the 12a.2 GUC design. The previously-flagged
  P3 from the 12a final review ("Vec<String> materialized before cap
  check") is now bounded — the per-batch cap fires upstream.
