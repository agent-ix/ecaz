## Findings (non-blocking)

- **Descriptor-state literal `"missing"`.** See 30571 feedback —
  the comparison at `snapshots.rs:854` is a string literal not yet
  in the central registry. Worth migrating once additional
  descriptor states (`stale`, `active_draining`) start appearing
  in the registration flow.

- **`recommendation` strings are duplicated across capability
  plan, descriptor readiness, and node snapshot.** The phrases
  "register remote node descriptor before libpq fanout execution"
  and "register remote node descriptor before capability check"
  appear in multiple files. As recommendations multiply, consider
  whether they belong in the central registry too — or whether
  they should be rendered by a small helper that takes the
  blocker's status string and emits the canonical
  recommendation.
