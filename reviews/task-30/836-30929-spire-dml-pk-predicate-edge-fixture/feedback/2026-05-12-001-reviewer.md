# Review: SPIRE DML PK Predicate Edge Fixture (30929)

## Summary

Closes 30896 H8. Negative-coverage fixture covering exactly the
classifier edge cases I flagged:

- Numeric PK equality with value outside int8 range — should
  reject as type mismatch, not truncate.
- NULL int8 PK equality — stable SQLSTATE.
- `WHERE id IN (...)` — should reject (not bigint equality).
- `WHERE id = $1 OR id = $2` — should reject (not single
  equality).
- Numeric equality not silently accepted via implicit coercion.
- Planner-hook fail-closed for unsupported SPIRE-fronted PK
  SELECT shapes with `feature_not_supported` SQLSTATE.

Bonus: covers the prepared-statement boundary correctly —
testing the prepared numeric parameter at `EXECUTE` time, not
`PREPARE` (PG's planning phase happens at execute for
parameterized statements with unknown parameter values).

The three review-focus questions:
- Fixture covers Phase 12.6 predicate edge cases without
  overclaiming broader isolation/EvalPlanQual coverage: yes.
- Prepared numeric parameter tested at EXECUTE time: yes —
  that's where the classifier actually sees the parameter
  type and value.
- Tracker scoped correctly to PK predicate shapes + SPIRE-
  fronted fail-closed: yes.

## Findings

### P3 — `feature_not_supported` SQLSTATE consistency

The classifier raises `feature_not_supported` (SQLSTATE
`0A000`) for unsupported shapes — same SQLSTATE used by the
embedding-UPDATE rejection (30841) and the DML hook fail-
closed guard (30861). Operator monitoring can grep on a single
SQLSTATE for "SPIRE rejected this DML shape." Good
consistency.

### P3 — Edge cases not enumerated

Worth confirming the fixture also covers (or pins as future
work):
- `WHERE id BETWEEN x AND y` (range, not equality)
- `WHERE id = ANY(ARRAY[...])` (array equality, similar shape
  to IN)
- `WHERE id = (SELECT max(id) FROM other)` (subquery RHS)

Each should reject. The first two have the same shape as IN/OR
covered here; the third should fall to `unsupported_subquery_shape`
from earlier coverage. Worth a one-line note in the fixture
naming the equivalence.

## Reviewer recommendation

Accept. Closes H8 cleanly with comprehensive coverage.
