# Review: SPIRE Coordinator Insert Dispatch Plan (30829)

## Summary

Side-effect-free dispatch readiness primitive that the mutating
INSERT slice will consume. Looks up the remote node descriptor by
classified `node_id`, validates the served-epoch window
(`stale_epoch` if `served > last_served`, `retention_gap` if `served
< min_retained`), checks the conninfo secret resolution status, and
emits one of `requires_remote_node_descriptor`,
`requires_conninfo_secret_resolution`, `stale_epoch`,
`retention_gap`, or `ready`. Returns the provider lookup key (not raw
conninfo), descriptor generation, and remote index regclass for
operator diagnostics.

The four review-focus questions:
- Right boundary before the mutating executor: yes — separates "is
  this dispatchable" from "do the dispatch."
- Status taxonomy matches Stage C: yes — same `requires_*` /
  `stale_epoch` / `ready` shape that the read path uses.
- SQL surface gives operator diagnostics without leaking conninfo:
  yes — only the lookup key surfaces, descriptor generation lets
  the operator correlate with the descriptor catalog.
- Tracker leaves true INSERT/PREPARE TRANSACTION/placement mutation
  open: confirmed.

## Findings

### P3 — Status precedence: epoch before secret

When both an epoch fault and an unresolved secret exist, the helper
surfaces the epoch fault first. That's consistent with Stage C
ordering (descriptor → epoch → secret). Worth a comment naming the
precedence so a future reorder doesn't accidentally promote secret
faults above epoch faults.

### P3 — Descriptor generation surface

`descriptor_generation` is exposed in the result row but not
consulted later in this primitive. It's useful for the mutating
INSERT slice to detect descriptor churn between dispatch readiness
and remote prepare. Worth pinning: either the next slice should
re-validate the generation before sending or the contract should
note that generation drift is tolerated within the same coordinator
txn.

### P3 — Remote index identity bytes returned but not validated here

The row returns `remote_index_identity_bytes` (size only). The
identity payload itself is part of the descriptor. The mutating
slice will need to confirm the identity hasn't drifted between
dispatch and prepare; flag for the next packet.

## Reviewer recommendation

Accept. Pure planning surface; cleanly extensible.
