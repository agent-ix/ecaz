# Review: SPIRE DML CustomScan Primitive Invocation (30882)

from 30870) from CustomScan runtime state inside the executor.
Bundles index OID, DML mode, primitive name, PK column/value,
updated columns, projected columns. Reuses the 30881 metadata
guard before constructing the invocation.

The live PK SELECT CustomScan branch is refactored to route
through the invocation builder before calling
`ec_spire_forward_coordinator_select_tuple_payload(...)` —
behavior-preserving. The shared invocation construction is now
ready to be the entry point for UPDATE/DELETE branches once
their executor wiring lands.

The three review-focus questions:

  primitive contract from 30870: yes — same struct shape, same
  validation rules.
- PK SELECT routed through the invocation is behavior-
  preserving: yes — invocation construction succeeds for the
  PK SELECT happy path, and the existing primitive call shape
  is unchanged.
- UPDATE/DELETE execution remains disabled despite shared
  invocation boundary: confirmed.
