# Review: SPIRE Bulk-Load Registration Docs (30926)

## Summary

Closes 30924 P3. Pure docs:

- `docs/SPIRE_DIAGNOSTICS.md` names the v1 bulk-load primitives
  (`ec_spire_classify_centroid(...)` and
  `ec_spire_register_placement_batch(...)`) so operators can
  find them without reading the ADR.
- Diagnostics runbook clarifies that
  `ec_spire_register_placement_batch(...)` is transactional
  within the calling session: entries become visible together
  at commit or not at all on rollback. Partial-visibility
  warning is now scoped to **across-batch** boundaries, not
  within one registration call.
- ADR-069 mirrors the same wording.

The two review-focus questions:
- Transaction-boundary wording accurately describes
  `register_placement_batch`: yes — that primitive runs INSERTs
  inside the calling transaction, so atomicity follows directly.
- Naming the two primitives in the runbook resolves 30924 P3:
  yes — operators can grep the docs for the function names
  directly.
