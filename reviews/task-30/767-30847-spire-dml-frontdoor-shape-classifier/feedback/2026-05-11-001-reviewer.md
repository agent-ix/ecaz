# Review: SPIRE DML Frontdoor Shape Classifier (30847)

## Summary

Excellent execution of the 30843 hook-boundary recommendation.
`src/am/ec_spire/dml_frontdoor.rs` is the right scaffold:
- A pure-Rust `SpireDmlFrontdoorShapeInput` struct that captures
  every safety-relevant fact about a candidate query.
- `classify_dml_frontdoor_shape(input)` returns a
  `SpireDmlFrontdoorShapeRow` carrying `supported`, `kind`,
  `status`, `error`, `hint`.
- Encodes every fail-closed rule from my 30843 feedback:
  single-table only, bigint PK equality only, no joins / subqueries
  / RETURNING / PK update; embedding UPDATE returns the exact
  ADR-069 error + hint.
- 5 Rust unit tests cover supported v1 shapes plus four classes of
  unsupported shapes. The classifier is unit-testable without a
  live PG instance, which is exactly the right shape for a safety-
  critical decision module.

The diagnostic surface advances from "modifytable_or_view_hook"
(placeholder) to "planner_customscan_hook" naming, matching the
direction.

The three review-focus questions:
- Supported/unsupported shape matrix matches 30843 direction: yes.
- Embedding UPDATE rejection mirrored at the classifier layer
  *and* the primitive guard: yes — defense in depth, exactly what
  I recommended.
- `planner_customscan_hook` is the right diagnostic label: yes.

## Findings

### P3 — `predicate_operator: Option<&'a str>` uses string compare

The classifier compares `predicate_operator == Some("=")`. That's
fine for a pure-Rust shape model, but the actual planner-hook
extraction (next packet) needs to map the PG operator OID to a
canonical string. Worth confirming the extraction layer normalizes
to `"="` exactly (not `"=="`, not `"int8="`, etc.).

### P3 — `ec_spire_distributed_table` precedes other checks

If the table isn't an ec_spire-distributed coordinator table, the
classifier returns `not_distributed_table` without looking further.
That's the right gate ordering (cheapest filter first). Combined
with the relation-context layer (30850), this means the planner
hook can short-circuit on non-marked tables without paying the
shape-classification cost.

### P3 — `ADR_069_HINT` constant

`See ADR-069 for the v1 SPIRE distributed DML shape.` — uniform
hint text across all unsupported_v1 cases. Good. Some cases use
this hint, others (embedding-UPDATE) use the ADR-069 cross-shard-
move hint. Worth ensuring the planner hook surfaces the hint
field correctly when it raises errors.

### P3 — `#[allow(dead_code)]` at the module level

The module is currently dead code outside the unit tests because
the planner hook doesn't call it yet. Will become live with 30849
+ subsequent slices. Worth removing the allow once the hook
consumes the classifier.

## Reviewer recommendation

Accept. This is exactly the scaffolding shape the 30843
recommendation called for: classifier as a pure decision module,
unit-tested in isolation, fed by a separate extraction layer
(30848), invoked from a separate planner hook (30849), and bound
to relation metadata (30850).
