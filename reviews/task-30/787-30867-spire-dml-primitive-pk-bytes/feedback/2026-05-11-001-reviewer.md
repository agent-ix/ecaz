# Review: SPIRE DML Primitive PK Bytes (30867)

- For `ParamBigint(_)`, returns Err with a message naming the
  required executor parameter evaluation step.

Reviewer-confirmed split: const-only path is the call-site for
plan-time evidence (the diagnostic surface), while parameter
evaluation needs runtime context that the executor will provide.

The three review-focus questions:
- Const PK byte conversion as the right narrow executor-boundary
  helper: yes.
- Parameter PK plans fail closed until runtime parameter
  evaluation is wired: yes — explicit error names "evaluate $n at
  runtime" so the executor follow-up has clear scope.
- No planner rewrite or remote DML dispatch implied: confirmed.

## Findings

### P3 — Two helpers (const-only and runtime) coexist

After 30868 lands the runtime helper, the const-only helper
remains useful for:
- The diagnostic SQL surface (no runtime ParamListInfo
  available).
- Plan-time tests that want to assert the byte shape without
  setting up a parameter context.

Worth a one-line comment naming when to use which (const-only =
no runtime ParamListInfo; runtime = executor path).

### P3 — Error message for ParamBigint

The error mentions the next step explicitly. Good operator-facing
trail.
