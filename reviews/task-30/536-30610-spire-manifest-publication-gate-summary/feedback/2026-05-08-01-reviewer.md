# Manifest publication gate summary review

`ec_spire_remote_epoch_manifest_publication_gate_summary` composes
publication readiness + libpq request count + dispatch count +
receive count + receive readiness + executor status + next
blocker into one row. Mirror of the search-side
`coordinator_gate_summary` for the manifest path. No blocking
findings.

## Confirmations

- Single-row gate is the right operator-facing shape: one
  query answers "is the next blocker descriptor / persistence /
  staleness / executor / something else?".
- Status precedence: `not_required` (local-only) → upstream
  blocker → executor-required → ready. Consistent with the
  search-side gate.
