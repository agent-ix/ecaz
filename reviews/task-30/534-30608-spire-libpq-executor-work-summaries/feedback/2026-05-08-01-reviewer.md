# Libpq executor work summaries review

One-row summary over the per-node work plans. Reports
ready/blocked work counts, remote PID counts (search) or bind-ready
counts (manifest), next executor step, executor status, and
effective status. No blocking findings.

## Confirmations

- The search and manifest summaries align on `ready_work_count`,
  `blocked_work_count`, `next_executor_step`, `executor_status`,
  and `status`, so an operator can write a single union query
  against the two paths if needed.
- PG18 coverage exercises both the descriptor-blocked path (zero
  ready work) and the active-descriptor path (one ready work
  row) for both search and manifest.
