# Review: SPIRE DDL Ordering Contract (30931)

## Summary

Closes 30896 H10. Pure docs:

- ADR-069 now spells out the v1 operator DDL sequence:
  1. pause writes and bulk-load placement registration
  2. apply DDL to the coordinator
  3. apply matching DDL to every remote
  4. refresh affected remote-node descriptors
  5. verify descriptor state
  6. resume writes
- `docs/SPIRE_DIAGNOSTICS.md` mirrors the operator runbook
  wording.
- Lightweight DDL-window guard decision: **v1 does not add a
  separate guard GUC/catalog flag.** The operational guard is
  the documented sequence; the planned (still-open) Phase 12.5
  schema-drift fingerprint is the fail-closed safety net for
  violated ordering.

The three review-focus questions:
- Sequence complete enough for v1 operator docs: yes — the
  six steps cover the full DDL-application lifecycle.
- No-separate-guard decision scoped correctly: yes — the
  decision doesn't preclude the Phase 12.5 schema-drift
  fingerprint (still tracked separately).
- Tracker not overclaiming beyond docs/decision: yes.

## Findings

### P3 — Sequence is operator-discipline-driven

The contract relies on operators following the sequence; PG
doesn't enforce it. That's the right v1 tradeoff (a real guard
would require a coordinator-side write-blocking mechanism with
its own complexity). Worth pinning in the runbook that **the
schema-drift fingerprint detection (Phase 12.5, still open)
is the catch-net** when operators get the order wrong. Until
that lands, the ordering is unguarded; document this gap
explicitly so operators don't assume PG will block them.

### P3 — Bulk-load registration paused alongside writes

Step 1 says "pause writes and bulk-load placement registration."
Good — bulk-load is a write path and DDL during bulk load
would corrupt the registration window.

### P3 — Descriptor refresh as step 4

The descriptor refresh after remote DDL ensures the
coordinator's cached identity reflects the new column shape.
Combined with the existing fingerprint check on read, schema
drift surfaces quickly.

## Reviewer recommendation

Accept.
