# Review: SPIRE Prepared Xact Helper Decision (30943)

## Summary

Pure docs slice. Closes the 30914 P3 about
`ec_spire_recover_orphaned_prepared_xacts(node_id)` helper:

- ADR-069 explicitly **defers the helper for v1** because remote
  `pg_prepared_xacts` does not include the affected primary key
  or coordinator transaction outcome needed by the documented
  commit/rollback rule.
- `docs/SPIRE_DIAGNOSTICS.md` tells operators to use the
  manual placement-directory recovery runbook until SPIRE
  records durable prepared-transaction intent metadata.
- Phase 12.4 tracker: helper decision marked complete-as-
  deferred; `max_prepared_transactions` readiness parent
  marked complete now that all child evidence rows are done.

The three review-focus questions:
- Deferring the helper is the right v1 safety decision: yes —
  automated bulk resolution from the remote side alone would
  be unsafe, and adding durable intent metadata is its own
  feature with its own ADR scope.
- Docs make clear that automated bulk resolution from the
  remote side alone is unsafe: yes — explicitly tied to the
  missing PK + outcome metadata.
- Tracker doesn't overclaim implementation beyond the
  documented manual recovery boundary: yes.

## Findings

### P3 — Future intent-metadata ADR

If a future ADR adds durable prepared-transaction intent
metadata (e.g., a coordinator-side
`ec_spire_inflight_2pc_intent` table that records the GID +
PK + intended outcome before issuing PREPARE), the helper
becomes implementable. Worth pinning the deferral in ADR-069's
"future ADRs" list so the metadata contract isn't forgotten.

### P3 — Operator runbook is already manual-recovery-friendly

The 30914 docs already cover manual recovery via
`pg_prepared_xacts` + placement-directory check. Adding the
deferral note here gives operators a complete picture: "this
is what's available now, this is why a helper isn't, this is
what would be needed for the helper."

### P3 — `max_prepared_transactions` parent now complete

With the docs (30921), the SPIRE-named hint (30921), the
descriptor-registration preflight (30923), and the deferral
decision for the recovery helper (30943), the H9 lane is
fully closed.

## Reviewer recommendation

Accept. The deferral is honest: a helper that can't safely
automate recovery without additional metadata shouldn't ship
just to look complete. Documented manual recovery is the right
v1 boundary.
