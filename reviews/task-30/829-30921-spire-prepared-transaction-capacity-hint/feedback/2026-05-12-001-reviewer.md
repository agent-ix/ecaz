# Review: SPIRE Prepared Transaction Capacity Hint (30921)

## Summary

First half of 30896 H9. Documents `max_prepared_transactions` as
required GUC on every remote, and adds a SPIRE-named hint to the
generic PG `PREPARE TRANSACTION` failure message so operators see
"set max_prepared_transactions on the remote" instead of just
"prepared transactions are disabled."

What landed:
- ADR-069 + `docs/SPIRE_DIAGNOSTICS.md` document the
  remote-side GUC requirement.
- Shared remote `PREPARE TRANSACTION` error wrapper used by both
  INSERT and DELETE prepare paths.
- Wrapper detects PG's "prepared transactions are disabled,"
  capacity-exhaustion, and `max_prepared_transactions`-named
  messages and appends the SPIRE remediation hint.
- Rust unit test for the classifier covering expected SQLSTATE
  + message combinations + unrelated false positives.
- Phase 12.4 marks docs + error-hint rows complete; descriptor-
  registration preflight is the second half (30923).

The three review-focus questions:
- Capacity classifier broad enough for disabled and exhausted
  failures without catching unrelated resource errors: yes —
  the unit test covers both directions (matches and false
  positives).
- Sharing the wrapper between INSERT and DELETE prepare paths:
  yes — both run `PREPARE TRANSACTION` and would surface the
  same error. Single wrapper avoids drift.
- Tracker correctly leaves descriptor-registration check open:
  yes — closed by 30923.

## Findings

### P3 — Classifier matching strategy

The classifier matches against PG's standard error messages plus
SQLSTATE. Worth pinning that PG message text can change between
major versions — the classifier should preferably match SQLSTATE
first (deterministic) and use message-text matching only as a
secondary check. Worth confirming the unit test exercises a
SQLSTATE-only match path so future PG version bumps don't drop
the hint.

### P3 — Shared wrapper for INSERT + DELETE

Same `PREPARE TRANSACTION` shape, same potential failures, same
hint. Right factoring.

## Reviewer recommendation

Accept. The 30923 follow-up closes H9's preflight half.
