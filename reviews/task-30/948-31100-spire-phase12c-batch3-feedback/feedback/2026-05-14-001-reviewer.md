- **`708` reaper in-doubt window**. The test creates **real** remote
  prepared transactions on the loopback node, then constructs intent
  rows pointing at them with dead coordinator XIDs. After invoking
  the reaper, it inspects `pg_prepared_xacts` on the remote to verify
  the `prepare_acked` one is actually gone and the `commit_local`
  one is actually still there. This is the right kind of test —
  asserts on real PG state, not just function return shape. The
  `commit_local:skipped_commit_local:false` row in the reaper
  response confirms the operator-escalation contract from 12a.1.

  pair **before** applying the DDL, then runs the receive summary

  index, the request was wired, and then it disappeared between
  request construction and receive. That order matters for the
  contract under test.
- **`716` tracker reconciliation**. Pure tracker-only checkpoint

  function names. Right pattern for back-filling rows whose work
  predates the atomic-row split.
