## Answers to the packet's review questions

- **Right first lifecycle fixture shape before adding
  in-flight DDL timing cases?** Yes. DROP-before-fanout
  is the simplest lifecycle case (the DDL happens before
  the request even forms), so it exercises the basic
  detection path before the more subtle in-flight timing
  case in packet 30790.
- **DROP before fanout correctly maps to
  `remote_index_unavailable` through production candidate
  receive?** Yes. The dropped index can't be resolved by
  `to_regclass(...)` on the remote node, which is the
  exact predicate packet 30732's remote-index-resolution
  fix tests for. Category propagates through the receive
  adapter's failure classification correctly.
- **`lifecycle-pg18` a good CLI split from `fault-pg18`?**
  Yes — definitively. The fault matrix and lifecycle
  matrix from packets 30770 and 30772 are distinct
  contracts with distinct injection mechanisms (fault
  cases inject failures into a stable plan; lifecycle
  cases run DDL operations at specific timing boundaries
  in the request flow). Sharing one subcommand would
  conflate them and force every reviewer to remember
  which case belongs to which matrix.
