### Blocking-like findings (require attention before next slice)

1. **30595 finding #1: generation-monotonicity not enforced.**
   `ec_spire_register_remote_node_descriptor` accepts re-registration
   with the same or lower `descriptor_generation`. Breaks the
   contract claim `must_advance_descriptor_generation` and
   defeats generation as a concurrency-resolution key. Suggested
   fix in 30595 feedback.

2. **30595 finding #2 + 30596 finding #1: descriptor_state filter
   bypass.** `load_remote_libpq_connection_descriptors` does not
   gate on `descriptor_state ∈ {active, draining}`. A `failed` or
   `disabled` descriptor surfaces as
   `conninfo_resolution = ready`, `pipeline_mode =
   libpq_pipeline`. The libpq dispatch plan will then report
   `open_pipeline_and_send_remote_search` for a node operators
   have explicitly disabled. Today no caller writes those
   states; the moment one does, the surface lies. Suggested
   fix in 30595 feedback.

   Both findings share the same code site
   (`root/remote_candidates.rs:1090-1167`) and one fix closes both.

### Non-blocking findings
