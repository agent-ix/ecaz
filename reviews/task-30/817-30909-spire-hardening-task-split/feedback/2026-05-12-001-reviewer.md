## Observations and minor follow-ups

### O1 — Phase 11 not reconciled

Phase 12.1 schedules tracker reconciliation but Phase 11's task
file still has the open Phase 11.6/11.7/11.8 items in their
original locations. Worth adding to Phase 12.1: "Mark Phase
11.6/11.7/11.8 items as 'continued under Phase 12.7/12.8/12.9'
so the Phase 11 file doesn't show pending work that has actually
moved to Phase 12." Otherwise readers of Phase 11 see
double-counted open items.

### O2 — Capacity targets are TBD

Phase 12.9 says "Publish local capacity targets for maximum
remotes, maximum concurrent coordinator queries, ..." but the
numbers themselves aren't in the file. That's correct — they
should land as part of Phase 12.9's benchmarking work. Worth
pinning that the harness packet's exit criterion includes
publishing concrete numbers, not "we measured something."

### O3 — `ec_spire_placement` partitioning is "evaluate"

Phase 12.4 says "Evaluate partitioning `ec_spire_placement` by
`index_oid` if contention evidence shows shared-table hot
pages." Good — investigate-first framing. The decision lever
should be the contention fixture's measured behavior; Phase 12.4
should explicitly state the threshold (e.g., "if p99 INSERT
latency under N parallel writers exceeds Xms, partition") so
the call doesn't get punted.

### O4 — Phase 12.6 EvalPlanQual decision

Phase 12.6 line 177–178: "Decide whether
`ec_spire_custom_scan_recheck` should remain unconditional for
v1 or re-run the primitive for a subset of read paths." This is
a real architectural decision, not just a task. Worth gating it
on a fixture that demonstrates the wrong-results case under
SERIALIZABLE, then making the call. Otherwise the decision is
"defer until somebody hits it."

### O5 — Phase 13 entry gate is binary

Phase 13's entry gate says "Phase 12 is complete OR every
remaining Phase 12 item has an accepted reviewer deferral."
That's the right shape — but worth pinning who can grant a
deferral (presumably the reviewer per AGENTS.md). Otherwise the
gate becomes "the coder unilaterally defers everything."

### O6 — ADR-069 deferred items not enumerated in Phase 12

ADR-069 explicitly defers cross-shard non-vector queries, DDL
propagation, foreign keys, sequences, rebalance, and
multi-coordinator. None of these appear in Phase 12, which is
correct (they're Phase 12+ ADR scope). Worth a one-line cross-
reference in Phase 12's Non-Goals saying "ADR-069 deferred items
remain Phase 12+ scope per their respective future ADRs" so a
future reader doesn't expect to find them here.

### O7 — JSON migration window length unspecified

Phase 12.2 says "Add endpoint negotiation so CustomScan prefers
typed tuple transport when the remote advertises support and
falls back to JSON only during the migration window. ... After
compatibility is sufficient, remove the JSON endpoint from the
production path and drop the `serde_json` dependency." Worth
defining what "sufficient compatibility" means — e.g., "all
clusters within one extension version of the typed-transport
release" or "after one full minor version release with both
endpoints live." Otherwise the JSON path lingers indefinitely.

### O8 — Phase 13 doesn't cite specific datasets

Phase 13's "Define datasets" is generic. Worth pinning concrete
public benchmark datasets (e.g., GIST-1M, SIFT-1B, ANN-Benchmarks
collection) so the AWS report is comparable to other systems'
published numbers. Optional — could land at AWS-packet-time.

## Reviewer recommendation

**Accept the task split.** Phase 12 is exhaustive vs the 30896
findings, the additional carry-over content from Phase 11 is
correctly placed, Phase 13 is appropriately minimal and gated.
The packet sequence preserves priority ordering.

The O1–O8 observations are small refinements for the next docs
touch on these files; none block the planning being usable as-is.

The biggest single ask for Phase 12 to actually deliver value:
pin **measurable exit criteria** for the items that say
"evaluate" or "decide" (O3, O4, O7). Otherwise Phase 12 risks
becoming a long checklist where each item lands as
"investigated, deferred to Phase 13" and the AWS phase opens
without the hardening actually happening.
