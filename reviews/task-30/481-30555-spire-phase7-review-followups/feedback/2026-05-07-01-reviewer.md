## Verified per finding

1. **Insert-path PID dedupe (30544).** `insert.rs:106-127` now dedupes
   `routed` by leaf PID through a `HashSet`, preserving first-occurrence
   order before splitting into primary plus replicas. Route order from
   the recursive router is preserved, so the IP-best leaf still wins
   the primary slot. Defensive against future hierarchies that allow
   shared children.

2. **IP helper consolidation (30550/30551).** `build/top_graph.rs:127,
   232` now call the shared `inner_product` from
   `build/routing_plan.rs:281` (both files are `include!`-merged into
   `build.rs`, so the call is direct). The local
   `top_graph_inner_product` is removed.

3. **`merge_remote_search_candidates` doc-comment (30554).**
   `root/remote_candidates.rs:30-35` adds an explicit rustdoc warning
   that inputs must share one coordinator-scoped vec_id namespace
   until global vec-id encoding lands.

4. **`row_locator` opacity comment (30553).**
   `root/hierarchy_snapshots.rs:90-91` now states locators must only
   be interpreted by the originating node.

5. **Top-graph distance offset (30550).**
   `max_centroid_inner_product` is replaced by `max_centroid_norm_sq`
   (`build/top_graph.rs:229-235`), reducing setup from O(n²) to O(n).

   **Correctness check:** by Cauchy-Schwarz,
   `IP(a, b) ≤ ‖a‖·‖b‖ ≤ max(‖a‖, ‖b‖)² = max(‖a‖², ‖b‖²)`.
   So `max_centroid_norm_sq` is an upper bound on every pairwise IP
   in the node set, and `distance_offset - IP(a, b) ≥ 0` is preserved.
   For a single-node graph the only IP is `‖c‖²`, giving `distance =
   0`, also preserved. The clamp `.max(0.0)` continues to absorb f32
   rounding error.

6. **Non-metric pseudo-distance documented (30550).** Both the design
   doc (`plan/design/spire-top-level-graph.md` paragraph at line ~98)
   and a code comment at `build/top_graph.rs:124-126` state the
   IP-derived pseudo-distance is order-preserving but not a metric,
   and that recall/latency measurement is the justification rather
   than a formal alpha-pruning proof.

7. **Unreachable `vec_id` tie-break dropped (30554).**
   `root/remote_candidates.rs:21` is removed.

8. **`assignment_flags` SQL surface (30553).** `lib.rs:1549, 1593` now
   use `i16` / smallint with a checked conversion.

9. **Top-graph bypass on storage-node remote search (30553).**
   Comment at `scan/candidates.rs:67-69` states selection is the
   coordinator's job and the storage-node never runs top-graph or
   recursive routing itself.

10. **Top-graph bypass on insert (30544).** Comment at
    `insert.rs:103-106` records the intentional asymmetry between
    graph-aware scan routing and recursive insert routing for v1.

11. **Global vec_id precondition (30552).**
    `plan/design/spire-remote-node-model.md` adds explicit
    "production blocker" framing for cross-node merge by raw
    `vec_id` bytes.

12. **Active-epoch-only fanout (30552).** Same design file now states
    that until retained-epoch serving lands, the only valid
    `requested_epoch` is the coordinator's active epoch, and any
    mismatch must fail closed.
