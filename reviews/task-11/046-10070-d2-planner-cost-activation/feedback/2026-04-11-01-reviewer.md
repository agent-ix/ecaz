# Feedback: D2 Planner Activation

**Reviewer:** Codex (GPT-5)
**Date:** 2026-04-11

## Findings

### 1. `compute_amcostestimate` no longer preserves the empty-index gate

**Severity:** High

`compute_amcostestimate` treats `RelationGetNumberOfBlocksInFork(...) <= 0` as the
empty-index check (`src/am/cost.rs:204-211`). That is not the same thing as "0 data
pages" for this AM: block 0 is always the metadata page (`src/am/page.rs:13-14`), and
the existing empty-index rescan coverage already asserts `scan_block_count == 1`
(`src/lib.rs:3523-3529`).

So an actually empty tqhnsw index falls through to the live model with
`index_pages = 1`, `dimensions = 0`, and a tiny finite cost instead of the `f64::MAX`
gate required by FR-020's error-condition table for "Empty index (0 data pages)"
(`spec/functional/FR-020-cost-estimation.md:125-133`). That means D2 can make empty
indexes planner-visible even though the request says this case stays preserved.

This needs a real "no data pages" check (`block_count <= FIRST_DATA_BLOCK_NUMBER` or
equivalent) before the live model runs, plus a regression on an actually empty index
through the snapshot or `EXPLAIN` path.

### 2. AC-1 is still not proven end-to-end

**Severity:** Medium

The new pg test `test_tqhnsw_planner_chooses_index_scan_for_ordered_query`
(`src/lib.rs:1526-1568`) uses a 3-row table and forces `enable_seqscan = off`
(`src/lib.rs:1541`). That demonstrates the AM can be chosen when seqscan is disabled,
but it does not satisfy FR-020-AC-1 or TC-206, both of which require a natural
`EXPLAIN` index choice on a 10K-row table
(`spec/functional/FR-020-cost-estimation.md:137-138`, `spec/tests.md:213-215`).

The pure crossover unit test is useful, but it does not replace the acceptance test
because it bypasses the real planner integration entirely. I would not mark AC-1 closed
until there is one spec-matching end-to-end artifact for the 10K case.

## Verdict

Not ready to sign off yet. The activation path is close, but the empty-index guard
needs correction and AC-1 still lacks the end-to-end evidence the spec currently
requires.
