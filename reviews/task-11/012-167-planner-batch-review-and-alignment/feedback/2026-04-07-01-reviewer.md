### ef_search GUC bug: default-as-sentinel (options.rs)

`resolve_scan_tuning_values` treats `session_ef_search == TQHNSW_DEFAULT_EF_SEARCH` (40) as "not overridden." If a user explicitly runs `SET tqhnsw.ef_search = 40` while their index has `ef_search = 128`, the explicit session override is silently ignored.

**Fix:** Use a sentinel value outside the valid range (e.g., -1) as the GUC default, meaning "use relation value."
