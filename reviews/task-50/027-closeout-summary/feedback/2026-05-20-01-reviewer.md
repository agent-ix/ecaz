# Conditional approve: Task 50 closeout summary — pending two narrow items

## Decision

The closeout content is correct: 15 / 15 top-15 files at the 30%
exit target, follow-on candidates named, soundness incidents and
their fixes recorded. Approving the summary substantively.

Two open items before I would mark Task 50 fully closed (both
called out honestly in the request.md itself):

1. **SIMD kernel bench evidence (packets 018 + 022).** The closeout
   acknowledges this gap:
   > "the remaining action is a post-change ecaz bench suite
   > comparison ... plus the SIMD kernel bench battery named in
   > the baseline manifest."

   `local-bench-plan.md` §SIMD slices makes this a *requirement*
   for any SIMD-touching slice, not a nice-to-have. `hadamard.rs`
   (packet 018) is the canonical SIMD slice; `ec_hnsw/source.rs`
   (packet 022) consolidates SIMD lane loads. A local x86_64 IAI
   capture (`iai_hadamard`, `iai_quant_score`) on
   the current head, diffed against
   `benchmarks/task-50-local-baseline/`, closes the contract. A
   ≈0% instruction-count delta is *the* expected evidence; if
   non-zero, the gap is interesting.

   The 028 ecaz bench suite smoke is a higher-level recall+latency
   smoke and partially mitigates this — see the 028 feedback. But
   it does not satisfy the SIMD-specific requirement on its own.

2. **A note on the soundness anti-pattern.** The closeout's "Notes"
   section accurately records the packet 024 → 025 sequence. I'd
   suggest expanding by one paragraph to capture the pattern as
   *template* for future task-50-style refactors:

   > Future refactor pattern: when a helper's input is a raw PG
   > pointer (`pg_sys::Relation`, `*mut EcParallelScanState`,
   > `pg_sys::IndexScanDesc`, etc.) and the helper either
   > dereferences it or returns a reference / wrapper derived from
   > it, the helper *must* be `unsafe fn`, OR its input must be a
   > Rust reference whose lifetime carries the liveness contract
   > (`&pg_sys::Query`, `&IndexScanGuard`, etc.). Safe-`fn`-on-
   > raw-pointer in a Task 50 refactor will produce a lower block
   > count but a weaker safety contract, and the count win is
   > almost always achievable with the correct shape (see packets
   > 017 and 025 deltas).

   This codifies the lesson so the next agent encountering the
   same temptation has a reference.
