## Drift note: HNSW source.rs +2

`ec_hnsw/source.rs` went **up** by 2 (51 → 53). From the diff:

```rust
+    unsafe { with_flat_float4_source_from_datum(source_datum, source_attribute.kind, label, f) }
+    let source = unsafe { FlatFloat4VarlenaRef::from_datum(source_datum, label) };
```

These are *new* call-site `unsafe { }` acknowledgements around the
`FlatFloat4` source-vector decoding path. Reading the surrounding
context, the source helper now uses the safe HeapSlotReader to
extract the Datum but then needs an explicit boundary
acknowledgement when handing the Datum off to the varlena decoder.

decoding lives in P6 (Datum / Vector wrappers), not P5 (heap
slot). What's happening here is P5 is correctly draining its
contract while exposing the P6 boundary at the call sites that
need it. The +2 in `source.rs` is best read as "P5 is done in
this surface, P6 is now the next attention point."

Not a blocker. Worth noting in the next P6 packet's request.md so
the audit trail says "P6 inherited these two acknowledgements
from packet 033" — that way the ledger row provenance stays clean.
