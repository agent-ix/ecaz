## Finding: `debug_scan_opaque` family has unbounded-lifetime returns

The new helpers take raw pointers and return references with a
free `'a` lifetime in *safe* signatures:

```rust
#[cfg(any(test, feature = "pg_test"))]
fn debug_scan_opaque<'a>(scan: pg_sys::IndexScanDesc) -> &'a TqScanOpaque {
    // SAFETY: Debug callers inspect the HNSW opaque while the scan descriptor is
    // live and after begin/rescan initialized the opaque pointer.
    unsafe { &*(*scan).opaque.cast::<TqScanOpaque>() }
}

#[cfg(any(test, feature = "pg_test"))]
fn debug_scan_opaque_mut<'a>(scan: pg_sys::IndexScanDesc) -> &'a mut TqScanOpaque { ... }
```

`pg_sys::IndexScanDesc` is itself a raw pointer (`type
IndexScanDesc = *mut IndexScanDescData`). The function has no
input lifetime to constrain the output `'a`. The Rust nomicon's
"unbounded lifetimes" rule applies: a caller can request any
`'a` they want — including `'static` — and the compiler will
accept. A future test could write:

```rust
let stale: &'static TqScanOpaque = debug_scan_opaque(scan);
debug_am_end_scan(scan);  // opaque is freed
let _ = stale.foo;        // use-after-free
```

The SAFETY comment names the right invariant but it lives in the
*function body*, not in the signature, so callers don't acknowledge
it. The Rust convention is that any function which converts a raw
pointer into a reference must be `unsafe fn`.

Same anti-pattern in this packet:

- `debug_scan_opaque`, `debug_scan_opaque_mut` — primary concern
- `debug_scan_has_opaque`, `debug_scan_opaque_is_null`,
  `debug_scan_heap_tid` — read `(*scan).field` in a safe function

The `debug_am_*` callback wrappers (`debug_am_begin_scan`, etc.)
that wrap `ec_hnsw_am*scan` are also safe wrappers around `unsafe
fn`, but those don't produce references — they just call through
to the AM. Still mildly questionable (the `unsafe fn` AM callbacks
were marked unsafe for a reason), but the blast radius is much
smaller than a use-after-free.

## Suggested fix (apply uniformly with packet 015)

Two clean shapes:

**Option A — mark `unsafe fn`:**

```rust
#[cfg(any(test, feature = "pg_test"))]
unsafe fn debug_scan_opaque<'a>(scan: pg_sys::IndexScanDesc) -> &'a TqScanOpaque {
    // SAFETY: caller asserts scan is live and opaque is initialized.
    unsafe { &*(*scan).opaque.cast::<TqScanOpaque>() }
}
```

This preserves the reduction (the unsafe is centralized) and keeps
the contract visible at each call site (still wrapped in `unsafe { ... }`).
Block count goes back up at call sites — but only by one block per
call site, and the helper still wins on local invariant centralization.

**Option B — take a proven-live wrapper:**

If the call sites already hold an `IndexScanGuard` (which I see in
the packet — `let scan = IndexScanGuard::...`), thread the guard
through the helper so the lifetime is bounded:

```rust
fn debug_scan_opaque<'a>(scan: &'a IndexScanGuard) -> &'a TqScanOpaque {
    // SAFETY: the scan guard owns the descriptor for 'a and the
    // begin/rescan callbacks have run.
    unsafe { &*(*scan.as_ptr()).opaque.cast::<TqScanOpaque>() }
}
```

The output `'a` is bounded by the input borrow. Sound and safe.

This packet is approved as-is because it is test-only. Apply the
fix in a follow-on alongside the packet 015 correction.
