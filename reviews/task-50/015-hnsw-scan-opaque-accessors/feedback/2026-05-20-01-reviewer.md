# Conditional approve — soundness regression in `scan_opaque_ref` / `scan_box_ref` (production hot path)

intent; **requesting a follow-on packet that marks the new helpers

## Finding: four safe helpers with unbounded-lifetime returns from raw pointers

The new helpers are safe functions that accept raw pointers and
return references with a free lifetime parameter:

```rust
fn scan_opaque_ref<'a>(opaque: *mut TqScanOpaque) -> &'a TqScanOpaque {
    assert!(!opaque.is_null(), "ec_hnsw scan opaque pointer must be live");
    unsafe { &*opaque }
}

fn scan_opaque_mut<'a>(opaque: *mut TqScanOpaque) -> &'a mut TqScanOpaque {
    assert!(!opaque.is_null(), ...);
    unsafe { &mut *opaque }
}

fn scan_box_ref<'a, T>(ptr: *const T) -> Option<&'a T> {
    if ptr.is_null() { None } else { Some(unsafe { &*ptr }) }
}

fn scan_box_mut<'a, T>(ptr: *mut T) -> Option<&'a mut T> {
    if ptr.is_null() { None } else { Some(unsafe { &mut *ptr }) }
}
```

`'a` is unbounded: a caller can choose any lifetime including
`'static`. The Rust nomicon's "unbounded lifetimes" rule applies
directly. The `assert!(!opaque.is_null())` only catches null; it
does not catch use-after-free, the wrong scan's opaque pointer, or
a stale Box-raw allocation. The SAFETY comment names the right
invariant, but it lives in the function body, where the caller
never sees it.

This is a **soundness regression** vs. the previous shape:

```rust
fn bootstrap_expansion_frontier_len(opaque: &TqScanOpaque) -> usize {
    if opaque.bootstrap_expansion.is_null() {
        0
    } else {
        // SAFETY: non-null `bootstrap_expansion` is Box-owned by this scan
        // opaque until `free_bootstrap_expansion` runs at scan teardown.
        unsafe { &*opaque.bootstrap_expansion }.frontier_len()
    }
}
```

Before, the input `&TqScanOpaque` *already constrained* the
lifetime of the projected reference — the borrow checker knew the
result couldn't outlive the opaque. The SAFETY comment named the
Box-ownership invariant in context.

After, `scan_box_ref(opaque.bootstrap_expansion)` strips that
binding: the helper has no idea the pointer came from
`&TqScanOpaque`, and a future caller can pass any raw pointer.

The block count went down by replacing the inline `unsafe { ... }`
with a function call — but the safety contract is now weaker than
before.

## Fix

Two acceptable shapes. Pick whichever produces cleaner call sites.

**Option A — mark the helpers `unsafe fn`:**

```rust
unsafe fn scan_opaque_ref<'a>(opaque: *mut TqScanOpaque) -> &'a TqScanOpaque {
    // SAFETY: caller asserts `opaque` is live and exclusively borrowed for 'a.
    unsafe { &*opaque }
}
```

Call sites wrap each invocation in a one-line `unsafe { ... }`.
Block count rises by one per call site relative to the current
packet — but the helper still wins by centralizing the dereference
shape, and the SAFETY contract is acknowledged at every call site.

**Option B — bind the lifetime to a `&TqScanOpaque` input (preferred):**

The original helpers (`bootstrap_expansion_frontier_len`,
`visited_tid_count`, `emitted_result_tid_count`, `cached_quantizer_ref`)
already take `&TqScanOpaque`. Keep that shape and project the
field inside, instead of accepting a bare `*const T`:

```rust
fn bootstrap_expansion<'a>(opaque: &'a TqScanOpaque) -> Option<&'a search::BeamSearch> {
    if opaque.bootstrap_expansion.is_null() {
        None
    } else {
        // SAFETY: the opaque borrow proves the scan is live; the field
        // pointer is Box-owned by the opaque until scan teardown.
        Some(unsafe { &*opaque.bootstrap_expansion })
    }
}

fn visited_tids<'a>(opaque: &'a TqScanOpaque) -> Option<&'a HashSet<...>> { ... }
```

This sounds the API: output `'a` is bounded by the input borrow's
lifetime, so a stale projection can never outlive the opaque.

For `scan_opaque_ref(*mut TqScanOpaque) -> &'a TqScanOpaque` —
that one is harder, since it converts the raw opaque pointer from
the scan descriptor into a reference. Option A (`unsafe fn`) is
the simplest answer; the call sites already had explicit unsafe
blocks before this packet, so adding the unsafe back is a wash.

## What is correct in the packet

The structural intent is right — centralizing the
`&*opaque.field` projection so the SAFETY comment lives in one
place is the §Techniques #1 / #2 pattern from the task doc. The
implementation just needs to keep the contract visible. The fix
above preserves the centralization.

`assert!(!opaque.is_null(), ...)` is a useful belt-and-braces check
to keep regardless of which fix is applied.
