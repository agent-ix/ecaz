## Finding: `unsafe fn` removed from raw-pointer-input helpers

Diff against the commit:

```rust
-unsafe fn coordinator_ptr(state: *mut EcParallelScanState) -> *mut EcParallelCoordinatorState {
+fn coordinator_ptr(state: *mut EcParallelScanState) -> *mut EcParallelCoordinatorState {

-unsafe fn worker_slots_ptr(state: *mut EcParallelScanState) -> *mut EcParallelWorkerSlot {
+fn worker_slots_ptr(state: *mut EcParallelScanState) -> *mut EcParallelWorkerSlot {

-unsafe fn reset_parallel_scan_layout(state: *mut EcParallelScanState) {
+fn reset_parallel_scan_layout(state: *mut EcParallelScanState) {

-unsafe fn initialize_parallel_scan_state(
+fn initialize_parallel_scan_state(
     state: *mut EcParallelScanState, worker_slot_count: u32,
 ) { ... }

-unsafe fn validate_parallel_scan_state(...)
+fn validate_parallel_scan_state(...)

-unsafe fn parallel_scan_state_ptr(parallel_scan: *mut pg_sys::ParallelIndexScanDesc) -> ...
+fn parallel_scan_state_ptr(parallel_scan: *mut pg_sys::ParallelIndexScanDesc) -> ...

-    pub(crate) unsafe fn worker_slot(...)
+    pub(crate) fn worker_slot(...)
```

These functions accept raw pointers
(`*mut EcParallelScanState`, `*mut pg_sys::ParallelIndexScanDesc`)
and either dereference them or return raw-pointer projections from
them. Marking them safe `fn` means any code in this module can call
them with a stale or wild raw pointer; the contract is no longer
acknowledged at the call site.

Note one interesting wrinkle in the diff: many *callers* still wrap
these helpers in `unsafe { ... }` blocks:

```rust
-    let attachment = unsafe { validate_parallel_scan_state(state) }?;
-    let slot = unsafe { attachment.worker_slot(slot_index) }?;
```

The `unsafe { ... }` blocks on the calls of a safe function are now
spurious — they will trigger `clippy::unnecessary_unsafe` (or
similar). Either (a) the clippy noise was the reason the helper
was made safe (wrong fix — the *helper* was correct as unsafe), or
(b) the callers are inconsistent and a future cleanup will remove
the `unsafe { }` wrappers, leaving the unsoundness fully exposed.

## Fix

Apply the same shape that packet 017 used. Choose whichever pattern
fits each helper best:

**For helpers that project raw pointer → raw pointer:**

Keep `unsafe fn`. Block count cost is minimal (callers already
wrap them in `unsafe { ... }` per the surviving call sites in the
diff). This restores the contract without rewriting the API.

**For helpers that produce a reference / Option / Result:**

Either mark `unsafe fn`, or take a *Rust reference* as input so
the borrow checker carries the lifetime. For example,
`validate_parallel_scan_state` could take a
`&EcParallelScanState` produced once by an `unsafe fn` constructor
at the PG callback boundary.

**Block-count cost of the fix:** roughly equal to the +1/+6/0/+1
deltas observed in packet 017. The file will still meet the 30%
target (start 63, current 44; the fix may bump count back to
~48–50, still under the ≤44 ceiling if we accept the slightly
higher post-fix count, or comfortably under ≤47 if planning
accepts an 30% threshold rounded up from 44.1).

If the post-fix count edges over 44, document the structural
ceiling per the Task 50 exit criterion language ("explains why a
lower reduction is the structural ceiling").
