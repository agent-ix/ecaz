## What's correct

- `palloc_copy_slice<T: Copy>(values: &[T], context: &str) -> *mut T`
  takes a *Rust slice reference* as input — the input liveness is
  type-checked. Inside, the `unsafe { pg_sys::palloc(bytes) }` is a
  bare allocator call with no soundness preconditions on the
  caller. This wrapper is sound.
- `store_*`, `free_*` family is lifted from `unsafe fn` to safe `fn`
  with `&mut EcIvfScanOpaque` input. The opaque reference proves
  liveness, the field projection (`opaque.query_values`) is bounded
  by the opaque borrow, and null-check + state-machine handles the
  freed slot. Sound.

## Finding 1: `pfree_scan_slice` is safe but takes a raw pointer

```rust
fn pfree_scan_slice<T>(ptr: *mut T) {
    // SAFETY: callers pass a pointer allocated by `palloc_copy_slice`
    // and owned by the current scan opaque.
    unsafe { pg_sys::pfree(ptr.cast()) };
}
```

The function is safe but it calls `pfree` on an arbitrary raw
pointer. The "callers pass a pointer allocated by `palloc_copy_slice`"
invariant is comment-only — there is no compile-time check. A
future caller could pass any `*mut T` and the function would happily
attempt the free.

In practice every current caller does
`pfree_scan_slice(opaque.field)` from inside a function that has
`&mut EcIvfScanOpaque`, so today this is fine. But the helper
itself does not carry the contract.

**Fix:** either `unsafe fn pfree_scan_slice(ptr: *mut T)`, or
restructure so the helper takes the opaque + a field selector
(e.g. close over the field write). Easiest is the `unsafe fn`
form — callers wrap each invocation in a one-line `unsafe { ... }`
and the contract is visible.

## Finding 2: `debug_scan_opaque` family has unbounded-lifetime returns

Same anti-pattern as packet 012:

```rust
#[cfg(any(test, feature = "pg_test"))]
fn debug_scan_opaque<'a>(scan: pg_sys::IndexScanDesc) -> &'a EcIvfScanOpaque {
    unsafe { &*(*scan).opaque.cast::<EcIvfScanOpaque>() }
}

#[cfg(any(test, feature = "pg_test"))]
fn debug_scan_opaque_option<'a>(scan: pg_sys::IndexScanDesc) -> Option<&'a EcIvfScanOpaque> { ... }
```

Same fix as packet 012: either mark the helpers `unsafe fn`, or
take an `IndexScanGuard` borrow so the output lifetime is bounded
by the input. See `012-hnsw-scan-debug-facade/feedback/2026-05-20-01-reviewer.md`
for the worked example.

This packet is approved as-is because the storage wrappers are
sound and the debug helpers are test-only. Consolidate the
unbounded-lifetime + `pfree_scan_slice` fix into the same follow-on
that addresses packets 012 and 015.
