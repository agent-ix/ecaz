## Findings

### 1. (substantive design choice) Top-k bounded Kendall-tau
treatment

The packet's choice to assign rank `k+1` to missing entries
and ignore ties between two missing entries is **the right
design** for ANN cross-comparison. Worth flagging because
it's a non-obvious detail:

**Alternative considered:** classical Kendall-tau on the
union of both sets would penalize "AM A puts X first" vs.
"AM B doesn't return X at all" as a maximally-bad pair
inversion. That's misleading — both AMs aren't claiming X
isn't relevant, only that they didn't return it within `k`.

**Implemented:** missing entries all get rank `k+1`,
indistinguishable from each other. Pairs between two missing
entries don't contribute to the tau computation (ignored
ties).

**Result:** rank drift among **commonly-returned** results
is still visible. Membership drift is reported via Jaccard.
The two metrics decompose cleanly:
- Jaccard says "do they agree on which set"
- Kendall says "of the ones they agree on, do they agree on
  order"

Worth a one-line doc comment in the implementation naming
this design choice, since it differs from textbook Kendall.

### 2. Predictions JSON format

`ecaz bench recall --predictions-output` writes per-query
top-k JSON. Worth confirming the format is:
- Stable (versioned schema, ideally)
- Forward-compatible (new fields don't break old consumers)
- Doesn't leak transient state (build epoch, run timestamp
  that would make diffs spurious)

If the JSON includes a `format_version` field, a future
consumer can refuse to read incompatible exports. If it
doesn't, worth adding before the format is widely consumed.
