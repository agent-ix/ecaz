## Findings

### 1. (substantive) Floors are conservative from a single measurement

The packet's recall numbers come from one run of the 512×64
synthetic CI fixture:
- HNSW: observed 0.8500, floor 0.84 (0.01pp margin)
- IVF: observed 0.8500, floor 0.84 (0.01pp margin)
- DiskANN: observed 0.5660, floor 0.55 (0.016 margin)

These are **very tight margins**. With n=64 queries, the
Wilson 95% CI half-width on recall=0.85 is ~0.087 — so a
single 0.85 observation is consistent with true recall
anywhere in [0.76, 0.94]. The floor at 0.84 isn't
statistically robust against natural per-run variance.

Two consequences worth flagging:
- **CI false positives possible.** A natural per-run variance
  drawing 0.83 instead of 0.85 would fail the gate without
  any code regression. Burn-in will reveal whether this
  fires.
- **CI false negatives possible.** A real regression that
  drops mean recall from 0.85 to 0.82 might still draw a
  single run at 0.84 by chance and pass.

Recommendation: see item 8 in the 003 plan
(`reviews/task-47/003-exhaustive-gate-coverage-plan/`). Add
confidence intervals to the gate output and consider
conservative comparison (`observed.low ≥ floor`) or multi-seed
runs to tighten the bound.

### 2. (substantive) Synthetic CI fixture is uniform-distribution

The generated 512×64×32 fixture is described as "generated
small PG18 fixtures." If it's uniform random vectors (the
default for synthetic ANN fixtures), it's the **easiest case**
for ANN heuristics — clustered or adversarial data exposes
many more bugs.

The 0.85 / 0.85 / 0.566 floors look reasonable for uniform
data, but a regression that breaks heuristic correctness on
clustered data could pass this gate while shipping a real bug.
The task doc explicitly names mixed-distribution corpora as
required scope; item 3/14 of the 003 plan tracks this.

### 3. (verification) Cost-gate scope is appropriately narrow

Only 2 cost rows (ivf-small, hnsw-small) — no DiskANN row,
no real-corpus row. The packet doesn't claim broader coverage;
003 plan items 10-12 track expansion. This is correctly
scoped as a starting point.

### 4. (low-severity) `recall_integration.rs` should be deleted
not "marked redundant"

The packet's summary says "marked as redundant for AM gating
and retained only as an ignored pure-Rust quantizer benchmark."
If it's redundant for the user-facing recall guarantee, retaining
it as a benchmark means:
- It still appears in `cargo test --list`
- A future contributor might wire it back into CI not knowing
  it's been superseded
- It carries maintenance cost

Cleaner options:
- Delete entirely (preferred — the gate is the source of truth
  now)
- Relocate to `benches/` if the quantizer-benchmark value is
  genuine
- Add a `#[ignore = "Task 47 gate is the source of truth"]`
  attribute with a doc-comment pointing at the gate

Tracked as item 15 in the 003 plan.

### 5. (verification) `scripts/check_cost_baseline.py` design

Spot-checked the script's logic — it reads the JSONL,
normalizes path identifiers, compares per-row against
baseline, and either passes/fails or rewrites under
`--accept-drift`. Right shape for the burn-in workflow.

One observation: the script enforces drift bands per-field
on **aggregate cost numbers** (modeled_*, index_pages,
reltuples). Per-node cost decomposition isn't checked — a
plan whose total cost stays the same but whose internal
node costs swap (leaf-scan halved, sort doubled) would pass.
Item 12 in 003 plan tracks this expansion.

Per `docs/recall-floors.md`:

> rerun with packet-local artifact paths before it is

reviews/task-47/002-live-small-gate/artifacts/make-*.log`
