## Findings

### 1. Coverage scope is honest about gaps

The "Remaining Task 42 Gaps" section names every concrete follow-up:
golden fixtures (002-011), byte-swapped rejection tests (014),
IVF/SPIRE static offsets (002-003), qemu (014-015), upgrade matrix
(012), WAL version tags (017), pg_upgrade smoke (016).

This is the right way to start a multi-packet effort — name the
endpoints, then close them one by one. The 18-packet sequence that
actually ran maps cleanly onto this list.

### 2. (verification) The constants encode wire bytes, not host structs

The review focus asks for confirmation that "layout assertions
cover encoded byte contracts rather than host Rust struct layouts
where the persisted form is a `Vec<u8>` or borrowed slice view."

Spot-checking the diff: the new constants are byte offsets into
the serialized payload (e.g., `pub const PAYLOAD_FLAG_OFFSET:
usize = 0` for the metadata flag at byte 0), with the assertion
testing the offset against the constant rather than against
`offset_of!(Struct, field)`. That's the right discrimination —
since `Vec<u8>` and slice views have no fixed Rust layout, asserting
against a named byte offset is what actually pins the wire format.

Confirmed.

### 3. (carry-forward to follow-up packets) Pattern is reusable

The pattern this packet establishes — typed constants in the codec
module, assertions in `tests/size_of_assertions.rs` — is exactly
what 002 (IVF/DiskANN extensions) and 003 (SPIRE) should follow.
The closeout audit (018) confirms they did.

For any future on-disk format addition outside Task 42's current
scope (e.g., codebook persistence, OPQ rotation matrices), the
acceptance criterion is "add the constants + extend the assertion
file." That should be a project rule, not just a Task 42
expectation.

### 4. (cross-reference) Compile-time vs runtime testing split

The compile-time check (`size_of_assertions.rs`) catches *layout
drift*. The runtime fixture check (introduced in 004-011 per the
closeout) catches *encoding drift* — i.e., the bytes that actually
hit disk match the bytes that hit disk a year ago. Both are
needed; this packet correctly establishes only the layout side.
The fixture packets layer the runtime side on top.
