# Findings

## 1. The design doc is stale on rerank-source ownership, and coder-2 should not implement hot+cold page persistence from it as written

The packet still says Phase 2 persists "hot + cold" pages and treats
the cold rerank payload chain as part of V0 `ecdiskann`. That no longer
matches the later branch decisions:

- ADR-045 says `rerank_tid` is reserved and stays `INVALID` until
  ADR-044 C1 reopens.
- Phase 6A / 6B scan docs assume exact rerank comes from the heap
  `ecvector` row, not from an index-owned cold payload.

If coder-2 implements from 11004 literally, they will overbuild the V0
storage format and drift from the later ADR thread. I would update the
doc so V0 build persists only the hot graph pages plus metadata, with an
explicit note that index-side cold rerank payload is deferred to a
future ADR-044 C1 reopen.

## 2. The medoid sample cap is inconsistent across the thread

11004 still specifies `S = min(10_000, N)` for medoid approximation,
while Phase 5C-2 later hard-codes `MEDOID_SAMPLE_CAP = 1000` and
describes that as "mirrors pgvectorscale".

That mismatch matters because the build design packet is supposed to be
the authoritative Phase-2 design surface. Either:

- update 11004 to match the later `1000` decision, or
- justify why the implementation should move back to `10_000`

Leaving both values live invites silent drift in future measurement or
benchmark expectations.
