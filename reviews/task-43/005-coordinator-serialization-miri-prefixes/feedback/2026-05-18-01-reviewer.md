## Findings

### Critical: Many-seeds is structurally empty

Coder discloses this honestly in the audit: "No current `miri_` test
was identified that creates real threads or atomics inside Miri." The
lane runs 128 seeds × 35 non-threaded tests — that is 128× the same
deterministic execution, not 128 interleavings of a concurrent
program. The exit criterion "Many-seeds run included where any
threaded `miri_` test exists" is satisfied **vacuously** (no such test
exists, so the criterion is empty). The spirit of the task is not.

For a packet whose stated north star is "extensive safety testing,"
having a zero-coverage concurrency lane is the single largest gap.

**Suggested follow-up scope (not required for this closeout to
land):**

- Add one pure threaded test exercising a real synchronization
  primitive, e.g. an `AtomicUsize` counter or a `OnceLock<Mutex<_>>`
  cache around the quantizer codebook init, so many-seeds has
  something to interleave.
- Lift a small pure subset of `src/am/common/parallel.rs` and run it
  under many-seeds with a memory-model probe.

### Coverage breadth gaps relative to strategy packet (001)

These are surfaces named in the strategy packet that did not receive
`miri_` prefixes across packets 003/004/005:

- HNSW: `select_next_with_refill`, deterministic frontier ordering
  invariant, stale-candidate removal.
- DiskANN graph: `build_vamana_graph_with_stats`,
  `build_vamana_graph_with_pass1_extra_candidates`.
- DiskANN vacuum: `mark_deleted`, `strip_dead_primary_heaptid`,
  `is_fully_dead`.
- SPIRE top-k: `scored_candidate_cmp`,
  `rerank_scored_candidates_by_ip`.
- SPIRE vacuum: zero `miri_` coverage (no
  `collect_visible_assignments`, no delete-delta grouping).
- Remote parser: `typed_payload_hex_decoded_bytes` and Row-independent
  typed-payload parser extraction explicitly named in the strategy
  was not done. Only payload-cap helpers landed.

Each is a small promotion. Pattern is two `miri_` tests per major
subsystem; strategy suggested wider coverage per subsystem.

### No mutation / bug-injection verification

The task spec's validation bullet says each newly added `miri_` test
should be shown to "fail predictably if an obvious bug is injected
(e.g., unsynchronized shared atomic without `Ordering::SeqCst`)."
None of the four closeout-series packets includes such a sanity
check. Without it, "Miri pass = no UB" is a weak signal — the test
might pass for the wrong reason (e.g. running before the suspect
code path).

### Tree Borrows agreement is unverified-by-disagreement

SB and TB both report 35 passed. That's the easy outcome. The
strategy packet asked for triage on disagreement; no
forcing-function for the triage path was demonstrated. If SB and TB
ever diverge on a future test, the runbook is in `docs/hardening.md`
but has not been exercised.

## Verdict

**Approve closeout with reservations on safety-testing breadth.**

The packet delivers what the exit criteria literally ask for and the
completion audit grounds every claim in artifact evidence. Coder is
upfront about the many-seeds-is-vacuous gap.

The reservations are not about correctness of what landed — they are
about whether the bar called for in the spec ("extensive safety
testing") is genuinely met when:

1. The concurrency depth lane has no real concurrent test to depth-check.
2. Roughly half the surfaces named in the strategy packet did not
   receive `miri_` prefixes.
3. No mutation-test sanity check exercises the "would Miri catch a
   regression?" question.

If "extensive safety testing" is non-negotiable, recommend a Task 43b
that closes the listed gaps before declaring the lane complete. If
the bar is "first systematic Miri/careful depth pass," this packet
meets it cleanly.

## Recommended follow-up tasks

1. One real threaded `miri_` test (smallest viable: atomic counter
   with two threads + many-seeds). Validates the many-seeds lane.
2. Promote the named-but-skipped surfaces from the strategy packet
   (`select_next_with_refill`, `scored_candidate_cmp`, etc.) — each is
   a one-line rename of an existing test.
3. One mutation-injection test per major subsystem demonstrating that
   the `miri_` lane catches a deliberately-broken variant.
4. Row-independent typed-payload parser extraction so the remote
   parser hot path lands under Miri.
