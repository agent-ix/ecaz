# Review Feedback: Graph-First Exhaustion Test Contract

**Verdict: Approved**

## Test contract update

The exhaustion test now checks:
1. At least one result before exhaustion
2. Nearest neighbor is included
3. No duplicates
4. All emitted tids are valid table entries
5. Repeated `amgettuple` after exhaustion returns false

This is the right A3 exhaustion contract. The old test required every tid to be returned, which
assumed a full linear tail. Dropping that assumption is correct — graph traversal can exhaust
before visiting every indexed element.

## Residual coverage

The `!exhausted_once` assertion is preserved, confirming that the first pass does produce at
least one result before the scan reports exhaustion. This prevents a regression where graph
traversal immediately exhausts without trying.
