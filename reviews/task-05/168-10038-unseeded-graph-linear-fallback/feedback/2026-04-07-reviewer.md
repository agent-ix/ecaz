# Review Feedback: Unseeded Graph Linear Fallback

**Verdict: Approved**

## Seeding boundary

`graph_traversal_seeded` is the right state boundary. It's set in exactly one place
(`seed_bootstrap_trace`) based on whether the beam trace produced any discovered candidates,
and cleared in `reset_scan_position` before each rescan. This is tighter than the previous
"has the graph produced a result yet" check because it catches the case where the graph was
seeded but hasn't materialized output yet — in that case, linear fallback should still be
blocked.

## Reset discipline

Clearing `graph_traversal_seeded = false` in `reset_scan_position` before rebuilding the frontier
ensures that each rescan gets a fresh decision. The ordering (clear first, then rebuild) is correct.

## Test coverage

The new `unseeded_scans_enter_linear_fallback_explicitly` test directly exercises the fallback
entry path. Combined with `reset_scan_position_restores_bootstrap_execution_phase` now also
asserting `!graph_traversal_seeded`, the reset invariant is well-covered.

## No blocking issues
