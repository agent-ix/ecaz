# Review: Seed Bootstrap Frontier Directly From Layer-0 Beam Trace (Review 163)

**Reviewer:** Claude Opus 4.6
**Date:** 2026-04-08
**Scope:** Review 163 — widened entry beam trace for bootstrap frontier
**Commit:** `5001844`

## Summary

Makes bootstrap entry seeding use the graph-owned beam runner with the full bootstrap width instead of a single expansion. Seeds the visible bootstrap frontier directly from the beam trace, keeping only the seeded entry candidate marked as expanded so later discovered candidates remain eligible for refill-on-consume.

**Verdict: Approved.**

## Assessment

This is the natural progression from review 161. Instead of seeding only the entry point and relying on incremental refill to build the frontier, the entry seeding now does a full-width beam traversal up front. This produces a better initial frontier because beam search finds candidates in score order globally, not just via local greedy expansion.

Preserving only the entry candidate in expanded-source state (rather than marking all trace-expanded nodes) is the right compatibility contract. It means discovered candidates from the trace are still available for refill-on-consume expansion — the refill logic will re-expand them and discover their successors. This avoids cutting off the traversal too early.
