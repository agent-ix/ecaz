### 2. Removing a dead branch instead of testing it: correct call

Three of the 18 missed mutants (`cost.rs:255:53` operator swaps on
`inputs.index_pages > 0.0`) were retired by deleting the branch
rather than by adding tests. The triage rationale — that the
earlier `<= 0.0` gate made the comparison redundant — is right.
This is the **mutation testing → dead code discovery** pattern in
action: cargo-mutants flags unreachable code because *no* test can
distinguish its operator swaps. The correct response is to delete
the dead code, not add a test that can't actually fire.

Worth pointing at this packet from `docs/hardening.md`'s mutation
triage policy section (line 298, which already talks about
`equivalent` verdicts) — except per the rule above, we don't add
new per-packet pointers there. The triage policy already covers
the pattern; the precedent is in this packet's `triage.md`.

### 3. cfg-gating pattern is the template for items 14-16

The cfg-gate (`#[cfg(any(feature = "pg17", feature = "pg18"))]` on
the callback glue) is exactly the pattern that lets a callback-mixed
AM file ship in the careful coverage lane. Pure Rust runs everywhere;
callbacks only build under pgrx feature flags.

This is **a partial workaround for the pgrx feasibility blocker
from packet 013**. The pure logic side of each callback-heavy file
can now ride this pattern into the careful lane, raising coverage
*without* needing live pgrx instrumentation. It doesn't fix items
14-16 directly (those still require live callback measurement), but
it raises the *non-callback fraction* of those files in the careful
lane.

Concrete follow-up: a separate scoping pass should identify which
of the currently-0.00% callback-mixed files have enough pure logic
that they can ride 015's pattern. Candidates from the snapshot:
- `src/am/ec_diskann/{routine,scan,build}.rs` — `scan.rs` likely
  has pure ranking logic; `build.rs` likely has pure graph
  construction. Each could be a packet.
- SPIRE coordinator paths — pure cross-AM dispatch is testable
  without live callbacks.

Not blocking for this packet. Note for future scoping.
