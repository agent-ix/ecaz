## Per-item detail

### Items 1-5: pure-Rust quant coverage (no pgrx)

These modules have no `pg_sys::*` in their core logic — unit
tests work directly.

**Test categories per module:**
- **Boundary inputs**: zero vector, unit vector, NaN, ±inf,
  denormal floats, all-bits-zero, all-bits-one
- **Round-trip invariants**: `decode(encode(x))` within
  documented quantization tolerance for `x` ∈ random batches
  at d ∈ {64, 128, 256}
- **Algebraic invariants**: e.g., `score(a, b) == score(b, a)`
  for symmetric distances; `score(a, a) == identity_value`
- **Known-answer tests**: pin specific input/output pairs from
  adjacent published implementations or hand-computed values
- **Property tests via `proptest`**: invariants above lifted to
  generators

**Test file pattern:**
```rust
// src/quant/prod.rs
#[cfg(test)]
mod tests {
    use super::*;
    use proptest::prelude::*;

    #[test] fn encode_zero_vector() { /* explicit */ }
    #[test] fn decode_inverse() { /* round-trip */ }
    #[test] fn score_symmetric() { /* algebraic */ }
    #[test] fn known_answer_d128() { /* pinned KAT */ }

    proptest! {
        #[test]
        fn round_trip_random(v in prop::collection::vec(any::<f32>(), 128)) {
            let encoded = encode(&v);
            let decoded = decode(&encoded);
            for (a, b) in v.iter().zip(decoded.iter()) {
                prop_assert!((a - b).abs() < QUANTIZATION_TOLERANCE);
            }
        }
    }
}
```

**Acceptance per file:**
- Line coverage ≥80% (≥70% for items 5 modules)
- Baseline TSV raised from 0.00 → actual value, with packet citation in note column
- `make mutants MUTANTS_MODULE=src/quant/prod.rs` produces survivors;
  every survivor triaged per item #19 rules

**Done across items 1-5 when:** every quant module has the
shape above, mutation runs cleanly, baselines raised.

### Items 6-10: page codec coverage via `hardening/careful`

These modules use `pg_sys::*` types but the `hardening/careful`
lane already imports `src/storage/page.rs` via
`#[path = "../../../src/storage/page.rs"]` mechanism. Same
trick can extend to per-AM page codecs.

**Per-AM file extension pattern:**
```rust
// hardening/careful/src/am_pages.rs
#[path = "../../../src/am/ec_hnsw/page.rs"]
mod ec_hnsw_page;

#[cfg(test)]
mod hnsw_page_tests {
    use super::ec_hnsw_page::*;
    // round-trip tests, share fixtures with tests/on_disk_fixtures.rs (Task 42)
}
```

**Test categories:**
- **Round-trip per tuple kind**: encode bytes, decode bytes,
  assert equality including header fields
- **Truncated buffer rejection**: feed N-1 bytes where N is the
  minimum tuple size; assert decoder returns error
- **Oversized header rejection**: bump tuple length field past
  page size; assert decoder errors
- **Format-version mismatch**: bump version byte; assert
  decoder rejects (Task 42 has this fixture)
- **Endian swap rejection**: byte-swap multi-byte fields;
  assert decoder rejects (Task 42's fixture pattern)

**Acceptance per file:**
- Line coverage ≥80% (≥70% for items 11 storage helpers)
- Reuses Task 42 golden hex fixtures where overlap exists —
  no fixture duplication

**Done across items 6-10 when:** every page codec exercised
by at least the 5 test categories above, baselines raised.

### Item 12: RAII guards (Task 41)

`src/storage/{relation, heap, snapshot, slot, scan, buffer,
lock, spi, wal, page}_guard.rs` and similar — the new RAII
surface from Task 41.

**Test categories:**
- **Constructor non-null acceptance**: each `open(...)` /
  `acquire_*(...)` returns a guard when given a valid PG handle
- **Drop calls the right release function**: instrument with a
  mock PG-sys layer in pure-Rust tests, assert Drop invokes
  the expected `*_close` / `*_release` / `*_unregister`
- **Panic-during-Drop semantics**: cfg-test harness that
  triggers a panic inside a guard's scope and verifies Drop
  still runs (proves the pgrx-unwind contract holds)
- **Borrow-lifetime enforcement**: `PinnedBufferLockGuard<'a>`
  trybuild test — a deliberately-buggy caller that tries to
  outlive the pin must fail to compile

**Acceptance:**
- Line coverage ≥80% per guard module
- Mutation survival rate <10% on Drop impls (most mutants
  should be killed by the "Drop calls right function" test)
- `trybuild` negative tests for any guard with a lifetime
  parameter

**Done when:** all 10 guards have the shape above. Adds to the
baseline TSV at non-zero values.

### Item 17: Coverage ratchet mechanism

Without a ratchet, baselines stay at whatever they were when
landed. New tests improve coverage temporarily; the next PR
that doesn't run those tests sees the same baseline and
doesn't ratchet up.

**Two options to choose between:**

**Option A — Manual baseline-update commits:**
- Coverage improves → reviewer runs `make coverage`, commits
  TSV update with note `"raised by packet NNN"`
- Pros: explicit, reviewable, no auto-trust
- Cons: requires discipline; baselines stale if reviewer forgets

**Option B — Auto-ratchet under explicit flag:**
- `scripts/check_coverage_delta.sh --ratchet` rewrites TSV
  when `actual > baseline + 2pp`
- CI runs `--ratchet` only on commits to `main`, not PRs
- Pros: zero-friction
- Cons: easier to accidentally relax if `--ratchet` is run on
  partial test data

**Recommendation:** Option B with CI-only guard. Manual override
via Option A still possible for explicit deratching.

**Acceptance:** `docs/hardening.md` names the chosen pattern;
`scripts/check_coverage_delta.sh` supports the chosen flag.
