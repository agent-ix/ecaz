## Confirmations

### The boundary-assertion fix is the right diagnosis

The triage of 026 had named "assert `bound.is_finite()` on the
equal-to-floor case" — this packet implements exactly that. The
former tests asserted `estimate.is_finite()`, which the `< → <=`
mutant satisfied trivially because the mutated return path was
`estimate = 0.0` (finite). Switching the assertion to
`bound.is_finite()`, where the mutant returns `f32::INFINITY`,
discriminates the case correctly.

This is a useful diagnostic pattern worth flagging:
**`is_finite()` is a weak boundary assertion** because both `0.0`
and the intended valid value pass it. For floor/ceiling boundary
tests, assert against the intended **failure mode** (infinity,
NaN, specific value) rather than the survival predicate.

### The `RABITQ_QUANT_RANGE` const refactor — same pattern as 016/017

`2.0 * c` with `c == 2.0` produced an equivalent mutant under
`* → +` because both evaluate to 4.0. Pulling the magnitude out
as `RABITQ_QUANT_RANGE = 4.0` literal removes the binop from
that expression, retiring the equivalent mutant structurally.

Same pattern as packets 016 (`1 << 0` → `0b...`) and 017
(derived constants → literals). Now three packets confirming
"on-disk / wire / quantization constants should be literal,
not derived." Worth a project-level note in any future style
doc.

### Timeout retirement via boundary helpers (017's pattern)

The 2 timeouts at former lines 1127/1132 were retired by the
same boundary-assertion strengthening that killed the missed
`< → <=` mutants. No global mutation-timeout bump needed. Same
shape as 017's timeout fix.

runs 447 mutants in 8m with `-j 4` scratch-copy. The faster
