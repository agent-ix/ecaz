# Review: Pure-Rust mutation lane target fix (006)

## Summary

Operational follow-up to packet 005. Fixes
`scripts/hardening.sh mutants --file src/quant/simd.rs` so it
correctly routes to the `ecaz-careful-hardening` package
instead of falling back to the pgrx test binary (which aborts
at load on this host).

The 005 packet ran the mutation command directly with the
correct `--package` flag. This packet wires that fix into the
generic `make mutants` lane so users don't have to remember to
override.

Mechanical fix; clean.

## Confirmations

- **`run_mutants_lane` now maps quant/page paths correctly.**
  For `src/quant/*` and `src/storage/page.rs`, the script
  rewrites to `hardening/careful/src/../../../...` and passes
  `--package ecaz-careful-hardening`. Right targeting.
- **Make lane preserves the same UI.** Per

  invokes `scripts/hardening.sh mutants --file

- **cargo-mutants sees 9 mutants via the rewrite.** Per
  `careful-simd-mutants-list.log`. Confirms the targeting
  works end-to-end (cargo-mutants discovers the same mutant
  set 005 ran).

## Findings

### 1. (suggestion) Generalize the path mapping

Current rewrite covers `src/quant/*` and `src/storage/page.rs`.
As Task 39's careful harness extends to more modules (per items
6-11 in my 003 plan: AM page codecs, SPIRE storage helpers,
storage/*_guard.rs), the rewrite list grows.

Two options:
1. Hard-code each new path as it joins the careful harness
2. Detect dynamically: if `<careful-path>/src/../../../<file>`
   exists (or equivalently, the file is path-imported by the
   careful crate's `lib.rs`), rewrite

Option 2 is more maintainable but requires walking the
careful crate's source for `#[path]` attributes. Not a
blocker for this packet; flag for when the path list expands
past ~5 entries.

### 2. (verification) PR-CI compatibility

The 005 packet ran with `--in-place` due to local untracked
artifacts. This packet's path-mapping change works for both
scratch-copy and in-place modes (it just changes which
package cargo-mutants targets). So CI running scratch-copy
mode will get the correct package targeting from this packet.
