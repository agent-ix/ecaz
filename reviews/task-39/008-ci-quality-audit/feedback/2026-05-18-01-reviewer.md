## Findings

### 1. (suggestion) Wire `test-quality-ci-audit` into `ci-quick`

Currently a manual target. If `ci-quick` runs it on every PR,
workflow drift gets caught immediately rather than in
quarterly audit packets. Cost: <1s (Python regex / YAML parse).
Worth adding.

### 2. (verification) The 8-seed × 10s budget

`FLAKE_HUNT_SEEDS=8` × `FLAKE_HUNT_FUZZ_SECONDS=10` = 80s of
fuzz compute per nightly run. With build overhead, the job
landed at 90min timeout per packet 002. The audit script
should fail if either variable shrinks from these values
(documented budget) without an explicit policy update.

If the audit currently checks for these specific values:
✓.  If it only checks they're present: worth tightening to
match the documented contract.
