# Review: Task 39 coverage lane portability (009)

## Summary

Operational hardening of the coverage lane discovered while
rerunning it on the rebased branch:

1. `scripts/hardening.sh coverage` uses `cargo +stable` only
   when the configured cargo binary supports rustup toolchain
   directives (avoids "unknown flag +stable" on non-rustup hosts)
2. Falls back to Homebrew LLVM (`llvm-cov`, `llvm-profdata`)
   when `llvm-tools-preview` isn't installed
3. `hardening/careful/src/lib.rs` `am` module conflict fixed —
   `cfg(coverage)` previously defined the module twice; quant
   page-format constants moved into the existing harness module

Lane still runs and produces the same coverage outputs; no
baselines moved.

## Confirmations

- **Lane still passes on its existing host.** Per artifacts:
  `make coverage` produces summary.txt, coverage.json, and
  careful-coverage.json. ✔
- **Delta check unchanged.** All 40 baseline paths at or above
  recorded floors. The `quant/simd.rs` reading at 95.18% vs
  baseline 94.59% is normal floor-vs-actual margin (within
  the ratchet window).
- **Module-conflict fix is the right shape.** Coverage
  instrumentation under `cfg(coverage)` was defining `am`
  twice, which would compile-error on coverage builds.
  Consolidating into one module fixes it.

## Findings

### 1. (verification) Cross-host portability matters now

This packet was prompted by the coverage lane breaking after
the rebase brought in different host context. As more agents
run the lane on different hosts (Mac M5 vs Intel Linux vs CI
runners), portability bugs will surface naturally. The
Homebrew-LLVM fallback is the kind of fix that prevents
"works on my machine" failure modes.

Worth flagging in `docs/hardening.md` (separate from this
packet's scope) that the coverage lane has documented host
requirements:
- A working cargo (any flavor)
- Either `llvm-tools-preview` rustup component OR Homebrew LLVM
- Suitable disk for `target/quality/coverage/`

### 2. (cross-reference) No baseline ratchet here

This packet preserves the lane; doesn't move baselines.
Packets 010 + 011 (this batch) do the ratchets.
