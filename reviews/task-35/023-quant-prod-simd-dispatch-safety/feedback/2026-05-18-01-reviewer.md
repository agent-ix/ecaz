## Notes (non-blocking)

1. **Two-packet split is the right call.** Quant Hadamard (022) and
   quant prod (023) had different invariant shapes — Hadamard was
   pointer-arithmetic-and-target-feature heavy, prod is
   dispatch-and-mode-precondition heavy. Bundling them would have
   diluted the SAFETY comment templates; splitting kept each one
   coherent.
2. **Structural opportunity (Task 50, deferred).** The dispatch
   pattern `match backend() { Avx2Fma => unsafe { ... }, Neon =>
   unsafe { ... }, Scalar => ... }` repeats four times in this file
   alone (and at least once in `hadamard.rs`). A small dispatch
   macro or generic helper that takes the three function pointers
   and asserts the runtime-feature contract once would eliminate the
   repeated `unsafe` blocks at the call sites. Real reduction, not
   relocation, because the unsafe would consolidate into one
   `safe-dispatch!` definition. Belongs to Task 50.
3. **No runtime test results cited.** The request doesn't claim test
   runs — appropriate given the slice is documentation-only and the
   PG-linker symbol issue still blocks the test binary. The
   scalar/AVX2/NEON correctness of these dispatch paths is unchanged
   by the slice, so this is fine.
