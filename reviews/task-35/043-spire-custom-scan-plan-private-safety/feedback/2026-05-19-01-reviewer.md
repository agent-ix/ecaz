# Approve: SPIRE custom_scan plan_private — first SPIRE slice

## Decision

Approve packet 043 at commit `5c0bf9e3`. First substantive SPIRE
slice lands cleanly. The CustomScan plan-private surface is one of
the easier-to-get-wrong files (raw NodeTag dispatch + offset-encoded
private metadata), and the comments name the right invariants.

## What I verified

**Comment quality:**

- **NodeTag dispatch sites:** the contract is two-step — null check
  + NodeTag check before cast. Comments explicitly cite the
  NodeTag confirmation as the precondition for the cast. This is
  the correct framing — without the NodeTag check, the cast would
  be UB; with it, the cast is sound. Naming the check at the cast
  site is exactly what protects future edits.
- **Const decoding (`float4[]`):** cites "const type/nullability
  were checked above; pgrx decodes the float4[] datum without
  taking ownership from PostgreSQL". The ownership phrasing is
  important — datum decoding is easy to misread as taking ownership
  when it's actually borrowing.
- **Plan-private offset-encoded reads:** the most error-prone
  pattern in this file. SAFETY comments name the offset convention
  *and the field it stores* ("mode at offset 0", "index OID at
  offset 1"). That's the binding that protects against silent
  drift if someone reorders the encoding. Good.
- **`custom_scan_plan` / executor entry:** cites
  "executor passes a live CustomScanState whose plan pointer is
  the CustomScan node installed by this provider" — names both the
  caller invariant and the provider-side coupling. Right level.
- **List construction:** cites "all appended strings are copied
  into PostgreSQL memory and the returned List is planner-owned
  plan-private metadata". Distinguishing copy-vs-reference here is
  what protects against use-after-free when the source strings go
  out of scope.

## Notes (non-blocking)

1. **First SPIRE slice — pattern established.** The plan-private
   safety surface is a representative sample of what the rest of
   SPIRE will look like: heavy use of PostgreSQL Node dispatch,
   offset-encoded private metadata, and ownership/copy distinctions
   for memory-context-bound allocations. The comment templates
   landed here are reusable for the rest of `custom_scan/*`,
   `coordinator/*`, and `dml_frontdoor/mod.rs`. The IVF series
   accumulated similar templates (read_page_tuple chains, exclusive
   lock + WAL pairs) — SPIRE's library will be Node dispatch + offset
   encoding + memory-context ownership.
2. **Structural opportunity (Task 50, deferred).** The
   `offset 0 = mode, offset 1 = index OID` convention is currently
   enforced by SAFETY-comment discipline. A typed
   `SpireCustomPrivate { mode: SpirePlanMode, index_oid: Oid }`
   struct with explicit `to_list` / `from_list` round-trip helpers
   would replace the per-offset unsafe reads with a single round-trip
   surface, eliminating ~5 unsafe sites and removing the silent
   drift risk if someone reorders. Strong Task 50 candidate.
3. **`final-diff.patch` artifact is a nice touch.** Including the
   final review diff as a packet-local artifact (rather than only
   linking the commit SHA) is helpful when reviewing offline or
   without git access. Worth recommending for future packets that
   have multiple validation cycles.
