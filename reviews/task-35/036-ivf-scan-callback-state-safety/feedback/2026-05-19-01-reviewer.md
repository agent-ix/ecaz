## Series-level notes

### Architectural seams identified

Like the page series, the scan series picked real seams:

- **036 callback state:** the boundary between PostgreSQL's
  callback ABI and the IVF scan opaque state — accessor patterns
  for stored query / selected lists.
- **037 allocation:** the boundary between IVF allocations
  (palloc/MemoryContext) and Rust slices.
- **038 rerank probe materialization:** the boundary between scan
  state and heap rerank materialization (probably touching
  `index_fetch_heap` patterns).
- **039 debug tail:** EXPLAIN counter extraction and series
  closeout.

These map to real layers in the file rather than arbitrary line
ranges, which kept each packet's SAFETY contracts coherent within
its slice.

### Structural opportunities (Task 50, deferred)

1. **`from_raw_parts(self.X, self.X_count)` accessors** appear
   multiple times in 036 with the same null/zero-length reject
   prelude. A small `slice_field!(self, X, X_count)` macro or a
   `RawSlice<T>` field type holding `(ptr, len)` with a safe
   `.as_slice()` method would absorb the unsafe at every accessor
   site. Real reduction.
2. **AM-callback `unsafe { pgrx::pgrx_extern_c_guard(|| ...) }`
   wrapping** — same as flagged in 024 cost callbacks. Cross-file
   cleanup.
3. **`(*scan_desc).opaque.cast::<EcIvfScanOpaque>()`** chain in 039
   debug code is a candidate for a typed accessor
   `scan_opaque_from_iss(index_state)` returning
   `Option<&EcIvfScanOpaque>`.

These belong in Task 50.
