**Invariant summary (artifacts/hnsw-invariant-summary.md):**

The 7-bullet core invariant graph is dense and accurate:

1. **AM callback boundary** — `pgrx_extern_c_guard` + raw pointers
   not retained beyond callback scope. Generalizes the pattern
   from every AM in the project (IVF, SPIRE, DiskANN, HNSW).
2. **Metadata at block 0 + GenericXLogTxn rewriting** — names the
   block-0 metadata convention, exclusive buffer lock, and borrow
   scope ("readers borrow metadata page bytes only while the
   buffer guard pins the page"). Matches the page-series template.
3. **Line-pointer chain** — the explicit chain `page pointer →
   pd_lower line pointer count → checked item id → tuple
   offset/length bounds → borrowed tuple byte slice`. This is the
   most important sentence in the document — it makes the
   bounds-check ordering auditable in one read. Future reviewer
   approaching HNSW page code reads this and knows what to check
   at every tuple access.
4. **Graph storage decoding driven by metadata descriptor** —
   tuple tag + layout checks before TurboQuant/hot/cold/grouped
   decoding. Names the dispatch invariant.
5. **Scan state owns caches/frontier/results/scratch/slots for
   scan lifetime** — names the scan-opaque ownership scope and
   the inner buffer/slot guard scope distinction.
6. **Heap source scorer chain** (heap_relation + snapshot +
   reusable slot) — generalizes the pattern that appears across
   insert.rs (098) and vacuum.rs (099). Names the "copied or
   consumed before slots are cleared" detoast-copy invariant from
   the SPIRE scan/relation (082) detoast pattern. **This is the
   same scorer pattern as the SPIRE active-epoch chain — different
   contents, identical safety shape.**
7. **Parallel build DSM-owned state** — bounds-checked node
   indices + lock-or-atomic protocol for atomic-field access.
   Generalizes the 094 SAFETY-comment pattern.

**RAII inventory (5 guard types):** `LockedBufferGuard`,
