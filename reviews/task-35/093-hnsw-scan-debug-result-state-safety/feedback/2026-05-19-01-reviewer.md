### Structural opportunities (Task 50, deferred)

The build_parallel series surfaces a major Task 50 candidate not
previously listed:

**DSM atomic field wrapper.** The pattern
`unsafe { atomic_u32.load_acquire() } / unsafe {
atomic_u32.store_release() }` in 094 is currently protected only
by SAFETY-comment discipline naming the lock-or-atomic precondition.
A typed `DsmAtomic<T> { field: *mut pg_sys::pg_atomic_uint32 }`
wrapper with safe load_acquire/store_release/compare_exchange
methods would absorb every parallel-build atomic site behind one
safe-API surface, and the unsafe consolidates into the wrapper
constructor. **Strong Task 50 candidate** — concurrent code is
exactly where structural enforcement pays back most.

The other deferred candidates (Active-epoch anchor, CustomScan
private offsets, NodeTag dispatcher, AM-callback wrapper macro,
composite heap+index guard) from 083 remain.
