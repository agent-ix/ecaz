# Suite explain profile-aware review

`ecaz bench suite` explain steps now accept an optional
`profile` field, fall back to suite default then `ec_ivf` for
backward compatibility. Emits the right tuning GUC
(`ec_ivf.nprobe`, `ec_spire.nprobe`) and rerank width GUC for
the selected profile. Includes the matching cost-snapshot
function in generated explain SQL. No blocking findings.

## Confirmations

1. Backward compatibility is preserved: existing suites that
   don't specify `profile` still target `ec_ivf`.
2. Cost snapshot function is selected based on profile, so the
   explain artifact carries both the EXPLAIN plan and the
   modeled cost inputs in one query — exactly the planner
   proof artifact a reviewer needs.
3. Tests cover both default IVF and SPIRE explain SQL
   generation.
