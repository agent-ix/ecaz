## Answers to the packet's review questions

- **Is 16 bytes the right Phase 11 fixed source payload width for Leaf
  V2 production writers?** Yes. 16 bytes covers UUIDv4/UUIDv7 directly
  and accommodates SHA-128-class hashes of composite source keys, which
  is the realistic shape of any DDL-level identity column or
  distributed-ingest stable key. It also keeps the Leaf V2 stored
  `SpireVecId` at 17 bytes (1 discriminator + 16 payload), which fits
  the Leaf V2 fixed-stride invariant from packet 30697 without a
  format bump.
- **Correct to keep variable global payloads available for diagnostics
  while requiring fixed for the live writer contract?** Yes. The
  variable path is needed for row-encoded compatibility helpers and
  for diagnostic ingest paths that may not yet canonicalize, while the
  Leaf V2 base storage path enforces fixed-width via the layout check.
  The split prevents silent variable-length writes into Leaf V2 base
  objects.
- **Does the provider deferral accurately capture the current AM
  limitation, or should the next slice target an identity-column DDL
  contract immediately?** The deferral accurately captures the
  limitation. The DDL contract is the right next slice target; see
  P2 #1 below.

## P2 — must close before live writer global-ID claims

1. **Provider selection ADR.** Two provider surfaces are named in the
   design doc with a recommendation toward an identity-column /
   expression contract, but no ADR locks the decision. Recommend
   opening a small ADR (number after ADR-062) before the next 11.2
   code slice that:
   - selects identity column / expression as the v1 production
     provider,
   - specifies how the AM reads it during build and insert,
   - specifies the canonicalization rule that yields the 16-byte
     payload (e.g. UUID-as-bytes, BYTEA passthrough, SHA-256-truncated
     of TEXT, etc.),
   - states behavior when the column is NULL or the expression is not
     stable across nodes (strict reject vs degraded fall-back to
     `0x01`),
   - states behavior on `ec_spire` indexes that lack the identity
     column (continue local-only, with diagnostics).

   Without this ADR locking the canonicalization rule, downstream
   tests for cross-node dedupe will be hard to write deterministically.

## P3 — informational
