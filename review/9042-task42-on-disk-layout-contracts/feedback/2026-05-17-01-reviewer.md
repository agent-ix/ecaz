# Review: Task 42 on-disk layout contracts (9042)

## Summary

First Task 42 hardening slice. Publishes named byte-offset
constants for the clearest currently-encoded on-disk layouts and
pins them as compile-time/test-time assertions in
`tests/size_of_assertions.rs`:

- generic `ItemPointer` wire fields
- HNSW legacy/current metadata payload sizes + current metadata
  offsets
- HNSW element/grouped-hot/turbo-hot/rerank/grouped-codebook/neighbor
  tuple fixed offsets
- DiskANN Vamana metadata payload size + field offsets

Also adds `docs/on-disk-format.md` as the Task 42 inventory doc,
documenting the little-endian convention and explicitly naming
remaining gaps.

I'm satisfied with the slice shape. This is the right opening
move for Task 42: lock the current layouts down with named
constants and assertions before doing any version-bump work, so
any future drift fails loudly at `layout-check` time rather than
silently corrupting reads.

## Confirmations

- **Constants reflect current codec without changing runtime.**
  Spot-checked the asserted offsets against the current
  encode/decode paths: the offsets are read-only assertions of
  what the codec already produces, not changes to encoding.
  Migration is zero-risk for runtime behavior. ✔
- **Encoded byte contracts, not host layout.** The assertions
  pin little-endian byte offsets (e.g.,
  `magic at byte 0..4`, `format_version at byte 4..6`) rather
  than `std::mem::size_of` of host structs. This is the right
  contract for the "qemu cross-arch decode lane" goal Task 42
  names — a big-endian host running the same code would still
  produce the same bytes if the codec respects this.
- **`docs/on-disk-format.md` is a useful inventory.** Records
  the current static coverage (`make layout-check`), the
  little-endian convention, and explicit "remaining gaps"
  pointing at SPIRE / IVF coverage to come. Reasonable
  Task 42 progress doc.
- **No behavioral changes.** No on-disk format changes; no
  encoder/decoder edits.

## Findings

### 1. Scope of `layout-check`

`make layout-check` runs `tests/size_of_assertions.rs`. As Task
42 expands, this single test file will grow to ~1000+ lines
across all AMs and SPIRE. Worth considering whether to split
into per-AM submodules (`tests/size_of/hnsw.rs`,
`tests/size_of/diskann.rs`, etc.) once the file approaches that
size — for navigation, not correctness. Not actionable in 9042;
flagging for when the file gets large enough to be unwieldy.

### 2. Endian-failure mode coverage

The current slice asserts byte **offsets**, but doesn't yet
verify that the bytes encode/decode correctly under a different
endian host. That's the qemu cross-arch decode lane Task 42
names in the task doc. 9045+ start the golden-fixture work
which provides the actual byte-stability evidence; together
those produce the full coverage story.

### 3. ItemPointer and shared types

`ItemPointer` is shared across AMs — the assertions pin its
wire shape once for all AMs. Right choice: avoid duplicating
the offset constants per AM. If `ItemPointer` ever bumps
format, this single set of assertions catches every AM at
once.
