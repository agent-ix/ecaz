# Feedback: N3 Duplicate Handoff Gate

**Head reviewed:** `a7a43ee` (landed)

## TL;DR

Foreign handoff now returns a `ParallelScanOutputState` with a new
`SuppressedDuplicate` variant so stale foreign rows whose heap TIDs another
worker already emitted are consumed off the source slot without being
surfaced to the caller. The round-robin debug helper is generalized to N
workers and a PG18 `n=3` gate wires it up.

## Verification

- `try_take_parallel_scan_handoff_output_state` now checks
  `foreign_worker_recently_emitted_heap_tid(opaque, output.heap_tid)` after
  draining the admitted/hidden source and, when true, clears local blocker
  state, republishes the worker snapshot, and returns `SuppressedDuplicate`
  without incrementing the foreign-selected counter (scan.rs:4886-4902,
  current line numbers).
- All three call sites (`resolve_local_only_parallel_scan_duplicate`,
  `try_take_parallel_scan_next_output`, deferred retry) handle
  `SuppressedDuplicate` explicitly and either continue or propagate upward
  without dropping live local staged rows (scan.rs:4663-4672, 5067-5075,
  5478-5497).
- Thin wrapper `try_take_parallel_scan_handoff_output` maps Suppressed→None so
  remaining legacy callers see a no-op (scan.rs:4947-4956).
- `DebugParallelRoundRobinDetails` with per-worker streams/snapshots is
  generalized; `..._parallel_round_robin` reuses `..._many_details` and slices
  the first two workers for back-compat with the n=2 helper (scan_debug.rs).
- n=3 PG18 gate checks combined==serial, all 3 workers contribute, no
  hidden/blocker/active stranded state, and both local-only +
  deferred-local-emit counters are zero (lib.rs:15931+).
- Unit test `..._suppresses_already_emitted_foreign_heap_tid` asserts
  `SuppressedDuplicate`, source slot cleared to `INVALID`,
  `stats_parallel_handoffs_foreign_selected_pending == 0`, local staged row
  preserved (scan.rs:10032+ in diff).

## Findings

None. Suppressing at the foreign-handoff layer is the right seam — callers
operate on a `ParallelScanOutputState` rather than re-inspecting source
slots. The suppression path republishes the worker snapshot, which is
necessary so subsequent workers see the advanced source generation.
