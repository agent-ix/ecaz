# Review Feedback

Use this directory for outside-review responses and follow-up discussion that refer to an open
review request.

Guidelines:
- Keep the review request files themselves in `review/`.
- Put feedback and back-and-forth notes under `review/feedback/<request-slug>/`.
- Store each outside response as its own dated reviewer document, for example
  `review/feedback/13-amgettuple-empty-index-noop/2026-04-05-claude-opus.md`.
- Leave requests open in `review/README.md` until outside review has happened; feedback files do
  not close a request by themselves.
