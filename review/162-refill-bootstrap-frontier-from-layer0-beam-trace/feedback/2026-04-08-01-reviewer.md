## Summary

Makes `scan.rs` use the graph-owned layer-0 beam runner for bootstrap refill after a consumed source. Uses single-step expansion (`ef_search = 1`) to preserve the existing bootstrap-after-success contract.

**Verdict: Approved.**

## Assessment

Using `ef_search = 1` for the refill beam trace is the right compatibility bridge. The current bootstrap contract is "expand one consumed source, discover its immediate successors." A single-step beam trace produces exactly this behavior — one expansion, frontier = immediate successors. The `refill_successors_with_successors` helper at line 207 encapsulates this contract cleanly.

The synthetic seed score of `0.0` for the single-source trace (line 221) is acceptable at this seam. The refill step is seeded from a consumed candidate whose score doesn't matter for expansion order (there's only one candidate to expand). The discovered successors get real scores from the `score_candidate` closure.

The runtime now uses the graph-owned beam runner in both the right places: entry seeding (review 161) and post-consume refill. This is the correct two-point integration before deeper traversal changes.
