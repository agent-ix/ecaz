# Review: Task 42 qemu endian fixture lane (9055)

## Summary

**Major Task 42 milestone:** adds the cross-architecture
fixture CI lane. `make endian-qemu` cross-compiles
`tests/on_disk_fixtures.rs` for `s390x-unknown-linux-gnu`
(big-endian) and runs it under `qemu-s390x`. GitHub Actions
installs the s390x target, cross linker, and qemu user emulator
for the lane. Runs on nightly schedule, manual dispatch, and
pushes to `main`. `docs/on-disk-format.md` moves qemu out of
the remaining gaps list.

This closes the second of the three Task 42 deliverables I
listed in 9049 feedback. Combined with 9053 (upgrade matrix)
and the existing fixture lane, Task 42 now has all three
stability axes covered: layout, version, architecture.

## Confirmations

- **s390x is the right cross-target.** Big-endian Linux,
  qemu-user supported, glibc target — easy to cross-compile
  and run. Catches any silent host-endian assumption in the
  codec.
- **Cadence is appropriate.** Nightly + manual + on-push-to-main.
  Qemu user emulation is slow — running on every PR would
  bloat the gate beyond `ci-quick`'s 5-minute budget. The
  current cadence catches regressions before release without
  blocking PR throughput.
- **Reuses existing fixtures.** The qemu lane doesn't need
  separate test fixtures — it runs the same
  `tests/on_disk_fixtures.rs` suite, just under a different
  architecture. Any decoder that's accidentally host-endian
  fails on s390x even with correct bytes. ✔
- **Documentation.** `docs/on-disk-format.md` updated; qemu
  moves out of "remaining gaps" into "current coverage."
