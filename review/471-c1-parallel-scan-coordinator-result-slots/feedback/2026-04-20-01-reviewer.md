# Feedback: Parallel Scan Coordinator Result Slots

**Reviewer:** Claude (Opus 4.7)

**Head reviewed:** `575210f` (landed; reviewed retrospectively)

## TL;DR for the agent

**Merged correctly.** One-result-slot-per-worker is the right interim
seam — it carries enough state for the next merge slice while keeping
the ownership rule trivial ("the worker that owns slot N owns result
slot N"). Tying mutation to worker-slot ownership + rescan epoch is
the correct lifetime contract.

## Verification performed

- `publish_parallel_scan_coordinator_result_slot_runtime_snapshot`
  (`src/am/common/parallel.rs:1175–1267`) double-guards: it checks
  the worker-slot's epoch AND claim state, plus the result-slot's
  own epoch. Correct — a released worker cannot publish into a
  result slot; a stale-epoch result slot cannot be mutated.
- `clear_parallel_scan_result_slot_with_attachment`
  (`parallel.rs:1089–1134`) is the release-side reset; the worker-slot
  release path calls it before flipping the worker flag to FREE. Order
  is right — if the order were reversed, a concurrent publisher could
  race with the clear. Today it's single-writer so no race, but the
  order-choice is still the right one.
- Release-side staged-result cleanup is wired through
  `release_parallel_scan_worker_slot` (`parallel.rs:1136–1173`) before
  the CAS-to-FREE. Good.

## Findings

### 1. Score-valid flag bit couples score and tid validity

**Severity:** Low
**File:** `src/am/common/parallel.rs:1238–1240`

```rust
if snapshot.score != 0.0 || snapshot.element_tid.is_valid() {
    flags |= EC_PARALLEL_RESULT_SLOT_SCORE_VALID;
}
```

This marks SCORE_VALID iff score is nonzero OR tid is valid. If a
genuine score of exactly 0.0 is ever meaningful (e.g. a normalized
distance that happened to hit 0), the flag becomes a function of
tid-validity alone — which is fine today but subtle. Worth a comment:
"SCORE_VALID tracks that a publish has produced a (tid, score) pair;
score == 0.0 alone is not evidence of an unpublished slot because
tid also gates the flag." Or tighten the API: callers always pass
`Option<f32>` for score and the flag mirrors `.is_some()`.

### 2. `approx_rank_base` encoding via `u32::from_ne_bytes(i32::to_ne_bytes(..))`

**Severity:** Informational
**File:** `src/am/common/parallel.rs:1227–1230`

Round-trips through native-endian bytes to stuff an `i32` into an
`AtomicU32`. Correct but terse. `approx_rank_base.unwrap_or_default()
as u32` / `as i32` on load is the same operation with less ceremony.
Cosmetic.

### 3. Ownership rule scales to the next slice

**Severity:** Informational (positive)

"Worker N owns result slot N" is a decision that survives packet 472's
heap/selection work unchanged. Good structural call.
