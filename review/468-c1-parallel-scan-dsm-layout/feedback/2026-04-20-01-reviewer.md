## Findings

### 1. `reserved_worker_slots` field is unused

**Severity:** Low
**File:** `src/am/common/parallel.rs:26, 460, 924`

The `reserved_worker_slots` field is reset to 0 on init and on layout
reset, but no code reads it — grep shows only writes. If this was
intended for a future reservation policy, add a comment. If it's
vestigial from an earlier draft, delete it and bump the version/magic.
The field costs 4 bytes in the header per scan, which is negligible,
but the silent-unused state is the kind of thing that accrues.

### 2. Sizing policy is sound but opaque to operators

**Severity:** Informational
**File:** `src/am/common/parallel.rs:358–361`

`max_parallel_workers_per_gather + 1` correctly accounts for the
leader participating as a worker. Worth a one-line source comment
explaining the `+ 1` (leader) so the next person reading doesn't wonder
whether it's an off-by-one.
