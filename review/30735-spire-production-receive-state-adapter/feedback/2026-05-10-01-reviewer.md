## Answers to the packet's review questions

- **`run_compact_candidate_receive(...)` the right executor-owned
  boundary before AM scan integration?** Yes. AM scan now has one
  call site (executor method) instead of two (request build +
  adapter run + result apply), with the state-machine transitions
  hidden inside the executor. C5 should be a thin shim that
  invokes this method and reads the resulting state, not a
  reimplementation of the receive flow.
- **Adapter failures should apply through the same
  `CandidateReceiveFailed` transition as decoded receive
  failures?** Yes. From the executor state machine's perspective,
  a per-node failure is a per-node failure regardless of whether
  the failure originated in request-building, transport, decode,
  or validation. The `failure_category` field carries the
  diagnostic specificity; the state transition stays uniform.
- **Checklist split honest?** Yes. Compact receive production
  state is now end-to-end wired (`Planned` → `TransportReady` →
  `CandidateReceiveReady|Failed` all reachable through executor
  methods). AM scan integration and final-row delivery (Stage D
  heap resolution) remain open.
