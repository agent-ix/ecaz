## TL;DR

**Merged correctly.** The probe-then-reject fast path keeps duplicate
and full-window-loser admissions off the coordinator serializer when
the probe state is still current. The "full coordinator snapshot"
currency check is the right granularity — anything finer risks
racing between sub-fields.

## Verification performed

- `coordinator_admission_probe_is_current` compares the whole
  coordinator snapshot, not per-field, so it can't decide "probe is
  half-stale." Correct for a validity check that gates a lock skip.
- `admit_parallel_scan_selected_pending_output` now branches:
  probe-result-would-not-admit + probe-still-current → return early,
  no lock. Otherwise → lock and revalidate via the existing path.
- Zero-limit coverage added explicitly so the "never admits" case is
  pinned outside of duplicate/full-window coincidences.

## Findings

### 1. Whole-snapshot equality is the right validity check

Narrower comparisons (e.g. "only admitted_generation and
published_generation match") would skip too much work that the
mutating helper still re-does under the lock. Whole-snapshot
equality is both correct and simple.

### 2. Zero-limit case is a good test pin

Tests that fail closed on zero-limit inputs catch the corner where
the probe should say "no admission ever" even for otherwise-admissible
candidates. Nice explicit coverage.
