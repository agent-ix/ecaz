# Review Feedback: Graph-First Primary Scan

**Verdict: Approved with notes**

## Phase boundary

Using `result_state.current().has_element()` as the signal for "graph traversal already produced
output" is the right runtime boundary. Once graph-ordered results have started flowing, falling back
to linear would break ordering guarantees. The change from always-available linear fallback to
graph-owns-to-exhaustion is the core A3 semantic shift and it's implemented at the right level.

## Test relaxation

The pg test changes move from exact `observed == expected` equality to:
1. Nearest neighbor is present
2. No duplicate emission
3. All emitted tids belong to the table

This is the right shape for A3 — the graph traversal doesn't guarantee it will visit every indexed
element, but it must not hallucinate or duplicate. The `observed_tids.len() < expected_tids.len()`
assertion in the multipage test explicitly encodes the "no silent linear tail" contract.

## One concern

The duplicate test now checks that the first two expected tids (the duplicate-coalesced pair) are
both present, but doesn't assert ordering between them. That's probably fine since duplicate drain
order is not part of the scan contract, but worth confirming that's intentional rather than an
oversight.
