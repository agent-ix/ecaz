### 1. `Makefile` bench targets invoke deleted scripts

**Severity:** Medium
**File:** `Makefile:123–129`

```

	bash scripts/bench_sql_latency.sh

bench-storage:

```

bench-sql-latency` / `make bench-storage` / `make bench-recall-sql`
will fail with "No such file or directory." Either retarget these
targets at the equivalent `ecaz` subcommand or delete the targets
outright.

### 2. `tests/recall_integration.rs` spawns deleted Python scripts

**Severity:** Medium
**File:** `tests/recall_integration.rs:1760–1761`


Both scripts were deleted. The surrounding code uses

off), but any run that hits it will fail. Route through the
equivalent `ecaz corpus prepare` / `ecaz corpus load` (or their
underlying crate APIs) instead.
