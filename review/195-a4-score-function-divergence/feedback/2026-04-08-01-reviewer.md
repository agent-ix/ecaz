# Feedback: A4 Score Function Divergence

Request:
- `review/195-a4-score-function-divergence/request.md`

**Reviewer:** Claude (Opus)
**Date:** 2026-04-08

## Verdict: Valid Concern, But Now Shown to Be Secondary

The divergence identified is real and correctly described:

- Build: `score_offset - MSE_IP_codes(a, b)` — symmetric, code-to-code, no gamma, no QJL
- Search: `-(MSE_LUT_sum + gamma * qjl_scale * QJL_sum)` — asymmetric, query-to-code, with gamma+QJL

However, evidence from review 201 (scorer ablation) now shows this is not the primary cause of poor
recall:

| Variant | Recall@10 |
|---------|-----------|
| exact (full scorer) | 49.5% |
| gamma_zero | 49.0% |
| code_proxy (symmetric, matching build) | 47.5% |
| decoded (reconstruct → exact dot) | 49.0% |

The code_proxy variant — which uses exactly the same symmetric scoring as the build — achieves 47.5%
vs the full scorer's 49.5%. A 2% gap. This confirms the build/search distance divergence is real but
contributes only ~2% recall, not the 40+ points needed to reach the A4 gate.

## Suggested Investigation Response

1. **Score correlation test**: Still worth doing if cheap, but the ablation data already implies high
   correlation since all variants land within 2%.
2. **Gamma-free search test**: Done in review 201 — gamma contributes <1%.
3. **The real explanation**: All scoring variants hit the same ~50% ceiling because the quantized codes
   themselves cannot preserve fine-grained ranking at d=1536 on this corpus. The divergence between
   build and search scoring is a secondary effect.

## Disposition

**Downgrade to secondary.** Keep on the investigation list but do not prioritize ahead of the
quantizer-ceiling work identified in review 201. If a future quantizer improvement raises the exact
ceiling above 80%, revisit whether the build/search divergence becomes the new bottleneck.
