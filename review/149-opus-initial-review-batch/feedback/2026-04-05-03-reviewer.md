# Review: with_locked_metadata_page Always Writes Back

**File:** `src/am/mod.rs:771-803`
**Severity:** Medium (WAL amplification, unnecessary I/O)
**Category:** Optimization / correctness

## Finding

`with_locked_metadata_page` always writes the metadata page back via GenericXLog, even when the closure did not modify the metadata:

```rust
unsafe fn with_locked_metadata_page<T>(
    index_relation: pg_sys::Relation,
    f: impl FnOnce(&mut page::MetadataPage) -> T,
) -> T {
    // ... read metadata ...
    let result = f(&mut metadata);
    // Always writes back, even if metadata is unchanged:
    let mut wal_txn = unsafe { wal::GenericXLogTxn::start(index_relation) };
    let page = unsafe { wal_txn.register_buffer(buffer, pg_sys::GENERIC_XLOG_FULL_IMAGE as i32) };
    let metadata_bytes = metadata.encode();
    unsafe { write_metadata_bytes(page, &metadata_bytes) };
    unsafe { wal_txn.finish() };
    unsafe { pg_sys::UnlockReleaseBuffer(buffer) };
    result
}
```

This is called on every `aminsert`, meaning every insert generates a full-image WAL record for the metadata page even when metadata hasn't changed (which is the common case after the first insert, since only the shape validation + duplicate check paths run).

Specifically, in `tqhnsw_aminsert` (line 211), the closure returns early on line 239 after `coalesce_duplicate_heap_tid` without modifying metadata, but the write-back still happens.

Wait -- actually re-reading: the early `return` on line 239 exits the closure, and `with_locked_metadata_page` still runs the post-closure write. This means even duplicate coalescing (which doesn't touch metadata at all) generates a metadata WAL record.

## Recommendation

Either:
1. Have the closure return a flag indicating whether metadata was modified, and skip the write-back when false.
2. Compare the encoded metadata before and after the closure; skip write if identical.

Option 1 is simpler and avoids the encode cost:

```rust
unsafe fn with_locked_metadata_page<T>(
    index_relation: pg_sys::Relation,
    f: impl FnOnce(&mut page::MetadataPage) -> (T, bool), // (result, modified)
) -> T
```

## Action Required

Refactor `with_locked_metadata_page` to avoid unconditional WAL writes. This will reduce WAL volume significantly under insert-heavy workloads.
