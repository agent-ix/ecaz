# Review: Lloyd-Max Empty Cluster Handling

**File:** `src/quant/codebook.rs:80-106`
**Severity:** Low (degenerate centroids)
**Category:** Correctness / edge case

## Finding

In the Lloyd-Max iteration, when a cluster has zero mass (no grid points assigned), the centroid is not updated:

```rust
for i in 0..num_centroids {
    if masses[i] > 0.0 {
        let new_c = next_centroids[i] / masses[i];
        max_diff = max_diff.max((new_c - centroids[i]).abs());
        centroids[i] = new_c;
    }
    // empty cluster: centroid stays at its initial position
}
```

This means an empty cluster's centroid remains at its initial position from the uniform spacing, which could be far from any actual probability mass. For high-dimensional vectors (d >> 100), the Beta distribution concentrates tightly around 0, so outer centroids might get zero mass.

The effect: some codebook entries would be "dead" (never nearest to any input coordinate), wasting quantization levels. This reduces effective codebook utilization but doesn't cause incorrect results.

## Recommendation

This is a known property of basic Lloyd-Max. Options for improvement (not urgent):
1. **Dead centroid splitting:** Move empty centroids to the midpoint of the highest-mass cluster
2. **Initial seeding:** Use quantile-based initial positions instead of uniform spacing (better adapted to the Beta distribution's shape)

The current behavior is acceptable because:
- The codebook is dimension-dependent (passed as parameter `d`)
- For realistic dimensions (384-4096), the Beta distribution isn't so concentrated that many centroids go empty
- `num_points = 20_000` provides good grid resolution

## Action Required

None for v0.1. Consider adding a debug log for empty clusters if quantization quality becomes an issue in benchmarks.
