# Review: Dimension Truncation to u16 in encode_to_tqvector

**File:** `src/lib.rs:244`
**Severity:** Medium (silent data corruption for large dimensions)
**Category:** Correctness / safety

## Finding

```rust
pack(
    embedding.len() as u16,  // silent truncation if dim > 65535
    bits,
    seed,
    encoded.gamma,
    &code_bytes,
)
```

If a user passes an embedding with more than 65,535 dimensions, `embedding.len() as u16` silently truncates. The code_len computed from the truncated dimension would not match the actual code bytes, causing corrupt datums.

In practice, embeddings this large are rare (most models produce 384-4096 dims), but this should be validated explicitly rather than silently truncating.

## Recommendation

Add a check before the cast:

```rust
let dim = u16::try_from(embedding.len())
    .unwrap_or_else(|_| pgrx::error!("embedding dimension {} exceeds maximum 65535", embedding.len()));
```

## Action Required

Add explicit dimension validation in `encode_to_tqvector`.
