## Findings

### 1. (cross-cutting, see 31180 feedback) New module-local guard struct

This packet predates the 31180 consolidation by ~1 hour, so the
`OpenedStoreRelation` / `OpenedRelationsGuard` shapes are
module-local. The "single auxiliary store relation" case is
**directly replaceable** by `IndexRelationGuard::open(oid,
lockmode, caller)` from `src/storage/relation_guard.rs`.

The aggregate `OpenedRelationsGuard` is **genuinely different**:
it manages partial-construction rollback for a set of guards,
which the single primitive doesn't model. Two options for the
follow-up:

1. Keep `OpenedRelationsGuard` specialized to this module ÔÇö it's
   a higher-level abstraction over `Vec<IndexRelationGuard>` with
   a specific transfer-on-success semantic. Document the
   specialization.
2. Promote a generic `RelationGuardSet` primitive into
   `relation_guard.rs` that wraps `Vec<IndexRelationGuard>` with
   `into_inner` / drain-on-Drop semantics. Useful if other
   modules end up needing the same shape (probably not many).

Either is acceptable. The mechanical part ÔÇö replacing
`OpenedStoreRelation` with the primitive ÔÇö should happen in the
follow-up "Migrate remaining module-local relation guards"
packet I proposed in 31180 feedback.
