## Findings

### 1. Foreign-SelectedPending source is the right read

Using `runtime.element_tid` of the selected slot (rather than some
historical pending-output payload) catches the current ownership
state under the coordinator lock. If the foreign worker advances
its pending output between blocker observation and retry, the
blocker's element changes with it â€” which is exactly what the
retry logic from 507 compares against.

### 2. INVALID guard on same-element check is correct

`blocker_element_tid != INVALID && blocker_element_tid == element_tid`
(scan.rs:3416â€“3417) correctly ignores AdmissionWindow blockers
(which always have `INVALID` element) and falls through to the
existing retained-blocker retry path. Without the INVALID guard,
an INVALID/INVALID false match could fire when the staged row
itself has no valid element.

### 3. Retry lifecycle still clean

Drop path clears `retained_parallel_owned_output_blocker` before
returning (scan.rs:3419), so the 507 retention machinery doesn't
see this drop as a stale retry candidate. Symmetric with the
AdmissionWindow branch.
