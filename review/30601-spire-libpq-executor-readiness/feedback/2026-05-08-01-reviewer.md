# Libpq executor readiness review

`ec_spire_remote_search_libpq_executor_readiness` decomposes the
broad `requires_libpq_transport` gate into six concrete steps:
descriptor → secret → connection → pipeline → send → receive →
merge. Plus a static step contract. No blocking findings.

## Confirmations

1. **Three-state precedence is right:** descriptor blocker →
   executor required (with first concrete step
   `conninfo_secret_resolution`) → pass-through to dispatch
   summary status. Each branch sets the action strings + next
   step coherently
   (`remote_candidates.rs:1497-1568`).
2. **Coordinator gate now reports
   `libpq_executor_status` and `libpq_executor_next_step`**
   (line 2124-2125). One SQL call answers "is the next blocker
   descriptor, secret resolution, connection open, pipeline
   mode, send, receive, or merge?" — that's the right shape for
   the eventual executor binding.
3. **Static step contract** lets a remote operator inspect the
   ordered executor steps without a query-specific plan, which
   matches the 30574 descriptor-contract pattern.
4. PG18 coverage exercises descriptor-blocked, local-only
   ready/no-op, and active-descriptor `requires_libpq_executor`
   states + coordinator-gate propagation.

## Findings

- **Six new action strings outside the central registry.**
  `"resolve_conninfo_secret_reference"`,
  `"open_libpq_connection"`,
  `"enter_libpq_pipeline_mode"`,
  `"send_remote_search_request"` (lines 1529-1532) are literals.
  Same pattern as the executor step name constants
  (`SPIRE_REMOTE_EXECUTOR_STEP_DESCRIPTOR`,
  `SPIRE_REMOTE_EXECUTOR_STEP_SECRET`, etc.) that this packet
  introduces — the action labels should live next to the step
  name constants for symmetry. Cosmetic; track with the
  ongoing string-registry work.

- **No invariant test pinning step contract vs. readiness
  output.** The static step contract lists the ordered steps;
  `remote_search_libpq_executor_readiness_from_dispatch_summary`
  open-codes the action labels. If a future change to the
  step contract reorders or renames steps, the readiness output
  would silently drift. Same pattern as the 30573 tie-breaker
  and 30588 degradation invariants — recommend a similar
  invariant test. Forward-looking, not blocking.
