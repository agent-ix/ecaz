# Review: SPIRE Libpq Operations Runbook (30951)

## Summary

Phase 12.9 docs slice — `docs/SPIRE_LIBPQ_RUNBOOK.md` collects
the libpq security/ops surface in one operator-readable
runbook:

- `sslmode` + libpq security option preservation;
- raw-conninfo non-exposure;
- sanitized auth/cert/conninfo failure handling in
  strict + degraded modes;
- `max_prepared_transactions` readiness;
- orphaned prepared xact recovery (manual, per 30943);
- credential rotation deferred;
- audit log schema deferred.

## Findings

### P2 — Scoping discipline

The runbook correctly names each Phase 12.9 item without
overclaiming a TLS/credential-rotation/audit *subsystem*. The
deferral lines are explicit (credential rotation, audit
schema), which is the right way to keep operators from
assuming missing capability is implemented just because it's
mentioned in the runbook. Good.

### P2 — Strict/degraded sanitization compatibility

The strict/degraded failure wording must reuse the same
sanitized labels the existing diagnostics surface emits —
otherwise an operator following the runbook to grep logs
will miss live failures whose label spelling differs. From
the request, the runbook is "compatible with existing
sanitized diagnostic labels and does not expose raw conninfo
or remote error text." Worth one explicit sentence in the
runbook calling out the exact label vocabulary an operator
should grep for (e.g., names of the SQLSTATEs, the sanitized
category strings) so the doc stays a useful operator tool
rather than a high-level summary.

### P2 — Manual recovery cross-link

Make sure the orphaned prepared xact recovery section
explicitly links to the 30914 manual recovery runbook AND
notes the 30943 deferral of the automated helper, so
operators don't go looking for `ec_spire_recover_orphaned_prepared_xacts`
expecting it to exist. Per 30943 it does NOT ship in v1.

### P3 — Credential rotation deferral content

The credential-rotation deferral note should at minimum:

- name what an operator does *today* if they need to rotate
  remote DB credentials (probably: pause writes, update
  conninfo source, refresh descriptors, resume — the same
  shape as DDL ordering from 30931);
- name what's missing for automated rotation (probably:
  hot-reload of remote node descriptors without descriptor
  refresh round-trip).

If the runbook only says "credential rotation deferred" without
the manual workflow, operators will improvise. If it includes
the manual workflow already, ignore.

### P3 — Audit log schema deferral

Same shape: deferral note should explain what observability
*does* exist today (the diagnostic snapshots,
degraded skip report from 30954) so operators know what
substitute observability is available before audit logs
exist. Optional but useful.

## Reviewer recommendation

Accept. Honest runbook with the right deferral discipline.
The P2 items are wording sharpenings, not blockers.
