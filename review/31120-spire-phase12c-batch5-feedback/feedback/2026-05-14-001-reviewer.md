## What's solid

- **`732` idle-in-tx timeout** — opens a transaction-scoped cursor
  over `EcSpireDistributedScan`, sets
  `idle_in_transaction_session_timeout = '100ms'`, verifies the
  cursor plans as the CustomScan node, then asserts the session is
  terminated per timeout. Right shape — pins both the timeout
  contract AND that we're actually testing the CustomScan path
  (the plan-shape check guards against the test accidentally
  measuring a non-SPIRE plan).
- **`733` remote restart rejoin** — uses remote backend
  self-termination during candidate receive as the restart
  simulation. Asserts (1) strict failure with the expected
  category, (2) degraded skip-and-report, (3) successful
  CustomScan execution after rejoin. Three orthogonal assertions
  on the recovery contract.
- **`734` coord DROP INDEX lock contract** — important: this test
  actually pins the PostgreSQL lock-contract behaviour, not a
  fantasy. It asserts coord-side `DROP INDEX` waits behind the
  active CustomScan relation lock and fails the DDL with
  `lock_timeout`; the in-flight scan is **not** asynchronously
  unwound by the DDL. That matches reality — Postgres DDL takes
  AccessExclusiveLock and waits.
- **`731` large text projection** explicitly cites the `31110`
  scope finding and stays within 12c's test-only boundary. Good
  cross-referencing in the request packet.
