# Review: Parallel Scan Recent Emitted TID History

**Verdict: Approved**

## What was read

- `git show 81ed936` full diff (`src/am/common/parallel.rs`, `src/am/ec_hnsw/scan.rs`)
- `EcParallelWorkerSlot` struct (parallel.rs:102)

## Summary

The change widens the foreign duplicate suppression window from a single `last_emitted_heap_tid` slot to a 32-entry ring per worker. Correct approach for the observed fast-worker skew pattern.

## Findings

**Ring cursor correctness.** `remember_recent_emitted_heap_tid` (scan.rs) writes to `index = cursor % 32` then sets `cursor = (index + 1) % 32`. This keeps cursor in [0, 31] at all times so the `u8` field cannot overflow in the ring sense. The reset path in `reset_scan_emitted_state` fills the ring with `INVALID` and sets `cursor = 0`. Clean.

**DSM field sizing.** Each `EcParallelWorkerSlot` now carries 64 additional `AtomicU32` fields (32 block + 32 offset). Given the struct is `maxalign`-padded, a 2-worker test needs approximately 1,400 bytes of backing storage. The `TestParallelScanStorage` buffer in `parallel.rs` was correctly bumped to 8,192 bytes. The `scan.rs` test helper introduced in packet 592 uses 4,096 bytes for `worker_slot_count=2`, which is sufficient.

**Publication path.** `publish_parallel_scan_worker_slot_runtime_snapshot` (parallel.rs) stores the ring alongside `last_emitted_heap_tid` via `store_parallel_item_pointer_array`. The suppression check in `foreign_worker_recently_emitted_heap_tid` (scan.rs) checks `last_emitted_heap_tid` first (fast path) then falls through to `.contains(&shared_heap_tid)` on the ring. The `contains` is O(32) but that is constant and bounded.

**DSM version bump.** 12 → 13. Correct for the layout change.

## No blockers.
