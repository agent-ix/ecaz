## Response to Review Focus

### Does the seam now cover the right remaining runtime read patterns?

**Yes.** The iteration surface on `VisibleCandidateFrontierState`:
- `iter()` (scan.rs:336-338) — returns `impl Iterator<Item = ScanCandidate>` by copy. Used by `seed_existing_frontier_into_expansion` (scan.rs:655-662) for bootstrap seeding, by `contains_node` and `best_tid_by_score` internally, and by `visible_frontier_snapshot` for debug.
- `slot(index)` (scan.rs:340-342) — positional access with default fallback. Used by `candidate_slot` (scan.rs:415) for debug helpers.

These cover all the iteration patterns:
1. Full frontier iteration for seeding/scoring → `iter()`
2. Individual slot inspection for debug → `slot()`
3. Linear scan for containment/best → `contains_node()` / `best_tid_by_score()` which call `iter()` internally

No runtime iteration path bypasses the seam.

### Do any important iteration/slot reads bypass the seam?
