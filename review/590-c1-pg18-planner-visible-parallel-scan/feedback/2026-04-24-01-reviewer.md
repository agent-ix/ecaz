## Summary

This is the activation commit. 714 lines added, all changes are interleaved correctly:

**`amcanparallel = cfg!(feature = "pg18")`** â€” enables the flag only on PG18. Serial PG17 behavior is unchanged.

**Cost model: startup cost moved to run cost** â€” `(0.0, graph_cost + graph_cpu + linear_cost + linear_cpu)`. The comment correctly explains the mechanism: `cost_index` cannot parallel-discount AM startup cost because the AM cost callback runs before `parallel_workers` is assigned. Moving to run cost lets the pathlist hook's manual discount (`discount_partial_ec_hnsw_index_path_cost`) reduce the parallel path's total cost by `1/N`, making it competitive under LIMIT queries that would otherwise select the serial path. Total work is unchanged; this is strictly a path-selection signal change.

**Cost discount in pathlist hook** â€” `discount_partial_ec_hnsw_index_path_cost` is only reachable through `ecaz_set_rel_pathlist_hook` â†’ `count_paths(..., partial=true)` â†’ which is only registered under `#[cfg(feature = "pg18")]` at `lib.rs:44`. On PG17 builds this function is unreachable. The modification of `indextotalcost` and `path.total_cost` in-place is sanctioned at pathlist hook time before Gather/Gather Merge paths are generated. The clamp `max(startup_cost)` prevents total_cost going below startup_cost after discount, which is correct.

**ORDER BY output storage** â€” `allocate_scan_orderby_output` in `ambeginscan` handles the lifetime of `xs_orderbyvals`/`xs_orderbynulls` for the executor lifetime. `reset_scan_orderby_output` in `amrescan` zeroes values and sets `xs_orderbynulls[i] = true` without freeing (the buffers are reused across rescans). `free_scan_orderby_output` in teardown frees and nulls. The pgrx::error! guard in `reset_scan_orderby_output` for missing storage is defensive for a contract violation (amrescan without ambeginscan) that cannot happen under normal PostgreSQL AM usage.

**Single emitter election** â€” `claim_parallel_scan_emitter_slot` under coordinator lock: first-write-wins on `emitter_slot_index`. The epoch check (`rescan_epoch != opaque.parallel_scan_rescan_epoch`) guards against stale callers from a previous scan epoch. `reset_parallel_scan_state` resets `emitter_slot_index = EC_PARALLEL_SLOT_INDEX_INVALID` on epoch increment, so each new rescan epoch can elect a fresh emitter. `non_emitting_parallel_backend_is_exhausted` exhausts non-emitters at amrescan time before the graph prefetch, preventing them from publishing duplicate local candidates into the shared coordinator.

**`should_release_parallel_scan_worker_slot` returns false on PG18.** Worker slots are therefore only cleaned up by `reap_parallel_scan_dead_root_slots_with_attachment`. The reaper runs at the start of every coordinator lock acquisition, so stale slots from completed workers will be reclaimed before the next scan uses the coordinator. This is the right lifecycle model for PG18's parallel executor where teardown timing differs from PG17.
