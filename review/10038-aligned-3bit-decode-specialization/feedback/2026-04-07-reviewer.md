# Review Feedback: Aligned 3-Bit Decode Specialization

**Verdict: Approved — strongest checkpoint in the batch**

## Performance

This is the standout commit: 4.7x scalar improvement and 4.2x AVX2 improvement on
`score_ip_encoded`. The aligned 24-bit load for 8 packed 3-bit indices replaces per-dimension
`read_bits_le` calls in the hot loop. For the common 4-bit quantizer (bits-1 = 3), this is exactly
the right specialization.

## Correctness

`decode_eight_3bit_aligned` loads 3 bytes into a `u32`, then shifts and masks 8 times. The
`debug_assert_eq!(dim_index % 8, 0)` catches misalignment at debug time. The unit test
`decode_eight_3bit_aligned_matches_packer` locks the helper against the existing packer — good.

One thing to verify: at the very end of the packed buffer, when `dim_index + 8 > original_dim` but
we're still in the 3-bit specialization, the code correctly falls through to the generic tail loop.
The `while dim_index + 8 <= self.original_dim` guard handles this. For dimensions that are exact
multiples of 8 (like 1536), there's no tail. For non-multiples, the generic loop picks up the
remainder. Correct.

## Experiment documentation

The discarded experiments (pointer-style FWHT rewrite, codebook_products precompute) documented
inline in the request packet are useful context. This level of detail is good for B1 review packets
— I'd keep this approach rather than a separate experiment-log file. It stays close to the decision
it explains.

## Generic fallback

Non-3-bit widths still go through `mse_index_at` per dimension. That's fine — B1 is driven by the
4-bit surface. No wasted work expanding to other widths prematurely.
