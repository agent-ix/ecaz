# Review: Task 41 DiskANN scan-state RAII (31217)

## Summary

Replaces the DiskANN scan path's `(Relation, owned)` and
`(Snapshot, owned)` tuples + manual
`release_owned_scan_heap_state` call with two scoped wrappers
in `src/am/ec_diskann/scan_state.rs`. Same `Option<Guard>`
pattern as 31209 / 31216 applied to both relation and snapshot
lifetimes.

## Confirmations

- **Two wrappers, same encoding.** Snapshot and relation each
  get an `Option<...Guard>` wrapper. Drop semantics differ
  (snapshot uses `UnregisterSnapshot`; relation uses
  `table_close`), but the type shape is identical.
- **Manual `release_owned_scan_heap_state` deleted.** Cleanup
  logic moves into the two wrappers' Drop impls.
- **Production scan path.** Every DiskANN scan that does heap
  rerank goes through this state. The migration retires the
  fan-out of error paths that each previously needed manual
  release calls.

## Findings

No blockers. Companion to 31216 for the scan side.
