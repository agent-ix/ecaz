# 06 — `expand_one` with empty neighbor function is intentional transition scaffolding

**Severity:** Observation (no action required)  
**File:** `src/am/scan.rs:658-660`

## Finding

`top_up_bootstrap_frontier` calls `expand_one(|_| std::iter::empty())`, using `expand_one` as a pop-only operation:

```rust
BootstrapExpandPolicy::ScoreOrder => bootstrap_expansion_mut(opaque)
    .expand_one(|_| std::iter::empty::<search::BeamCandidate<page::ItemPointer>>())
    .map(|candidate| candidate.node),
```

`expand_one` pops the best candidate and seeds neighbors from the callback. With an empty callback, it's functionally identical to `pop_best()`.

## Assessment

The initial review recommended adding a `BeamSearch::consume_best()` method. On re-examination, this is **intentional scaffolding**: when A3 wires in real graph search, the empty neighbor function will be replaced with the actual `load_graph_adjacency` + scoring callback, and the external `refill` call will move inside `expand_one`. Using `expand_one` here pre-shapes the call site for the transition.

Adding `consume_best()` would create a dead-end API that gets replaced immediately when A3 lands.

## Suggested shape

Add a comment if desired:
```rust
// Neighbor function is empty — actual neighbor loading happens in the refill callback.
// This call site gains a real neighbor function when graph traversal wires in (A3).
```

## Impact

None. The code is readable enough in context.
