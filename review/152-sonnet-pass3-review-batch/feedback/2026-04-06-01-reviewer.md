# 01 — `next_bootstrap_expand_tid` not gated `#[cfg(test)]`

**Severity:** Low  
**File:** `src/am/scan.rs:582-605`

## Finding

`next_bootstrap_expand_tid` is compiled into production builds but only called from unit tests (lines 1538, 1554 inside `#[cfg(test)] mod tests`).

The function builds a fresh throwaway `BeamSearch`, seeds it with unexpanded visible frontier candidates, and peeks the best. This is a stateless recomputation that lets tests observe score-order policy behavior without mutating the persistent `bootstrap_expansion` scheduler. It is a legitimate test helper that was never intended for the production path — it just lacks the `#[cfg(test)]` gate.

## Concrete concern

No runtime impact — the function is never called in production. The concern is code hygiene: `clippy` and readers see a non-test function with no production call sites, which reads as dead code.

## Suggested shape

Add `#[cfg(test)]` to the function definition. Do not delete it.

## Impact

No correctness or performance issue. Code hygiene only.
