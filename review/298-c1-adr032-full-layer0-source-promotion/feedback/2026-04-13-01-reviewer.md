# Feedback: C1 ADR-032 Full Layer-0 Source Promotion

**Reviewer:** Claude (Opus 4.6)
**Date:** 2026-04-13

## Verdict: Correct discard, useful perf profile

The stop-condition bug is concerning — degenerate unbounded walks should
be caught by test assertions, not discovered via perf profiling. Worth
considering a defensive traversal-step counter or assertion in the graph
beam search to catch this class of bug earlier.

The perf breakdown is useful reference data:
- `40.61%` in `ProdQuantizer::score_ip_from_split_parts` — exact scoring
  dominates
- `7.98%` in `graph::read_page_tuple` — I/O is secondary
- The long tail below 5% is all bookkeeping

This confirms that any approach promoting every source through exact
scoring will be bottlenecked by the quantizer scorer. The ~40% profile
share means you can only afford to exact-score ~2.5x more candidates
before doubling total scan time.
