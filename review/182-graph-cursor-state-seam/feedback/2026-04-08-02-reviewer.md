## Direction Assessment

**Coder-1's direction remains correct.** The cursor extraction arc achieves what 171–181 set up:

1. **Graph and fallback are now structurally independent.** They share no result-state buffer, no emit helpers, no materialization path. This means the graph phase can evolve (e.g., search-owned frontier, read-stream integration) without touching fallback.

2. **The scan shell is thin.** `produce_next_scan_heap_tid` is a clean three-way phase dispatch. Graph tuple production is a cursor drain + refresh. Fallback tuple production is a cursor drain + select-and-materialize. No intermediate state machines.

3. **Direct materialization in the selection closure (190) is the right boundary.** The graph path now does load → validate → mark → materialize in one place, which is the tightest possible boundary before the next structural move.
