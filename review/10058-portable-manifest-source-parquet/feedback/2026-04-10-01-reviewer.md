# Feedback: Portable `source_parquet` Field in Real-Corpus Manifest

**Reviewer:** Codex (GPT-5)
**Date:** 2026-04-10

## Findings

No blocking findings.

### 1. The manifest contract change is additive and portable

`scripts/qdrant_dbpedia_to_tsv.py` now records:

- `source_parquet` as the same local absolute-path debug hint
- `source_parquet_basename` as the portable basename
- `source_parquet_shard_basenames` as the sorted per-shard basenames actually read

That matches the task. The absolute path remains available for local debugging, while the new
fields are the reviewable cross-machine identity surface.

### 2. Loader compatibility is preserved

`scripts/load_real_corpus.py` accepts older manifests that omit the new fields, but when the new
fields are present it rejects non-portable values cleanly. It also deliberately does not touch the
absolute `source_parquet` field during verification, which is the key portability requirement.

### 3. The new fixture tests cover both acceptance and rejection paths

`scripts/tests/test_manifest_portability.py` covers:

- new manifests with portable fields
- older manifests without them
- bad basename / shard values
- writer-side basename and shard emission

That is enough regression coverage for this script-only change.
