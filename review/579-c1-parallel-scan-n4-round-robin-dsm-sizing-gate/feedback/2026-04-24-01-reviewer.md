# Feedback: N4 Round-Robin DSM Sizing Gate

**Head reviewed:** `a2a86c9` (landed)

## TL;DR

Debug DSM allocation now derives the descriptor size from the requested
worker count rather than the GUC-default capacity, unblocking `n>=4`
harnesses. n=3 assertions are factored into a shared helper and an n=4
PG18 gate is added.

## Verification

- `ec_parallel_scan_descriptor_size_for` is now crate-visible
  (parallel.rs:548). Callers outside this file: only
  `debug_allocate_parallel_scan` in `scan_debug.rs` (ripgrep confirms).
- `debug_allocate_parallel_scan` replaces the fixed-size helper with the
  per-worker one (scan_debug.rs:529+ in diff).
- `assert_parallel_round_robin_many_matches_serial` factors the five
  invariants (combined==serial, workers.len, all contribute, no
  hidden/blocker/active stranded, no local/deferred local emits) into one
  helper parametric on worker count + fixture label (lib.rs).
- `test_ech_parallel_n4_round_robin_matches_serial_scores` uses 10-row
  fixture and calls the shared helper with `worker_slot_count = 4`.

## Findings

None. Exposing `ec_parallel_scan_descriptor_size_for` to the crate is the
narrow seam needed — the single debug caller is the only internal user and
it's already a debug-only code path. Hiding it behind a debug wrapper would
only add indirection.
