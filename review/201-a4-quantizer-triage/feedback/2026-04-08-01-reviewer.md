### Should the next A4 slice move into the quantizer formulation?

**Yes, but with a structured-data checkpoint first.** Before investing in quantizer redesign, test the
current quantizer on structured data. The current test corpus (uniform random unit vectors at d=1536)
is adversarial for any ANN method — all pairwise distances concentrate, making ranking precision
impossible regardless of representation quality.

## Why Uniform Random Is Adversarial

For random unit vectors in d=1536:
- All pairwise IPs ~ N(0, 1/1536), stddev ≈ 0.026
- The gap between the 10th and 100th nearest neighbor's IP is O(1/sqrt(N*d)) — vanishingly small
- Even perfect 128-centroid scalar quantization adds enough noise to shuffle these indistinguishable
  rankings

This explains why 4-bit → 8-bit barely helps: the improvement in per-coordinate MSE doesn't translate
to ranking improvement because the ranking signal is smaller than the noise floor of ANY fixed-rate
scalar quantizer at this dimension.

Real-world data (LLM embeddings, image features) has intrinsic dimensionality ~50-200 even at ambient
d=1536. Neighbors are much more separable. The same quantizer may perform dramatically better.

## Recommended Structured Test Data

To properly evaluate the quantizer ceiling, test on data with realistic structure. In order of
preference:

### 1. Real embedding corpora (best signal)

- **OpenAI/Cohere/Voyage embeddings at d=1536**: These match the target dimension exactly. Generate
  10k embeddings from diverse text (Wikipedia paragraphs, StackOverflow posts, news articles). The
  distribution will have clusters, varying norms, and meaningful nearest-neighbor structure.
- **MTEB benchmark datasets**: Standard retrieval benchmarks (NQ, MS MARCO, BEIR) provide pre-computed
  embeddings with ground-truth relevance. Many have d=768 or d=1024 but some go to d=1536.

### 2. Standard ANN benchmarks (good baselines, wrong dimensionality)

- **GloVe-200**: 1.2M vectors at d=200. Well-studied recall baselines exist for all major ANN methods.
  Useful for validating the quantizer against known results, but d=200 is not the target regime.
- **Deep-1M**: 1M vectors at d=96, derived from deep learning features. Clustered, realistic.

### 3. Synthetic structured data (cheapest, still informative)

- **Gaussian mixture model**: Generate K=50-200 clusters in d=1536, each with isotropic Gaussian noise
  at stddev σ=0.1-0.3. This creates realistic neighbor structure while controlling the difficulty.
  The existing `random_clustered_corpus` helper could work if it generates tight enough clusters.
- **Low-rank + noise**: Generate data as `X = U @ V + ε` where U is N×r, V is r×d, r=50-200. This
  mimics the intrinsic-dimensionality structure of real embeddings.
- **Spherical cap queries**: Generate corpus uniformly, but construct queries as perturbations of
  specific corpus points. This guarantees each query has identifiable true neighbors, unlike uniform
  random where "nearest" is barely distinguishable from "average."

### 4. What to measure

For each corpus type, run the same exact-only recall harness from review 201:
- Bitwidth sweep (2,3,4,5,6,7,8) to verify that more bits actually help on structured data
- Scorer ablation (exact, gamma_zero, code_proxy, decoded) to check if gamma becomes relevant
- Near-duplicate preservation at various angles
- Compare against a simple baseline: raw float dot product on PCA-reduced (e.g., d→128) vectors

If exact-only recall jumps to 80%+ on structured data at 4-bit, the quantizer design is sound and the
A4 gate just needs a realistic corpus. If it stays at ~50%, the quantizer formulation itself needs
work.

## On the Quantizer Architecture

The current architecture (SRHT rotation → scalar Lloyd-Max per coordinate → QJL residual with
data-free codebook) is elegant and has zero training cost. Before considering a move to subspace
PQ/OPQ with learned codebooks, there are cheaper investigations:

1. **Check the effective number of bits**: The codebook uses `bits-1` for MSE centroids (1 bit goes to
   QJL sign). At 4-bit, that's only 8 MSE centroids. At 8-bit, it's 128. Verify whether reallocating
   the QJL bit to MSE (since gamma/QJL is inert) improves recall.

2. **Codebook range vs data range**: The Lloyd-Max codebook is optimal for the theoretical Beta
   distribution. Verify that the actual SRHT-rotated coordinates match this distribution closely.
   Any mismatch means the codebook is suboptimal.

3. **SRHT quality at d=1536**: The SRHT should make coordinates approximately i.i.d. Verify this
   empirically — if some coordinates carry more variance than others after rotation, the uniform
   codebook wastes bits on low-variance coordinates.

4. **Reconstruction error audit**: Compute the actual MSE between original vectors and
   decode_approximate vectors. Compare against the theoretical minimum for scalar quantization at this
   rate. If there's a gap, there may be an encoding/decoding bug worth finding before an architectural
   change.

## Disposition

Review 201's evidence is strong and the triage conclusion is correct. The next A4 slice should:
1. Run structured-data checkpoint (cheapest, highest-signal next step)
2. Run the quantizer-internal checks above (cheap, may reveal a fixable defect)
3. Only then consider architectural changes to the quantizer
