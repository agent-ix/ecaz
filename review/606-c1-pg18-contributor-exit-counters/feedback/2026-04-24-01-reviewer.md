## What This Does

Adds two new DSM counters to distinguish *why* non-emitting parallel contributors exit:
- `output_limit_exits` — hit `PARALLEL_CONTRIBUTOR_MAX_OUTPUTS`
- `poll_limit_exits` — hit `PARALLEL_CONTRIBUTOR_DRAIN_POLL_LIMIT`

Previously both exit paths collapsed into a single `||` branch with no attribution. The diagnostic log confirms the diagnosis: all four non-emitting contributors are hitting the poll limit (4 poll-limit exits, 0 output-limit exits), not the output budget.

### scan.rs

**The split of the combined `||` exit is a behavior improvement, not just instrumentation.** Previously:

```rust
if staged_outputs >= PARALLEL_CONTRIBUTOR_MAX_OUTPUTS
    || drain_polls >= PARALLEL_CONTRIBUTOR_DRAIN_POLL_LIMIT
{
    return false;
}
```

Now split into two sequential `if` blocks. This means if both conditions are simultaneously true, `output_limit_exits` fires first (it's checked first), and `poll_limit_exits` does not double-count. This is correct — the semantics are: whichever limit was reached, that's the exit reason. No functional regression.
