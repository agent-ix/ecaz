## P2 — please reconcile before Phase 11 exit

1. **TLS / credential lifecycle scope.** Pre-AWS gate item 7 currently
   requires "TLS mode, certificate validation, `conninfo_secret_name`
   sourcing, secret rotation, auth failure, sanitized audit logging, and
   raw-conninfo non-exposure" all to pass before AWS. This is broader
   than what packet 30694 ultimately recommended (TLS deferred to a
   post-Phase-11 packet, with PostgreSQL/libpq `sslmode` left as the
   operator-owned mechanism and only strict-mode behavior on cert
   verification failure as the hard contract). Recommend softening
   gate item 7 to the narrow contract:
   - `sslmode` is preserved through `conninfo_secret_name` resolution
     and not stripped or downgraded;
   - strict-mode coordinator rejects connections that fail libpq cert
     verification with a sanitized error; degraded mode reports the
     remote as skipped;
   - `conninfo_secret_name` indirection and raw-conninfo non-exposure
     remain enforced (already in ADR-058).
   Credential rotation, audit-log shape, and a TLS runbook can move to
   a future post-Phase-11 packet without blocking the local
   production-readiness bundle. This keeps Phase 11 testable without
   pulling in a credential-store can of worms.

## P3 — informational

2. **Numeric capacity targets.** Gate item 8 says local targets must be
   stated; the runbook should commit to specific numbers (e.g. max 4
   remotes, max 16 concurrent coordinator queries, max 100k PIDs/node)
   even if those are revised after AWS evidence. Loose phrasing now
   makes the gate hard to falsify.

3. **Quantizer fingerprint shape.** Gate item 3 requires a
   "quantizer/index fingerprint" but does not pin a representation.
   Recommend stating the wire shape (e.g. RaBitQ profile id +
   training-stat hash + code length, or a single 16-byte digest) when
   the remote endpoint contract is drafted in Phase 11.3.
