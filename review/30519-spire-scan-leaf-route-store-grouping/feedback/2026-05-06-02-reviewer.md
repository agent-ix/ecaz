# Follow-up: SPIRE Scan Leaf Route Store Grouping (30519)

Re-review at HEAD `d79f41b5` after `0ca3b1db Update SPIRE phase 4 review
response` and `e582739a Address SPIRE phase 4 review feedback`.

All three items from `2026-05-05-01-reviewer.md` are addressed.

## 1. Grouping is now load-bearing — addressed

`prefetch_store_object_read_group` (scan/snapshot.rs:302) issues
`object_store.prefetch_object(...)` calls grouped by store before the
synchronous read+score loop. The grouping is now the seam through which
multi-store prefetch flows — exactly the missing consumer the previous
review flagged. Good.

## 2. Duplicate group struct collapsed — addressed

`SpireStoreLeafRouteGroup` is gone; only `SpireStoreObjectReadGroup`
remains (scan/types.rs:114). The wrapper helper that used to remap one
into the other is also removed. Clean.

## 3. Silent delta-route drop now observable — addressed

`group_leaf_and_delta_reads_by_local_store` (scan/snapshot.rs:347–353)
now emits `observer.dropped_unselected_delta_route(epoch, placement)` on
the unselected-parent filter path. A regression that orphans a delta
will be visible in scan diagnostics rather than silently dropping
candidates.

The output-order comment ("store-keyed, not phase-keyed") at line 330
is also a nice touch.

## New observation — duplicate `require_lookup` per route

`require_lookup` is now called twice per route on every routed scan: once
in `group_leaf_and_delta_reads_by_local_store` to bucket by
`(node_id, local_store_id)`, and again in `prefetch_store_object_read_group`
to recover the same placement for `prefetch_object`. Two binary searches
into the placement directory per leaf and per delta, every scan.

Cheap fix: have the group carry the `placement` pointer (or the lookup
result) it already resolved during grouping, so prefetch consumes that
instead of re-resolving. Slightly bigger group struct, but it eliminates
a per-route map probe on the hot scan path.

Not blocking — file-it-and-move-on. Worth picking up when the parallel
fetch surface lands, since that path will want batched placement
metadata anyway.
