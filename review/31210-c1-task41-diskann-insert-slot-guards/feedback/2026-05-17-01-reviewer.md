# Review: Task 41 DiskANN insert slot guards (31210)

## Summary

Migrates two DiskANN insert-planning heap tuple slots in
`src/am/ec_diskann/routine.rs` from manual
`MakeSingleTupleTableSlot` / `ExecDropSingleTupleTableSlot`
pairs to `TupleTableSlotGuard::single_for_heap`. Baseline
neutral (4,140 → 4,140).

**First production hot-path migration of `TupleTableSlotGuard`** —
prior slot-guard work was all in pg_test or planner paths. The
DiskANN insert path is one of the AM's actual write paths.

## Confirmations

- **Both slots use `single_for_heap`.** The duplicate-probe
  slot and the unique-insert forward-neighbor planning slot
  both go through the shared constructor. Drop via
  `ExecDropSingleTupleTableSlot` shared. ✔
- **Slot lifetimes cover all uses.** Guards alive across
  `fetch_heap_source_vector` and
  `exact_heap_rerank_distance` calls. ✔
- **Multiple exit paths now use Drop.** Previously each of the
  normal path, exact-rerank error path, and early
  duplicate-return path needed a manual
  `ExecDropSingleTupleTableSlot`. All three now unwind through
  the guard's Drop. Strictly safer — three places where a
  forgotten close would have leaked the slot.
- **Production path.** This is in `ec_diskann_aminsert`
  (`#[pg_extern]`-equivalent AM callback) — not a debug helper.
  Important validation that the shared slot guard works in real
  AM execution.
