# Feedback: ADR-032 Cross-Packet Synthesis and Investigative Paths Forward

**Reviewer:** Claude (Opus 4.6)
**Date:** 2026-04-13
**Scope:** Packets 291-306, ADR-032 branch as a whole

## The 0.8080 recall ceiling is structural

The low-ef recall number 0.8080 has now appeared across six independent experiments
(297, 299, 303, 304) regardless of:

- exact-score budget: 2 calls (297) vs 41 calls (303) vs 60 calls (302)
- pruning policy: on (297) vs off (304)
- spending strategy: local per-source (302), global frontier (303), head-window (300)

Meanwhile, ADR-031 T1 at ef=40 on the same 1000-query corpus measured 0.8397-0.8428
(packets 283, 287). The ADR-032 regression is 3.2-3.5 percentage points, real and
consistent.

The evidence from 303+304+305 conclusively shows the ceiling is not about exact-scoring
policy. It is about the **search trajectory** — at low ef, the approximate-scored frontier
expands sources into different graph neighborhoods than an exact-scored frontier would.
No post-hoc scoring, reranking, or ordering policy within the discovered candidate set
can recover candidates that were never discovered.

## What the follow-up packets (298-306) have in common

Every attempt from 298-306 operates on the same side of the problem: adjusting what
happens **after** candidates enter the frontier. None of them change **which sources get
expanded** — the mechanism that determines the graph neighborhoods the search visits.

This is why they all hit the same ceiling or make things worse.

## Recommended investigative paths forward

### Path 1: Exact-scored seed anchoring (highest priority)

The approximate search diverges from the exact search because the first few source
expansions go in the wrong direction. If the initial seed candidates (from upper-layer
greedy descent + first bootstrap) are exact-scored before entering the priority queue,
the early expansion trajectory anchors to the correct graph region. Later expansions
can still use approximate scores cheaply.

Cost: only ~3-5 exact scores (the seeds). This is the minimum exact-score budget that
could shift the trajectory rather than rerank the output.

This is different from 299 (bounded promotion budget) because 299 spent exact scores
on arbitrary expansion candidates. Seed anchoring spends them on the candidates that
have the most leverage over the search direction.

### Path 2: Periodic exact-score trajectory correction

Instead of exact-scoring at frontier head (297) or per-source (298), exact-score every
Nth expanded source (e.g., every 4th or 8th). Use the exact score to decide whether to
continue expanding in the current graph region or backtrack to a different frontier
candidate.

This is different from all 298-306 attempts because it affects which nodes are expanded
next, not how discovered candidates are ordered. The exact scores function as trajectory
correction, not reranking.

### Path 3: Dual-frontier with exact trajectory correction

Run the approximate-scored frontier as the primary expansion driver (preserving ADR-032
speed). Maintain a small secondary "exact frontier" of size ~k (e.g., 10) tracking the
best exact-scored candidates seen so far. When the approximate frontier's head diverges
from the exact frontier (e.g., the exact frontier's worst score is better than the
approximate head's exact score), force one exact-scored expansion from the exact
frontier's best candidate.

This uses the exact frontier as a trajectory correction mechanism rather than a
reranking mechanism.

### Path 4: ef-adaptive operating point (simple, likely sufficient for production)

Accept the 0.8080 ceiling at ef=40. Measure ef=48, 56, 64 to find the exact crossover
point where ADR-032 recall matches or exceeds ADR-031 T1's ef=40 recall of ~0.84.

If ADR-032 at ef=64 gives recall >= 0.84 at latency <= 1.5ms, the practical
recommendation is: "when upgrading to ADR-032, increase ef_search from 40 to 64."
This is the lowest-risk path to shipping.

This does not require any code changes — just measurement. It should be run regardless
of whether the other paths are pursued.

### Path 5: Binary-sign quality improvement (longer term, ADR-031 domain)

The rank correlation between binary sign scores and exact scores is ~0.63 (packet 283
spearman). If that correlation improved to ~0.8, the approximate frontier would track
the exact frontier much more closely and the 0.8080 ceiling would lift naturally.

This is an ADR-031 improvement (better binary encoding, more bits, or a different
approximate scoring function) rather than an ADR-032 change, but it directly addresses
the root cause of the trajectory divergence.

## What NOT to pursue further

The following strategies are conclusively ruled out by the experimental evidence:

- More exact-scoring budget variants (302, 303 killed this — 30x more work, no gain)
- Pruning relaxation (304 killed this — recall unchanged)
- Post-search reranking (305 killed this — both axes regressed)
- Score calibration / affine transforms (301 killed this — recall worsened)
- Frontier-internal ordering policies (300, 306 — likely same ceiling)

Any approach operating within the discovered candidate set cannot break through 0.8080.
The next productive direction must change which candidates are discovered — i.e., the
expansion trajectory.
