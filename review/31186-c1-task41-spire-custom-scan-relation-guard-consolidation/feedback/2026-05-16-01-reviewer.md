# Review: Task 41 SPIRE custom scan relation guard consolidation (31186)

## Summary

Removes two module-local relation wrappers from the SPIRE
custom-scan path:

- `OpenIndexRelation` in `src/am/ec_spire/custom_scan/explain.rs`
- `OpenTableRelation` in `src/am/ec_spire/custom_scan/planner.rs`

Call sites now use `IndexRelationGuard::try_access_share` for
SPIRE index relation opens and `HeapRelationGuard::try_access_share`
for placement table opens. Closes the 31180 finding items 3 and 4.

## Confirmations

- **Module-local structs deleted.** `grep -rE 'struct OpenIndexRelation|struct OpenTableRelation' src/`
  returns nothing. ✔
- **`try_access_share` chosen correctly.** Both prior structs
  had Option-returning `open()` semantics (skip the operation
  on null relation), so `try_access_share` is the right match —
  not `access_share` (which panics) and not the variable-lockmode
  `try_open` (these are AccessShareLock-only).
- **Behavior preserved.** The `let Some(...) else { ... }`
  pattern preserves the prior null-check-and-skip behavior in
  both explain and planner paths.

No blockers.
