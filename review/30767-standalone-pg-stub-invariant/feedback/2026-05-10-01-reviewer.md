# 30767 — standalone PostgreSQL stub invariant review

Validation pass on `a388370f` (`Document standalone PG stub
invariant`). Closes packet 30764 P3 #1.

## What I checked

- `csrc/standalone_pg_backend_stubs.c` now documents the
  classification rule for newly-required PostgreSQL
  symbols:
  - **inert helper stubs** for symbols that pure-Rust tests'
    transitive dependencies reach for during initialization
    (memory contexts, type metadata, transaction id helpers,
    binary coercion);
  - **panicking backend-execution stubs** for symbols whose
    fake implementations could let pure-Rust tests silently
    exercise database paths the test environment cannot
    actually validate (SPI, heap, catalog, executor).
- The wording is strict: "Anything that would fake SPI,
  heap, catalog, or executor behavior belongs in the
  pgrx/pg_test lane instead." That's exactly the invariant
  I asked for in 30764 P3 #1.
- No behavior changes — docs-only.

## Answers to the packet's review questions

- **Does the invariant make the standalone-vs-backend test
  boundary explicit enough for future symbol additions?**
  Yes. A future contributor who hits a missing symbol now
  has explicit guidance: classify deliberately, don't fake.
  The default reaction ("just stub it to return something")
  is now contradicted by file-local documentation, which is
  the right level of friction for a discipline that
  affects test-validity correctness.
- **Wording strict enough to prevent fake backend behavior
  in direct cargo tests?** Yes. Naming SPI / heap / catalog
  / executor explicitly covers the four most likely classes
  of "I'll just stub this to return data" temptation. A
  future contributor adding a stub that returns fake data
  from any of these surfaces is directly contradicting the
  documented invariant.

## Status

Closed. Standalone test stub invariant is now self-
documenting. Future Stage E / F / G packets that add
symbols can classify deliberately rather than discovering
the rule by reverse engineering the existing stubs.
