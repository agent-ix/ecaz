# Feedback: 225-a4-cheaper-external-gate

## Summary

Clean fix, correct result, good operational improvement. The 50-query real 50k
gate at 94.4% is the strongest A4 number recorded on real data so far and the
3-minute runtime confirms the harness cost problem is solved for the gate path.

## What Looks Right

### Exact-quantized exclusion from gate path

The split is correctly scoped. The gate report only needs graph Recall@10 vs
fp32 truth — it has no use for exact-quantized top-10. Removing it from
`run_graph_scan_recall_gate_from_external` (via the `include_exact_quantized_top10`
flag) is the minimal, correct change. The summary path still pays the full cost
because it needs the exact-vs-graph metrics.

The 22-minute → 3-minute reduction confirms the exact-quantized SPI loop was
the dominant cost at 50 queries, not the fp32 truth computation itself.

### Detached capture via server-side COPY

Using `COPY (<query>) TO '<file>'` on the server side is the right fix for the
session-loss problem. The result lands in a file on the server regardless of
client connectivity. The timestamped output file path makes the run artifacts
auditable.

### The numbers

```
8   40   0.926       t
8   128  0.944  0.89 t
8   200  0.948       t
16  200  0.952       t
```

All four configs pass. The gate point `(m=8, ef_search=128)` at `94.4%` is
comfortably above the `89%` threshold and materially above the earlier
`10`/`25`-query directional reads (`90%`, `93.6%`). The curve is also shaped
correctly — `ef_search=40` is lower but still passes, `ef_search=200` adds
a small margin, `m=16` adds another small step. No pathological behavior.
