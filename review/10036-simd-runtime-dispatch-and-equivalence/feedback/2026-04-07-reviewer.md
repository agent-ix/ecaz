# Review Feedback: SIMD Runtime Dispatch And Equivalence

**Verdict: Approved with minor notes**

## Dispatch design

The `OnceLock`-backed detection with `TQVECTOR_SIMD` env override is clean and the right shape for
this stage. `SimdBackend` enum variants gated behind `#[cfg(target_arch)]` means you get
compile-time proof that no AVX2 codepath ships on aarch64 and vice versa. Good.

Minor: `forced_backend_from_env` panics on unrecognized values. That's fine for a bench/dev knob
right now, but if this ever becomes a production GUC you'll want to degrade to auto rather than
crashing the backend. Worth noting in a follow-up comment but not blocking.

## Scalar preservation

Keeping the scalar implementations side-by-side rather than behind `cfg(not(...))` is the right call
for the 1000-input equivalence coverage. The test in `dispatched_score_matches_scalar_on_random_inputs`
exercises the full surface across multiple bit widths — this is solid.

## QJL sign LUT

The compile-time `build_qjl_sign_lut` replacing per-bit extraction is a nice improvement. 256x8
f32 entries is only 8 KiB, well within L1, and the lookup is branchless.

## Scope boundary

Dispatch stays within `quant/` and the public quantizer API surface is unchanged — correct for B1.
No scan, planner, or graph-search files are touched. Clean checkpoint boundary.

## No blocking issues
