## Findings

### 1. Two-pass retry is sufficient for single-writer semantics

**Severity:** Informational (positive)

With `amcanparallel = false`, the only writer is the single leader
refreshing selection. One stale read → refresh → retry will find the
live selection or correctly report `None`. No unbounded retry loop.
Document this assumption at the function so the next reader doesn't
widen the loop speculatively.

### 2. Validate-then-trust vs always-rescan

**Severity:** Informational (positive)

Validating the coordinator-named slot (rather than always rescanning
all slots) is the right scaling choice. The validation predicates are
cheap (three atomic loads + a tid check), and the refresh fallback
guarantees correctness when the cache is stale.
