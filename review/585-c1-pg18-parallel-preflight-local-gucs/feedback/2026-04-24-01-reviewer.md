# Review: PG18 Parallel Preflight Local GUCs

**Verdict: Approved**

## What was read

- `git show 663ad81` full diff (ecaz-cli `test.rs`, `install.rs`, `scratch.rs`, `support.rs`)

## Summary

Clean and correct. The seqscan control query is now wrapped in a transaction with `SET LOCAL` for each planner GUC, so the candidate ordered-path session state is automatically restored after the control query completes. The function signature changes from `client` to `mut client` to support `client.transaction()`. Clippy fixes (`&PathBuf` → `&Path`, redundant `Ok(?)` unwrap) are small and correct.

## Findings

**Transaction boundary.** The transaction wraps all four `SET LOCAL` calls and the `EXPLAIN ANALYZE` query, then commits. Since `SET LOCAL` reverts on transaction end, the subsequent candidate-path query runs with the restored `enable_*` state. Correct.

**`commit().await?`** is called after the rows are collected. If the rows collection fails, `transaction` would drop without committing, effectively rolling back. This is appropriate — the control query's diagnostic value is in the EXPLAIN output, not in any persistent state, so an abandoned transaction is harmless.

**No behavior change** to the control plan output or `--expect-parallel` assertion. The artifact log confirms same results.

## No blockers.
