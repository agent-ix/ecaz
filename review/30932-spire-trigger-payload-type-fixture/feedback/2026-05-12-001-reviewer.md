# Review: SPIRE Trigger Payload Type Fixture (30932)

## Summary

Closes 30896 H7. Comprehensive type round-trip coverage for the
existing JSON trigger payload bridge:

- `numeric(12,4)` precision preserved.
- `timestamptz` value preserved + normalized to UTC.
- nested `json` and `jsonb` round-trip.
- text containing quotes, backslash, newline (JSON-special
  characters).
- domain-over-text column.
- SQL NULL on a nullable column.
- **NOT NULL violation path** that fails closed before a
  prepared transaction is left behind (load-bearing safety
  property).
- DEFAULT-valued column after PG materializes `NEW`.

Bonus: improves coordinator INSERT remote-SQL error formatting
so remote PG message/detail/hint fields are preserved instead
of collapsing to "db error". Operator-facing improvement.

The three review-focus questions:
- Fixture covers Phase 12.5 trigger payload classes without
  overclaiming typed transport or schema-drift fingerprint: yes.
- Preserving remote `postgres::Error` message/detail/hint in
  the wrapper is acceptable operator improvement: yes — was
  effectively losing diagnostic information before.
- NOT NULL path proves no remote row + no SPIRE prepared xact
  left after failed dispatch: yes — load-bearing safety, the
  abort callback rolls back the prepared remote.

## Findings

### P2 — H7's NULL/DEFAULT additions I called out in 30896

H7 originally listed: numeric, timestamptz, json/jsonb,
text-with-edge-chars, domain. After my 30896 addendum, NULL +
DEFAULT cases were added. This fixture covers all of them
including the addendum cases. Closes the H7 closeout cleanly.

### P3 — JSON bridge type coverage parity with typed transport

The same type classes that work through the existing JSON
trigger payload should work through the Phase 12.2 typed
transport once the receive-switch slice lands. Worth pinning
in the typed transport design that this fixture is the parity
target — the typed receive should round-trip the same set
byte-for-byte.

### P3 — Domain-over-text column

Same shape as the 30917 typed-endpoint domain fixture — domain
metadata preserved in the trigger payload, base type
(`text`) used for value bytes. Consistent semantics across
endpoint and trigger paths.

### P3 — Improved error formatting

Pulling remote message/detail/hint out of `postgres::Error`
into the wrapper's error message means an operator hitting a
remote-side constraint violation sees the actual remote SQL
error, not just "db error". Real operator-experience win.

## Reviewer recommendation

Accept. Closes H7. Combined with 30911 (operator compat docs)
and 30924 (2PC latency docs), trigger-side payload behavior is
now thoroughly documented + tested.
