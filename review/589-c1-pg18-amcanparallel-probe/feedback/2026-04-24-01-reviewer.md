# Review: PG18 amcanparallel Local Probe

**Verdict: Approved**

## Summary

No committed code to review. This is a diagnostic observation captured from a temporary local one-line change (`amroutine.amcanparallel = cfg!(feature = "pg18")`) that was not committed.

The artifact confirms: with `amcanparallel=true`, PostgreSQL generates a partial `ec_hnsw` index path with `parallel_workers=4 parallel_aware=true pathkeys=1`. The final plan still selects the serial `Index Scan` because the partial path carries the same startup-heavy cost as the serial path, leaving the parallel path unable to compete at small LIMIT sizes. This directly motivated the cost model change in packet 590.

The restored normal build was verified (`amcanparallel_seen=false` post-restore). Clean diagnostic hygiene.
