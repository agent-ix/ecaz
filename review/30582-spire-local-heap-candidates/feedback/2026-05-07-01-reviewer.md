# Local heap candidates review

`ec_spire_remote_search_local_heap_candidates(...)` and the matching
summary attach decoded heap block/offset to ranked candidates.
Executable bridge to the future heap-fetch step. No blocking
findings.

## Confirmations

1. Returns the full ranked candidate set with score and
   assignment_flags preserved alongside the decoded heap_block /
   heap_offset, so a future heap-fetch path has everything it
   needs to fetch and rerank in one row contract.
2. Fail-closed for remote-target plans: when
   `gate.next_blocker != "none"` the summary reports zero local
   heap candidates and propagates the upstream blocker.
3. Degraded local plans (degraded_skipped placement plus local
   executable work) correctly surface as `degraded_ready` after
   30583's status-preservation fix, not silently
   reclassified to `ready`.

## Findings

- **Same pipeline-redundancy concern.** The summary internally
  calls both `remote_search_heap_resolution_summary_row` AND
  `remote_search_local_heap_candidate_rows`, each of which
  re-runs the full coordinator-local pipeline. Cross-cutting on
  30584.
