# Review Feedback: SIMD Bench Warmup Pass

**Verdict: Approved**

## Warmup shape

256 warmup iterations covering the full 256-element scorer pool is the right choice. The first-run
drift dropped from 16% to 1.5% for the encoded scorer, which is now within noise of the rerun.
This means future review packets can trust the first 20K-iteration sample without needing a "discard
and rerun" caveat.

## Rejected 100-iteration variant

Documented clearly. 100 iterations only covers 100/256 of the scorer pool, leaving ~60% of payloads
cold for the first timed pass. The remaining 8-9% first-run drift at 100 warmup confirms this.

## Implementation

`time_loop` now always runs `iterations.min(WARMUP_ITERATIONS)` warmup iterations before starting
the `Instant::now()` timer. The `clamp(1, WARMUP_ITERATIONS)` in the print statement matches
the `.min(WARMUP_ITERATIONS)` logic. Minor: the print uses `clamp(1, ...)` while the actual warmup
uses `.min(...)` — if `iterations` is 0 these disagree (print says 1, warmup runs 0). Not a
practical issue since iteration counts are always >= 1 from the CLI, but could align them both to
`.clamp(1, WARMUP_ITERATIONS)` for consistency.

## FWHT warmup

The warmup also covers FWHT benches. FWHT doesn't have a pool to warm, but the warmup still primes
instruction caches and branch predictors, which helps the first-iteration measurement. 256 iterations
of FWHT at 2048 dims is only ~0.3 ms — negligible overhead.
