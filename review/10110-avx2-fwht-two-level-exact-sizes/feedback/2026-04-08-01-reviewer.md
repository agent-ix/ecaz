# Feedback: AVX2 FWHT Two-Level Exact-Size Schedules

Request:
- `review/10110-avx2-fwht-two-level-exact-sizes/request.md`

**Reviewer:** Claude (Opus)
**Date:** 2026-04-08

## Response to Review Focus

### Is the two-level exact-size specialization a reasonable complexity/perf tradeoff?

**Yes.** The `fwht_in_place_avx2_two_level` helper (`hadamard.rs:79-100`) is clean: inner tiles get full FWHT (bootstrap + inner stages), those are combined inside outer tiles, then a final full-array stage finishes the transform. The code is only 20 lines and reuses the existing bootstrap and stages helpers. The 2.4% improvement at both 2048 and 4096 is consistent and meaningful at this stage of optimization.

The dispatch in `fwht_in_place_avx2` (`hadamard.rs:52-61`) is now three paths: exact-size two-level for 2048/4096, generic tiled for other large sizes, and direct bootstrap+stages for small sizes. This is a reasonable specialization tree — the two ADR-020 query-prep sizes (2048 for both 1536-dim and 2048-dim vectors) get the most attention.

### Is flat 1024 acceptable given the 2048/4096 wins?

**Yes.** 1024 moved from 521.7 to 523.6 ns — within noise. The 1024-dim path already uses the generic tiled schedule, and review 10109 showed that tighter tiles didn't produce stable wins there. No reason to over-specialize a path that's already fast.

### Does the packet separate this from prior work clearly?

**Yes.** Clear cross-references to 10107 (discarded recursive), 10108 (generic tiled), 10109 (tile-width tuning). The progression from generic tiling to exact-size two-level scheduling is well-documented.

## Additional Findings

1. **Debug assertions in `fwht_in_place_avx2_two_level`** (`hadamard.rs:84-89`) are thorough: power-of-two checks, minimum size, divisibility constraints. Good defensive code for a hot path that's only entered via hard-coded constants today.

2. **Overall SIMD speedup summary across the 10105-10110 arc:**
   - `fwht/1024`: ~4.3x scalar (523 ns vs 2267 ns)
   - `fwht/2048`: ~4.2x scalar (1085 ns vs 4559 ns)
   - `fwht/4096`: ~3.7x scalar (2541 ns vs 9473 ns)
   These are solid numbers for an AVX2-only path without AVX-512.

## Verdict

No blocking issues. The two-level schedule is a clean capstone for the tiled FWHT arc. The FWHT SIMD work appears ready to stabilize — further gains would likely require AVX-512 or a fundamentally different algorithm.
