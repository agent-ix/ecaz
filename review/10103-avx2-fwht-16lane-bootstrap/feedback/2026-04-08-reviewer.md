# Review Feedback: AVX2 FWHT 16-Lane Bootstrap

**Verdict: Approved**

## Compositional design

`fwht16_avx2_block` calls `fwht8_avx2_block` on both halves, then applies the width=8 butterfly
with a single add/sub pair. This is the correct recursive decomposition — FWHT of size 2N is:
apply FWHT-N to both halves, then butterfly between them. The implementation is two function calls
plus two SIMD instructions. Clean.

## 5.2% improvement

This is smaller than the 8-lane bootstrap (57%) because the 8-lane step already eliminated the
three worst stages. The 16-lane step removes one more outer-loop pass (width=8 → width=16). At
dim=2048, that's one fewer read-write of the full 8 KiB array. The 5% number is consistent with
removing one of ~6 remaining outer passes (each costing roughly the same).

## Scalar baseline instability

The packet honestly reports that an earlier scalar run suggested 3x+ but reruns didn't hold it.
This is the right call — BC-008 should be evaluated on stable, repeatable numbers. The unstable
scalar baseline on this WSL2 host is likely caused by frequency scaling or contention from the
host OS. If BC-008 closure matters, the definitive measurement should be on dedicated hardware
or at minimum with CPU frequency pinned.
