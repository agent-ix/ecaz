# Review: SPIRE DML Relation Context Hardening (30852)

## Summary

Closes both P2 items from the 30850 review:

1. **Multi-`ec_spire`-index rejection**: instead of silently picking
   the lowest-OID index, the helper now errors with
   `ec_spire DML frontdoor v1 requires at most one ec_spire index
   per heap relation`. Operators with two ec_spire indexes on one
   table see a clear v1 limitation, not silent route-by-OID.
2. **INCLUDE-column regression fixture**: a new test creates an
   `ec_spire` index with `INCLUDE (source_identity) WITH
   (source_identity = 'include')` and asserts the relation
   context's `embedding_columns` returns only `embedding`, locking
   the `idx.indnkeyatts` boundary against future regression.

ADR-069 now pins:
- at most one `ec_spire` index per distributed heap relation;
- one `bigint` primary-key column;
- composite and non-`bigint` PKs are deferred and fail closed.

The three review-focus questions:
- Multi-index fail-closed error is the right v1 behavior: yes.
- INCLUDE-column fixture covers the `indnkeyatts` regression risk:
  yes — directly tests the boundary.
- ADR-069 pins single-index / single-column-bigint-PK rule clearly:
  yes.

## Findings

### P3 — Multi-index error message uses "v1 requires"

`ec_spire DML frontdoor v1 requires at most one ec_spire index per
heap relation` — clear and version-scoped. Future packets that
relax the constraint can update both the message and the ADR
together. Worth a SQLSTATE so callers can catch programmatically;
defaults to `XX000` otherwise.

### P3 — Composite-PK rejection text

ADR-069 documents composite PK rejection but the relation-context
status currently maps to `unsupported_pk_shape`. Worth confirming
the SQL diagnostic surface text uses an operator-friendly
description (e.g., "v1 requires single bigint primary-key column;
got composite/uuid/...") rather than the bare status code.

### P3 — INCLUDE-column fixture exercise

Worth a parallel test that creates a *non*-INCLUDE ec_spire index
on a multi-column key (if the AM ever supports that) so a future
contributor adding multi-column ec_spire support sees a clear test
to update.

## Reviewer recommendation

Accept.
