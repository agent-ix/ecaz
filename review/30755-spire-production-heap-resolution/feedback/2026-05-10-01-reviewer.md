## Answers to the packet's review questions

- **Origin-node heap visibility boundary keeps remote locators
  opaque enough for final AM tuple delivery?** Yes —
  definitively. ADR-059 mandated that origin-node heap
  resolution is the *only* path that decodes remote row
  locators; the coordinator must treat them as opaque bytes.
  This implementation upholds that contract: locators travel
  through the executor as opaque bytes, the origin SQL surface
  resolves them under its own snapshot, and the coordinator
  receives only post-visibility heap diagnostics. Final AM
  tuple delivery (still open) inherits this discipline.
- **Production executor state transition from
  `CandidateReceiveReady` to `RemoteHeapReady` /
  `RemoteHeapFailed` correct under strict and degraded
  consistency?** Yes for strict (visible → ready, missing →
  fail-closed). Need to confirm degraded mode treats
  `RemoteHeapFailed` consistently with the fault matrix from
  packet 30751 — see P3 below.
- **Summary/proof surface acceptable as a Stage D checkpoint
  while final `amrescan`/`amgettuple` result-stream is open?**
  Yes. Same shape as packet 30754's scan handoff: SQL surface
  for reviewer/operator inspection, AM integration deferred
  to a separate slice. Packet 30756 starts that AM-side state
  with the result-stream refactor.

## P3 — informational

1. **Degraded-mode behavior for `RemoteHeapFailed`.** The
   strict-mode missing-row fixture asserts
   `remote_heap_resolution_failed`. The fault matrix from
   packet 30751 reserved Stage D categories; worth confirming
   that the next test fixture (or this one extended) verifies
   degraded mode treats a missing remote row as `skip_node`
   on that dispatch rather than fail-closed. Without it, the
   matrix's degraded row for `remote_heap_resolution_failed`
   isn't exercised by tests.

2. **Origin-node visibility guarantee.** The origin node
   resolves visibility under its PostgreSQL snapshot. Worth
   confirming in design doc commentary that the origin-node
   snapshot is the *coordinator's* requested-epoch snapshot
   semantics — i.e. that origin uses `MVCC` against the same
   epoch the coordinator asked for, not just whatever local
   snapshot the origin happens to have. ADR-059 likely
   already says this; cross-link.
