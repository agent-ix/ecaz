# Review: PG18 Contributor Shared Counters

**Verdict: Approved**

## What was read

- `git show 368a31e` full diff (parallel.rs, scan.rs, explain.rs)
- `record_parallel_scan_contributor_hidden_publish`, `record_parallel_scan_contributor_duplicate_retire`, `read_parallel_scan_contributor_counters`
- `refresh_parallel_contributor_explain_counters_from_shared_state` and its call sites in `amgettuple`

## Summary

The contributor counters are promoted from backend-local `explain_counters` fields into two `AtomicU32` fields in `EcParallelCoordinatorState` (no coordinator lock). The elected emitter folds the shared DSM totals into its backend-local `explain_counters` at each `amgettuple` call via `refresh_parallel_contributor_explain_counters_from_shared_state`.

## Findings

**Lock-free shared counters.** `contributor_hidden_publishes` and `contributor_duplicate_retires` use `AcqRel` on increment and `Acquire` on load. These are monotonically incrementing advisory counters — no coordinator lock is needed, and the only contract is that the emitter eventually sees the contributor's increments. `AcqRel/Acquire` provides that visibility. Correct.

**Fold strategy: `max(shared, local)`.** The emitter may have a prior local snapshot value (e.g., if a previous `amgettuple` refresh already loaded a larger value). Using `max` ensures the local counter only goes up when the shared counter exceeds the cached local value. Since the emitter never directly increments these counters locally (only contributors do), the local value before the first fold is always 0. After the first fold it tracks the shared value. The `max` is correct and idempotent.

**Fold called at both `amgettuple` return paths.** One refresh call before the `IndexScanDesc` read (loop exit) and one at the normal production path. Both execution paths through `amgettuple` pick up contributor progress. Correct.

**`explain_counters_from_index_scan_state` refactor.** Changed from `unsafe { (*opaque.cast::<TqScanOpaque>()).explain_counters }` to borrowing through a named reference. Same semantics (TqExplainCounters is Copy), cleaner code.

**DSM version bump.** 15 → 16 for the two new `AtomicU32` fields in `EcParallelCoordinatorState`. Reset path initializes both to 0 in `reset_parallel_scan_layout`. Correct.

**Diagnostic lane confirmation.** The artifact shows `Parallel Contributor Hidden Publishes: 8 / Parallel Contributor Duplicate Retires: 4` in the contributor diagnostic lane — non-emitting workers are publishing hidden rows and the elected emitter is reporting the shared totals. Default lane still shows zeros, confirming the counters are gated behind the contributor diagnostic env.

## No blockers.
