## Summary

CLI-only diagnostic change. The ordered query now collects `(id, score)` pairs, which lets the CLI derive both the ID list and the projected ORDER BY scores in a single round-trip. The serial HNSW order and the candidate order are compared for score-level adjacent inversions, extending the existing exact-score inversion diagnostic introduced in packet 593.

Any future `Gather Merge`-compatible design must either preserve HNSW rank through a sequencer or switch the parallel path to an explicit exact-key-sorted result contract.
