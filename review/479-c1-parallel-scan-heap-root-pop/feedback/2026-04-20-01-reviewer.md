## Verification performed

- `pop_parallel_scan_heap_root_with_attachment` (`parallel.rs:897–932`):
  captures root slot index, promotes the last entry to the root (when
  the heap has >1 entry), invalidates the vacated tail, decrements
  `live_entry_count`, bumps `heap_generation`, then sifts the new root
  down. Correct heap-pop primitive.
- `take_parallel_scan_selected_result_slot_snapshot`
  (`parallel.rs:1514–1534`): after a successful clear with
  `refresh_selection_snapshot = false`, pops the heap root and compares
  the popped slot index against the cleared slot. If mismatch (heap
  drifted), falls back to full `refresh_coordinator_selection_snapshot`.
  On match, calls `refresh_coordinator_selected_result_fast_path` —
  the narrow drain path. Correct.
- `refresh_coordinator_selection_snapshot` (`parallel.rs:975–981`):
  now composed of reap → rebuild → fast-path refresh. Clean separation.

## Findings

### 1. Same non-atomic `ptr::swap` concern carries forward

**Severity:** Medium (deferred blocker; no-op today)
**File:** `src/am/common/parallel.rs:778–784`

`swap_heap_entries` still uses `unsafe { std::ptr::swap(lhs, rhs) }`
on raw `*mut u32`. This is the same issue flagged in packets 472 and
478 and now called from `sift_up_heap_entry`, `sift_down_heap_entry`,
**and** the pop path. The cumulative must-fix-before-`amcanparallel=true`
item just grew another caller. No new risk today (single writer), but
the LWLock-around-refresh plan must now also cover
`pop_parallel_scan_heap_root_with_attachment`.

### 2. Drift-fallback on popped slot mismatch is the right safety net

**Severity:** Informational (positive)
**File:** `src/am/common/parallel.rs:1524–1532`

The `popped_slot_index != slot_index` check catches the case where
clear drops a slot that was not the heap root (e.g. selection cache
and heap root diverged mid-flight) and falls back to full refresh.
This preserves correctness while letting the common path stay fast.

### 3. `clear_parallel_scan_result_slot_with_attachment` bool grows a second meaning

**Severity:** Low
**File:** `src/am/common/parallel.rs:1190–1236`

The new `refresh_selection_snapshot` bool now means "skip both reap
AND rebuild AND fast-path refresh — caller will do the narrow drain."
Packet 475's suggestion to split into two named entry points
(`clear_and_refresh_full` / `clear_without_refresh`) gets more weight
with this second consumer. Optional; still correct.

### 4. Heap `live_entry_count` store → `heap_generation` bump ordering

**Severity:** Informational (positive)
**File:** `src/am/common/parallel.rs:923–928`

`live_entry_count.store(Release)` precedes
`heap_generation.fetch_add(AcqRel)`. Readers doing
generation-based invalidation will see the count update first, then
the generation bump — correct ordering for the single-writer case.
When concurrent writers land, readers will still need the two-pass
retry that already exists in
`read_parallel_scan_selected_result_slot_snapshot`.

### 5. Pop primitive doesn't update reverse membership

**Severity:** Informational (for 480 context only)

`pop_parallel_scan_heap_root_with_attachment` writes the replacement
slot index into entry 0 directly without touching any slot-owned
heap-membership field. That's fine for this packet (no such field
exists yet), but packet 480 adds reverse membership — confirming
there that the pop path is superseded/kept in sync.
