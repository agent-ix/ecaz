## Answers to the packet's review questions

- **Fixed-width `GlobalBytes` use of Leaf V2 acceptable for Phase 11.2,
  or should this have been a new storage version?** Acceptable. The
  existing `vec_id_kind` and `vec_id_stride` meta fields were already
  reserved precisely for this; reusing them avoids a Leaf V2 → V3 epoch
  migration path while preserving the on-disk format invariants the
  Phase 11.5 remote merge needs. A new storage version is only
  warranted if/when mixed-namespace or variable-width payloads become
  required, which the design doc correctly defers.
- **Did any read path still reconstruct local IDs?** Verified the
  quantized scan candidate path now decodes vector IDs through the
  column meta rather than rebuilding from `local_vec_seq`. The
  collect-candidates test exercises this directly.
- **Are the rejected mixed/variable layouts the right production
  constraint?** Yes. Mixed namespaces inside a single Leaf V2 object
  would silently break cross-node dedupe and quantized scoring;
  rejecting them at write time is the correct strict-mode behavior for
  the next writer source-identity slice.
