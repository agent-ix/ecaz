## Verification performed

- Bonus subtlety at scan.rs:3416–3421: if the blocker's element

- Unit coverage pins the four documented control-flow shapes
  (AdmissionWindow drop, first-foreign retry, stable-foreign
  fallback, changed-generation reopen).

## Findings

### 1. Element-tid keying is the right local boundary

Retaining by element rather than by blocker kind means a *new*
staged row gets a fresh retry budget, while the *same* staged row
facing the *same* foreign state correctly falls back. This
prevents both starvation (row held forever) and churn (retrying
indefinitely on unchanged state).

### 2. Foreign-duplicate drop is a quiet but important case

The `blocker_element_tid == element_tid` short-circuit catches the
case where a peer already has the same element as an owned output —
trying to emit it would be a duplicate. Drop-and-continue is
correct. It would be easy to miss without the test.
