# Review: Task 41 HNSW graph debug index guards (31183)

## Summary

Replaces manual `index_open` / `index_close` lifetimes with the
shared `IndexRelationGuard::access_share` from
`src/storage/relation_guard.rs` in three no-scan graph debug
helpers (`debug_all_top_level_heap_tids`,
`debug_top_level_reachable_heap_tids`,
`debug_layer0_reachable_live_element_tids`). Also adds the
module-level doc comment to `relation_guard.rs` documenting the
split between low-level resource ownership and AM-validation
helpers (addressing 31180 finding #2).

## Confirmations

- **Uses shared primitive.** No new module-local guard struct.
  The three helpers call `IndexRelationGuard::access_share(oid,
  "...")` and pass `guard.as_ptr()` into the graph helpers. ✔
- **Doc comment lands as proposed.** New top of
  `src/storage/relation_guard.rs`:
  ```
  //! Low-level RAII wrappers for PostgreSQL relation handles.
  //!
  //! AM-type validation lives in the `open_valid_ec_*_index_guard`
  //! helpers, which build on `IndexRelationGuard::access_share`.
  ```
  Closes my 31180 finding #2. ✔
- **Guard lifetime covers graph loader calls.** Each helper opens
  the guard once, runs graph traversal that loads adjacency /
  elements through raw pointer reads, then drops at function
  return. Same lifetime pattern as 31181 / 31171.
