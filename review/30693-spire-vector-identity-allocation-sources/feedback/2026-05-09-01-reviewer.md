## Acceptance

The hook is the right shape: global IDs do not leak into the local
sequence, malformed payloads do not advance state, and replicas share
identity by construction. Local-default wrappers preserve the v1 writer
path.

## P2 — must close before remote merge claims

1. **Live writer source-identity input.** As the packet notes, AM build
   and insert paths still default to `AllocateLocal` because there is no
   stable source-identity input plumbed through the AM layer. Until
   that lands, no live `ec_spire` index produces global IDs and the
   Phase 11.5 remote merge dedupe contract cannot be tested end to end.
   Open the next 11.2 slice to thread source identity from the writer
   into the assignment builders, or pin its design ADR before 11.3
   opens.

2. **Leaf V2 base-object format gap.** Leaf V2 still rejects global
   `0x02` IDs because the vec-id column is fixed to the local format.
   Phase 11.2 must land either a Leaf V2 format bump or an explicit
   compatibility shim before any boundary replica or remote endpoint
   can claim global-ID end-to-end. Recommend a small storage-format
   ADR alongside the next 11.2 slice.

## P3 — informational

3. **Observe contract for global IDs.** `observe` returns early on
   `local_sequence() == None`. That is correct for the local sequence
   invariant, but a future allocator may want to track the highest seen
   *global* identity payload kind for diagnostics (e.g. mixed-format
   index detection). Worth a comment noting the deliberate no-op rather
   than leaving it implicit.

4. **Default trait clarity.** `Default for SpireVecIdSourceIdentity`
   returns `AllocateLocal`. That matches existing call sites, but the
   default makes silent local fallback easy when adding new builders.
   Consider removing `Default` and forcing call sites to be explicit
   once the live writer plumbing lands; that would catch the next
   round of "forgot to thread identity" bugs at compile time.
