# Review: Task 41 SPIRE maintenance heap relation guard consolidation (31187)

## Summary

Removes the module-local `SpireHeapRelationGuard` from
`src/am/ec_spire/coordinator/lifecycle.rs`. SPIRE coordinator
maintenance split path now uses a helper that resolves the heap
relation OID from the index relation and returns the shared
`HeapRelationGuard`. Maintenance call sites use
`heap_relation.as_ptr()` instead of the local guard's
`relation()` method. Closes the 31180 finding item 5.
Baseline 4,256 → 4,254 (−2).

## Confirmations

- **Module-local struct deleted.** `grep -rE 'struct SpireHeapRelationGuard' src/`
  returns nothing. ✔
- **`HeapRelationGuard` used consistently.** Both lifecycle.rs
  open path and maintenance.rs consumer use the shared type;
  signature `heap_relation: &HeapRelationGuard` is compiler-
  enforced (Rust borrow with real lifetime, not a raw pointer).
  Strict safety upgrade over the prior `relation()` method on
  the module-local guard.
- **OID resolution helper.** The new helper opens the index
  guard, copies `rd_index.indrelid`, opens the heap guard, and
  returns the heap guard. Mirrors the
  `ec_spire_index_heap_relation_oid` pattern from 31166 but
  returns the open heap relation rather than just the OID.
  Justified by the maintenance path needing the live heap
  relation, not just the OID.
