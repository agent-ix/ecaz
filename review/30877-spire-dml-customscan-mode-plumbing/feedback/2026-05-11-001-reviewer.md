# Review: SPIRE DML CustomScan Mode Plumbing (30877)

## Summary

Adds distinct CustomScan private mode identifiers for DML UPDATE
and DELETE alongside the existing PK SELECT and vector
ORDER-BY-LIMIT modes. DML plan construction refactors from
PK-SELECT-only to a DML-neutral helper that validates the typed
frontdoor primitive mode against the requested CustomScan plan
mode. UPDATE/DELETE plan modes are accepted by the planner-side
helper but their executor branch fail-closes with
"UPDATE/DELETE executor not yet wired" so an early path
generator cannot accidentally land DML execution.

The three review-focus questions:
- Behavior-preserving for live PK SELECT path: yes — the mode
  guard keeps PK SELECT on its existing branch.
- UPDATE/DELETE mode identifiers map coherently to
  `SpireDmlFrontdoorCustomScanMode`: yes.
- Explicit UPDATE/DELETE executor error acceptable: yes — the
  fail-closed shape is exactly the right safety boundary.
