## Notes

### 1. Good boundary for the planner lane

`tqhnsw_index_admin_snapshot(regclass)` is a sensible current seam for planner/admin scaffolding.
It gives the planner/productization lane something concrete to build around without touching the
runtime execution lane in `scan.rs`.

### 2. `NULL` for `inserted_since_rebuild` is the right temporary contract

Returning `NULL` is better than inventing a fake value. It makes the staging status explicit and
avoids training later work around a placeholder metric that means nothing.

### 3. Keep the surface descriptive, not prescriptive

This function should keep reporting current state and tuning sources. It should not become a back
door for planner enablement decisions while ADR-011 is still in force.

Good fields for now:
- effective `ef_search`
- tuning source
- planner-gate state
- live-node count / block count

Avoid for now:
- inferred readiness claims
- approximate recall/quality metrics
- placeholders that look like real maintenance drift accounting

## Net

This looks like useful scaffolding and a good fit for the planner lane. The main integration
caution is still the same as the companion request: rebase onto current `main` so the broader PG18
spec surface is preserved when this branch lands.
