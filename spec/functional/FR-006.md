# FR-006: SQL Operators and Operator Class

## Requirement

The extension SHALL register SQL operators and an operator class for HNSW index integration.

### Operator: `<#>`

```sql
CREATE OPERATOR <#> (
    leftarg   = tqvector,
    rightarg  = tqvector,
    procedure = tqvector_negative_inner_product,
    commutator = <#>
);
```

- Returns negative inner product (for ORDER BY ASC = highest similarity first)
- Commutative: `a <#> b = b <#> a`

### Operator Class: `tqvector_ip_ops`

```sql
CREATE OPERATOR CLASS tqvector_ip_ops DEFAULT FOR TYPE tqvector
    USING tqhnsw AS
    OPERATOR 1 <#>(tqvector, tqvector) FOR ORDER BY float_ops,
    FUNCTION 1 tqvector_inner_product(tqvector, tqvector);
```

- Default operator class for `tqvector` under the `tqhnsw` access method
- OPERATOR 1 is the ordering operator
- FUNCTION 1 is the distance function used by the index AM

## Acceptance Criteria

### FR-006-AC-1: Operator usable in ORDER BY
`SELECT * FROM t ORDER BY col <#> $query LIMIT 10` SHALL parse and execute.

### FR-006-AC-2: Index scan chosen
EXPLAIN of the above query on an indexed table SHALL show an Index Scan using `tqhnsw`.

### FR-006-AC-3: Operator commutativity
`a <#> b` SHALL equal `b <#> a`.
