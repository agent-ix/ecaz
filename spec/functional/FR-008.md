# FR-008: HNSW Index Access Method — Build and Insert

## Requirement

The extension SHALL implement the `ambuild` and `aminsert` callbacks for the `tqhnsw` access method.

### `ambuild` — Bulk Index Build

Called by `CREATE INDEX ... USING tqhnsw`.

1. Read index options: `(m, ef_construction)` from WITH clause, `(dim, bits, seed)` from the column type
2. Initialize `hnsw_rs::Hnsw` with M and ef_construction
3. Scan the heap relation: for each row, extract the `tqvector` code bytes
4. Insert each vector into the `hnsw_rs` in-memory graph
5. Serialize the completed graph to Postgres pages (TqElementTuple + TqNeighborTuple per node)
6. Write metadata page (page 0)
7. All page writes SHALL use GenericXLog for WAL safety

### `ambuildempty` — Empty Index

Called for `CREATE INDEX` before any data exists:
- Write metadata page with null entry point
- No element or neighbor pages

### `aminsert` — Single Row Insert

Called when a row is INSERTed into a table with an existing tqhnsw index:

1. Extract tqvector from the inserted datum
2. Search the existing graph for neighbors using `hnsw_rs` beam search
3. Allocate a new TqElementTuple on an index page
4. Allocate a corresponding TqNeighborTuple
5. Write neighbor back-links: update the neighbor tuples of the new node's neighbors
6. All page writes SHALL use GenericXLog

### Index Parameters (WITH clause)

| Parameter | Default | Range | Description |
|---|---|---|---|
| `m` | 8 | 2–100 | Max neighbors per layer |
| `ef_construction` | 64 | 10–1000 | Build-time beam width |

## Acceptance Criteria

### FR-008-AC-1: Bulk build populates graph
After `CREATE INDEX ... USING tqhnsw` on a table with 1000 rows, the index SHALL contain 1000 element tuples.

### FR-008-AC-2: Insert updates graph
After inserting a new row into an indexed table, the new vector SHALL be reachable via HNSW search.

### FR-008-AC-3: Crash safety
If the server crashes during ambuild, the partially-built index SHALL be automatically cleaned up on recovery (standard Postgres behavior for unfinished index builds).

### FR-008-AC-4: GenericXLog usage
Every page modification in ambuild and aminsert SHALL be wrapped in GenericXLogStart/GenericXLogFinish.
