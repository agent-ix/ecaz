# FR-013: Two-Stage Vector Quantization Pipeline

## Requirement

The extension SHALL implement the TurboQuant two-stage vector quantization algorithm as an internal module (`quant/`). The quantizer compresses fp32 vectors into bit-packed integer codes with provably unbiased inner product estimation.

### Stage 1: MSE Quantization (b-1 bits per dimension)

#### 1a. SRHT Rotation

Before quantization, each input vector SHALL be rotated using a Subsampled Randomized Hadamard Transform (SRHT):

1. Pad the input vector to the next power-of-two length in a temporary transform workspace
2. Apply a diagonal sign matrix: multiply each coordinate by a pseudorandom sign (+1 or -1) determined by `seed`
3. Apply the Fast Walsh-Hadamard Transform (FWHT) in-place
4. Scale by `1 / sqrt(dim)`

The SRHT rotation spreads energy uniformly across coordinates, making the per-coordinate distribution approximately Beta on [-1, 1]. This enables a single data-oblivious codebook to be used for all coordinates.

#### 1b. Lloyd-Max Codebook Generation

The extension SHALL generate `2^(bits-1)` optimal scalar quantizer centroids using the Lloyd-Max algorithm:

1. The target distribution is the Beta PDF: `f(x) = C * (1 - x^2)^((d-3)/2)` on [-1, 1], parameterized by vector dimension `d`
2. Centroids are initialized uniformly and iteratively refined to minimize MSE
3. Convergence criterion: maximum centroid shift < 1e-7, or 1000 iterations
4. The codebook is **fully determined by (original_dim, bits)** — no training data needed

#### 1c. Scalar Quantization

Each original-dimension rotated coordinate SHALL be quantized to its nearest codebook centroid. The index of the nearest centroid (0 to `2^(bits-1) - 1`) is stored as a `CodeIndex` (u16). These indices are bit-packed into `ceil(original_dim * (bits-1) / 8)` bytes. Any padded transform coordinates are scratch-space only and SHALL NOT be persisted.

### Stage 2: QJL Residual Correction (1 bit per dimension)

1. Compute the quantization residual: `residual = rotated_vector - quantized_approximation`
2. Project the residual through a random Gaussian matrix (generated from `seed` via ChaCha20 PRNG)
3. Store the sign of each projection as a single bit (1 = positive, 0 = negative)
4. Bit-pack the signs for the original dimensionality into `ceil(original_dim / 8)` bytes

The QJL (Quantized Johnson-Lindenstrauss) correction preserves directional information about the residual, improving inner product estimation accuracy.

### Combined Output

The packed code format is:
```
[mse_packed: ceil(original_dim * (bits-1) / 8) bytes][qjl_packed: ceil(original_dim / 8) bytes]
```

At 1536-dim, 4-bit: 576 + 192 = **768 bytes** (8x compression from 6,144 bytes fp32).

### Data-Oblivious Property

The quantizer SHALL require **no training data, no calibration, and no warm-up**. Given `(original_dim, bits, seed)`, the codebook and projection matrix are fully determined. The first vector encoded after extension installation produces valid codes.

### Decode (Approximate)

The extension SHALL provide approximate decoding for diagnostics and prepared-query construction:
1. Unpack MSE indices, map each to its codebook centroid
2. Apply inverse SRHT rotation (inverse FWHT + inverse diagonal signs)
3. The decoded vector is an approximation — quantization error is not recoverable

### Estimator Fidelity Contract

The implementation SHALL port the exact packed-code estimator semantics from the chosen TurboQuantDB reference implementation for:
- prepared-query LUT scoring
- code-to-code scoring
- QJL correction and its normalization

The extension SHALL NOT substitute a different heuristic correction term unless the specification is updated accordingly.

## Acceptance Criteria

### FR-013-AC-1: Codebook centroids are symmetric
For any even number of centroids, the codebook SHALL be symmetric around zero (sum of all centroids < 1e-3).

### FR-013-AC-2: Beta PDF integrates to one
The Beta PDF for any dimension d >= 2 SHALL integrate to 1.0 ± 0.001 over [-1, 1].

### FR-013-AC-3: SRHT rotation preserves norm
`||SRHT(v)|| == ||v||` within floating-point tolerance (< 1e-5 relative error).

### FR-013-AC-4: Encode-decode round-trip fidelity
For 1536-dim 4-bit encoding, the cosine similarity between the original and decoded vector SHALL be > 0.85 on average over random unit vectors.

### FR-013-AC-5: Deterministic encoding
Encoding the same vector with the same `(bits, seed)` SHALL produce byte-identical codes.

### FR-013-AC-6: QJL bit count correctness
The QJL output SHALL be exactly `ceil(original_dim / 8)` bytes for any input dimension.
