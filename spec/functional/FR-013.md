# FR-013: Two-Stage Vector Quantization Pipeline

## Requirement

The extension SHALL implement the `tqvector` two-stage vector quantization algorithm as an internal module (`quant/`). The quantizer compresses fp32 vectors into bit-packed integer codes plus a residual norm scalar, and it defines both a high-quality query-to-code estimator and a lower-fidelity symmetric code-to-code estimator.

### Stage 1: MSE Quantization (b-1 bits per dimension)

#### 1a. SRHT Rotation

Before quantization, each input vector SHALL be rotated using a Subsampled Randomized Hadamard Transform (SRHT):

Let `d` be the original input dimension and `n = next_power_of_two(d)`.

1. Pad the input vector with zeros to length `n` in a temporary transform workspace
2. Apply a diagonal sign matrix: multiply each coordinate by a pseudorandom sign (+1 or -1) determined by `seed`
3. Apply the Fast Walsh-Hadamard Transform (FWHT) in-place
4. Scale by `1 / sqrt(n)`

The SRHT rotation spreads energy uniformly across coordinates, making the per-coordinate distribution approximately Beta on [-1, 1]. This enables a single data-oblivious codebook to be used for all coordinates.

#### 1b. Lloyd-Max Codebook Generation

The extension SHALL generate `2^(bits-1)` optimal scalar quantizer centroids using the Lloyd-Max algorithm:

1. The target distribution is the Beta PDF: `f(x) = C * (1 - x^2)^((d-3)/2)` on [-1, 1], parameterized by vector dimension `d`
2. Centroids are initialized uniformly and iteratively refined to minimize MSE
3. Convergence criterion: maximum centroid shift < 1e-7, or 1000 iterations
4. The codebook is **fully determined by (original_dim, bits)** — no training data needed

#### 1c. Scalar Quantization

Each of the first `d` rotated coordinates SHALL be quantized to its nearest codebook centroid. The index of the nearest centroid (0 to `2^(bits-1) - 1`) is stored as a `CodeIndex` (u16). These indices are bit-packed into `ceil(d * (bits-1) / 8)` bytes. The remaining `n - d` rotated coordinates are discarded from the persisted representation and SHALL be treated as zeros during MSE-only reconstruction.

### Stage 2: QJL Residual Correction (1 bit per dimension)

1. Reconstruct the MSE-only approximation in the original input domain:
   - place the persisted `d` centroid values into positions `[0, d)` of an `n`-vector
   - fill positions `[d, n)` with zero
   - apply inverse SRHT and truncate back to `d`
2. Compute the residual in the original input domain:
   `r = x - x_tilde_mse`
3. Compute the residual norm:
   `gamma = ||r||_2`
4. Apply the QJL projection to the residual: `s_r = Q_seed(r)` where `Q_seed` is the seeded SRHT projection for the QJL stage
5. Store the sign of each of the first `d` projected coordinates as a single bit (1 = positive, 0 = negative)
6. Bit-pack the signs into `ceil(d / 8)` bytes

The QJL (Quantized Johnson-Lindenstrauss) correction preserves directional information about the residual, improving inner product estimation accuracy.

### Combined Output

The packed payload format is:
```
[gamma: 4 bytes][mse_packed: ceil(d * (bits-1) / 8) bytes][qjl_packed: ceil(d / 8) bytes]
```

At 1536-dim, 4-bit: 4 + 576 + 192 = **772 bytes payload** (783 bytes including the 11-byte datum header).

### Data-Oblivious Property

The quantizer SHALL require **no training data, no calibration, and no warm-up**. Given `(original_dim, bits, seed)`, the codebook and projection matrix are fully determined. The first vector encoded after extension installation produces valid codes.

### Decode (Approximate)

The extension SHALL provide approximate decoding for diagnostics:
1. Unpack MSE indices and map them to centroid values
2. Form an `n`-vector whose first `d` positions are the centroid values and whose tail `[d, n)` is zero
3. Apply inverse SRHT rotation and truncate to `d`
4. Optionally add the decoded QJL residual approximation `gamma * sqrt(pi / (2d)) * inverse_qjl(signs)` when full reconstruction is requested
5. The decoded vector is an approximation — quantization error and discarded transform-tail information are not recoverable

### Estimator Contract

The extension SHALL implement the following two estimators exactly as specified:

#### Query-to-Code Estimator

For raw query `q ∈ R^d` and candidate code `(gamma, idx[0..d), z[0..d))`:

1. Compute `y_q = R_seed(q)` where `R_seed` is the MSE SRHT rotation into `R^n`
2. Compute `s_q = Q_seed(q)` where `Q_seed` is the QJL projection into `R^n`
3. MSE term:
   `mse(q, x) = Σ_{i=0}^{d-1} centroid[idx_i] * y_q[i]`
4. QJL term:
   `qjl(q, x) = gamma * (sqrt(pi / 2) / d) * Σ_{i=0}^{d-1} s_q[i] * sign(z_i)`
5. Final score:
   `score_query_to_code(q, x) = mse(q, x) + qjl(q, x)`

#### Code-to-Code Estimator

For two stored codes `a = idx_a[0..d)` and `b = idx_b[0..d)`:

`score_code_to_code(a, b) = Σ_{i=0}^{d-1} centroid[idx_a_i] * centroid[idx_b_i]`

The symmetric code-to-code estimator SHALL NOT apply the QJL residual term in v0.1.

## Acceptance Criteria

### FR-013-AC-1: Codebook centroids are symmetric
For any even number of centroids, the codebook SHALL be symmetric around zero (sum of all centroids < 1e-3).

### FR-013-AC-2: Beta PDF integrates to one
The Beta PDF for any dimension d >= 2 SHALL integrate to 1.0 ± 0.001 over [-1, 1].

### FR-013-AC-3: SRHT rotation preserves norm
`||SRHT(v)|| == ||v||` within floating-point tolerance (< 1e-5 relative error).

### FR-013-AC-4: Encode-decode round-trip fidelity
For 1536-dim 4-bit encoding, the cosine similarity between the original and decoded vector SHALL be > 0.85 on average over random unit vectors.

### FR-013-AC-5: Deterministic encoding
Encoding the same vector with the same `(bits, seed)` SHALL produce byte-identical codes.

### FR-013-AC-6: QJL bit count correctness
The QJL output SHALL be exactly `ceil(original_dim / 8)` bytes for any input dimension.
