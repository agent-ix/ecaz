# FR-004: encode_to_tqvector — fp32 to Compressed Code

## Requirement

The extension SHALL provide an SQL-callable function that compresses a raw fp32 vector into a `tqvector` value using the `turbo-quant` crate.

### Signature

```sql
encode_to_tqvector(embedding float4[], bits int, seed bigint DEFAULT 42) RETURNS tqvector
```

### Behavior

1. SHALL call `turbo-quant` crate's encode function with the given parameters
2. SHALL reject `bits` outside range 2–8 with ERROR
3. SHALL accept any positive dimensionality supported by the `turbo-quant` crate
4. SHALL return a valid `tqvector` value that round-trips through text and binary I/O
5. SHALL be marked `IMMUTABLE`, `STRICT`, `PARALLEL SAFE`

### Determinism

Given the same `(embedding, bits, seed)`, the function SHALL always produce the same output. This is guaranteed by the data-oblivious nature of TurboQuant — the quantizer is fully determined by `(dim, bits, seed)`.

## Acceptance Criteria

### FR-004-AC-1: Encode produces valid tqvector
`encode_to_tqvector(ARRAY[1.0, 2.0, 3.0, 4.0]::float4[], 4, 42)` SHALL return a value that can be stored and retrieved from a `tqvector` column.

### FR-004-AC-2: Deterministic output
Two calls with identical arguments SHALL produce byte-identical results.

### FR-004-AC-3: Reject invalid bits
`encode_to_tqvector(ARRAY[1.0]::float4[], 0, 42)` SHALL raise ERROR.
