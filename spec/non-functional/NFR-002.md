# NFR-002: Storage Compression

## Requirement

### Index Size Targets

At 1536-dim, 4-bit, per-agent with 50K vectors:

| m | Target Index Size | Comparison (pgvector fp32) |
|---|---|---|
| 8 | ≤ 34 MB | vs ~88 MB (2.6x smaller) |
| 4 | ≤ 17 MB | vs ~44 MB (2.6x smaller) |

### Per-Vector Storage

- Raw fp32: 6,144 bytes
- tqvector 4-bit code: 779 bytes (11 header + 768 codes)
- Compression ratio: ≥ 7.8x

### Disk-Level Target

1 million vectors at 1536-dim, 4-bit SHALL occupy < 1 GB of total index space (including graph overhead).

## Measurement

Report `pg_relation_size(index_oid)` after bulk loading known row counts.
