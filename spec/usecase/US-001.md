# US-001: Store a Compressed Embedding

**As** an application developer inserting agent memories,
**I want** to compress an fp32 embedding into a `tqvector` column during INSERT,
**So that** the vector is stored at 8–10x compression with no separate encoding step.

## Acceptance Criteria

1. `encode_to_tqvector(embedding float4[], bits int, seed bigint)` accepts a 1536-dim fp32 array and returns a `tqvector` value
2. The returned value can be stored in a `tqvector` column
3. No external process or training step is required before the first INSERT
4. Bits parameter accepts values 2–8; invalid values produce a clear error
